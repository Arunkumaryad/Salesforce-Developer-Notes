# What are sObjects in Salesforce? | sObject Data Type in Apex | sObject Class

## Overview

### What is an sObject?

An **sObject (Salesforce Object)** is a special Apex data type that represents a record from any Salesforce object.

An sObject can represent:

- Standard Objects (Account, Contact, Opportunity, Lead, Case, etc.)
- Custom Objects (Student__c, College__c, Employee__c, etc.)

### Definition

> Any standard or custom object is an sObject.

> sObject is used to represent any row/record from a standard or custom object.

---

# Why Use sObject?

Normally, we work with a specific object type:

```apex
Account acc = new Account();
Contact con = new Contact();
Student__c stu = new Student__c();
```

But Salesforce also provides a generic data type called **sObject** that can store records from any object.

```apex
sObject record;
```

This makes Apex code more dynamic and reusable.

## Collection Data Types

Salesforce supports three collection types:

### List

Stores ordered collection of records.

```apex
List<Account> accounts = new List<Account>();
```

---

### Set

Stores unique values.

```apex
Set<String> names = new Set<String>();
```

---

### Map

Stores key-value pairs.

```apex

```

# Using Custom Object as Data Type

Suppose we have a custom object:

```text
Student1__c
```

We can directly use it as a data type.

---

## Retrieve Records Using Object Type

### Example

```apex
// Retrieve all students

List<Student1__c> students = [
    SELECT Name,
           Roll_Number__c,
           Branch__c
    FROM Student1__c
];

for(Student1__c s : students)
{
    System.debug(
        'Name : ' + s.Name +
        ' Roll Number : ' + s.Roll_Number__c +
        ' Branch : ' + s.Branch__c
    );
}
```

### Output

```text
Name : Arun Roll Number : 11 Branch : Computer

Name : Kundan Roll Number : 1 Branch : IT

Name : Suddu Roll Number : 87 Branch : ME

Name : Pappu Roll Number : 23 Branch : CSE

Name : Arun Kumar Roll Number : 12 Branch : CSE
```

---

# Using sObject as Data Type

Instead of using:

```apex
List<Student1__c>
```

we can use:

```apex
List<sObject>
```

because every Salesforce object is an sObject.

---

## Example

```apex
List<sObject> students = [
    SELECT Name,
           Roll_Number__c,
           Branch__c
    FROM Student1__c
];

for(sObject s : students)
{
    System.debug(
        'Name : ' + s.get('Name') +
        ' Roll Number : ' + s.get('Roll_Number__c') +
        ' Branch : ' + s.get('Branch__c')
    );
}
```

---

# Accessing Fields Using get()

When using a specific object type:

```apex
student.Name
student.Roll_Number__c
```

When using sObject:

```apex
record.get('FieldName')
```

Example:

```apex
s.get('Name');

s.get('Roll_Number__c');

s.get('Branch__c');
```

---

# Type Casting with sObject

Since the get() method returns Object type, explicit casting can be performed.

Example:

```apex
for(sObject s : students)
{
    System.debug(
        'Name : ' + (String)s.get('Name') +
        ' Roll Number : ' + (Decimal)s.get('Roll_Number__c') +
        ' Branch : ' + (String)s.get('Branch__c')
    );
}
```

---

# Accessing Parent Object Using sObject

Suppose Student has a lookup relationship with College.

```text
Student1__c
      |
      ↓
College__c
```

---

## SOQL Query

```apex
List<sObject> students = [
    SELECT Name,
           Roll_Number__c,
           Branch__c,
           College__r.Name
    FROM Student1__c
];
```

---

## Access Parent Object Fields

Use:

```apex
getSObject()
```

method.

### Example

```apex
for(sObject s : students)
{
    System.debug(
        'Name : ' + s.get('Name') +
        ' Roll Number : ' + s.get('Roll_Number__c') +
        ' Branch : ' + s.get('Branch__c') +
        ' College : ' +
        s.getSObject('College__r').get('Name')
    );
}
```

---

## Output

```text
Name : Arun Roll Number : 11 Branch : Computer College : PTU

Name : Kundan Roll Number : 1 Branch : IT College : DTU

Name : Suddu Roll Number : 87 Branch : ME College : DTU

Name : Pappu Roll Number : 23 Branch : CSE College : AKTU

Name : Arun Kumar Roll Number : 12 Branch : CSE College : PTU
```

# Creating New sObject Instance Dynamically

One of the biggest advantages of sObject is dynamic object creation.

---

## Syntax

```apex
sObject record =
    (sObject)Type.forName('Object_API_Name')
    .newInstance();
```

---

## Example

```apex
sObject record =
    (sObject)Type.forName('Student1__c')
    .newInstance();

record.put('Name', 'Madhvi Sharma');

record.put('Roll_Number__c', 76);

record.put('Branch__c', 'AIML');

record.put(
    'College__c',
    'a1GgL00004nInK9UAK'
);

insert record;
```

---

# put() Method

The put() method is used to assign values dynamically.

### Syntax

```apex
record.put(
    'Field_API_Name',
    value
);
```

### Example

```apex
record.put('Name', 'Madhvi Sharma');

record.put('Roll_Number__c', 76);

record.put('Branch__c', 'AIML');
```

---

# get() vs put()

| Method | Purpose |
|----------|----------|
| get() | Retrieve field value |
| put() | Set field value |

---

## Example

```apex
record.put('Name', 'Arun');

String name =
    (String)record.get('Name');
```

# Difference Between Object Type and sObject Type

| Object Type | sObject Type |
|-------------|--------------|
| Strongly Typed | Generic |
| Direct field access | Uses get()/put() |
| Better compile-time checking | More flexible |
| Less dynamic | Highly dynamic |
| Faster to write | Better for frameworks |

# Interview Questions

## What is an sObject?

An sObject is a generic Apex data type that represents a record of any standard or custom Salesforce object.

---

## Is Account an sObject?

Yes.

Account is a standard object and every standard/custom object inherits from sObject.

---

## How do you access fields in sObject?

Using:

```apex
get('FieldName')
```

Example:

```apex
record.get('Name');
```

---

## How do you assign values dynamically?

Using:

```apex
put('FieldName', value);
```

---

## What is getSObject() used for?

It is used to access parent relationship records dynamically.

Example:

```apex
record.getSObject('College__r')
      .get('Name');
```

---

## Can we insert records using sObject?

Yes.

```apex
insert record;
```

works exactly like normal object records.

---

## What is the difference between get() and getSObject()?

| Method | Purpose |
|----------|----------|
| get() | Access field values |
| getSObject() | Access parent relationship records |

---
