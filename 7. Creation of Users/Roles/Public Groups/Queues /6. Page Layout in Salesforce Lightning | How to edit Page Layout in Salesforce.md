# Page Layout in Salesforce Lightning | How to Edit Page Layout in Salesforce

## Introduction

In Salesforce, **Page Layouts** control the layout and organization of buttons, fields, custom links, related lists, Visualforce pages, and sections that appear on an object's record page.

Page Layouts help administrators customize the user interface so users see only the information that is relevant to them.

---

# What is a Page Layout?

## Salesforce Definition

> Page layouts control the layout and organization of buttons, fields, s-controls, Visualforce, custom links, and related lists on object record pages. They also help determine which fields are visible, read only, and required. Use page layouts to customize the content of record pages for your users.

---

## Simple Definition

A **Page Layout** determines:

- Which fields appear on a record page
- The order of fields
- Which fields are required
- Which fields are read-only
- Which sections appear on the page
- Which related lists are displayed
- Which buttons and actions users can access

---

# Why Do We Use Page Layouts?

Page Layouts help us:

- Organize record information
- Improve user experience
- Hide unnecessary fields
- Make fields mandatory
- Make fields read-only
- Display related records
- Customize pages for different business processes

---

# Page Layout vs Lightning Record Page

| Page Layout | Lightning Record Page |
|------------|----------------------|
| Controls fields and sections | Controls page structure |
| Controls field visibility | Controls component visibility |
| Object-specific | App Builder based |

# Interview Questions

## What is a Page Layout in Salesforce?

A Page Layout controls the arrangement of fields, buttons, related lists, and sections on an object's record page.

---

## What can Page Layout control?

- Fields
- Sections
- Related Lists
- Buttons
- Quick Actions
- Read-Only Fields
- Required Fields

---

## Can a field be made required from a Page Layout?

Yes.

Using Field Properties, a field can be marked as Required.

---

## Can a field be made Read-Only from a Page Layout?

Yes.

Field Properties allow fields to be marked as Read-Only.

---

## What is the purpose of Sections?

Sections organize fields into logical groups.

---

## What is Page Layout Assignment?

Page Layout Assignment determines which layout is assigned to a profile.

---

## What is the difference between Page Layout and Record Page?

Page Layout controls fields and sections.

Lightning Record Page controls Lightning components and overall page design.

# Summary

Page Layouts in Salesforce are used to control how record pages appear to users. They determine which fields, buttons, related lists, and sections are displayed and how they are organized. Administrators can make fields required, read-only, add custom sections, and assign different layouts to different profiles. Properly designed Page Layouts improve usability, data quality, and user productivity across Salesforce applications.
