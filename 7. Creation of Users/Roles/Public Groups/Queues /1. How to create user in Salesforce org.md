# How to Create a User in Salesforce Org

## Introduction

In Salesforce, a **User** is an individual who can log in and access the Salesforce organization. Every user must have:

- A unique Username
- A valid Email Address
- A User License
- A Profile

Users are assigned permissions and access through Profiles, Roles, Permission Sets, and Sharing Settings.

---

# What is a User in Salesforce?

## Definition

A **User** is a person who can log in to a Salesforce organization and perform actions based on the permissions assigned through Profiles, Roles, Permission Sets, and Sharing Settings.

Examples:

- Salesforce Administrator
- Sales Representative
- Marketing User
- Support Agent
- Manager

---

# Steps to Create a New User

## Step 1: Open Setup

Click the **Gear Icon (⚙️)** in the top-right corner.

Select:

```text
Setup
```

---

## Step 2: Search for Users

In the Quick Find Search Box, type:

```text
Users
```

Select:

```text
Users
```

from the Setup Menu.

Navigation:

```text
Setup → Users → Users
```

---

## Step 3: Open Users Page

The Users page displays:

- Existing Users
- User Profiles
- Active Status
- User Licenses
- Roles

Click:

```text
New User
```

button.

---

# User Creation Form

The User Edit page opens.

Fill the required information.

---

## General Information Fields

### First Name

User's first name.

Example:

```text
Test
```

---

### Last Name

User's last name.

Example:

```text
User
```

---

### Alias

A short identifier automatically generated by Salesforce.

Example:

```text
tuser
```

---

### Email

User's valid email address.

Example:

```text
arunkumar2023712@gmail.com
```

Salesforce sends verification and password reset emails to this address.

---

### Username

A unique login name for Salesforce.

Example:

```text
arunkumar12023712@gmail.com
```

### Important

Username must be unique across all Salesforce organizations globally.

Even if the user belongs to a different Salesforce org, the username cannot be duplicated.

---

### Nickname

Used in Chatter.

Salesforce automatically generates it.

Example:

```text
User17804264054947279003
```

---

### Title

Optional designation.

Example:

```text
Sales Executive
```

---

### Company

Optional company name.

Example:

```text
ABC Technologies
```

---

### Department

Optional department name.

Example:

```text
Sales
```

---

# User Access Configuration

## Role

Determines record visibility hierarchy.

Example:

```text
Director, Channel Sales
```

Roles control:

- Record Access
- Data Visibility
- Reporting Hierarchy

---

## User License

Defines Salesforce features available to the user.

Example:

```text
Salesforce
```

Common Licenses:

- Salesforce
- Salesforce Platform
- Chatter Free
- Customer Community

---

## Profile

Defines user permissions.

Example:

```text
Standard User
```

Profiles control:

- Object Permissions
- Field Permissions
- App Access
- Tab Access

---

## Active Checkbox

```text
✓ Active
```

Checked:

- User can log in.

Unchecked:

- User cannot access Salesforce.

---

# Email Verification Process

After creating the user, Salesforce sends a verification email.

Subject:

```text

```

Email sent from:

```text
support@salesforce.com
```

---

## Verification Steps

Open the email.

Click:

```text
Verify Email Address
```

---

Salesforce displays:

```text
Please click the button below to verify your new email address and complete the change.
```

Click:

```text
Verify Email Address
```

# Account Verification Email

A second email may be received:

```text

Click below to verify your account.
```

Click:

```text

```

# Password Setup

Salesforce then redirects to:

```text

```


# Create Password

Enter:

### New Password

Must include:

- Minimum 8 characters
- At least 1 letter
- At least 1 number

---

### Confirm Password

Re-enter the password.

---

### Security Question

Select a question.

Example:

```text
In what city were you born?
```

---

### Security Answer

Example:

```text
Noida
```

---

Click:

```text

```

The user can log in successfully.

```text

```

Return to:

```text
Setup → Users → Users
```

You should see the newly created user in the user list.

Example:


with:

```text
Profile = Standard User
Role = Director, Channel Sales
```

Suppose a company hires a new Sales Executive.

# Interview Questions

## What is a User in Salesforce?

A User is an individual who can log in and access Salesforce based on assigned permissions.

---

## Can two users have the same Username?

No.

Salesforce usernames must be globally unique across all Salesforce organizations.

---

## What is the difference between Role and Profile?

### Profile

Controls:

- Object Permissions
- Field Permissions
- App Access

### Role

Controls:

- Record Visibility
- Record Sharing

---

## What happens if Active is unchecked?

The user cannot log in to Salesforce.

---

## Why is Email Verification required?

To verify ownership of the email address and activate the user account.

---

## What is a User License?

A User License determines which Salesforce features and functionality a user can access.

# Summary

A Salesforce User is an individual who can access the Salesforce organization. To create a user, navigate to Setup → Users → Users and click New User. Enter the required details such as Name, Email, Username, User License, Profile, and Role. After saving, Salesforce sends verification emails. Once the email is verified and a password is created, the user can successfully log in and access Salesforce according to the permissions assigned through Profiles, Roles, and other security settings.
