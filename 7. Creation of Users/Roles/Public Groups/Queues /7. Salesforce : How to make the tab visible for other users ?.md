# Salesforce : How to Make the Tab Visible for Other Users?

## Introduction

In Salesforce, users can only see tabs and objects if they have the required permissions through their **Profile**, **Permission Set**, and **Tab Settings**.

Sometimes a custom object is visible to the System Administrator but not visible to other users. This happens because the tab is not enabled in the user's Profile.

This guide explains how to make a tab visible for other users by cloning a profile, enabling tab visibility, assigning object permissions, and assigning the profile to the user.

---

# Problem Statement

Suppose you have created custom objects:

```text
Students
Teachers
Colleges
Staff Rooms
```

The tabs are visible for the System Administrator but are not visible for other users.

### Why?

Because:

- Tab visibility is not enabled.
- Object permissions are not assigned.
- User is using a profile without access.

---

# Salesforce Definition

## Profile

A Profile determines:

- Object permissions
- Field permissions
- Tab visibility
- Record type access
- App permissions
- User permissions

Every user must have exactly one profile.

---

To make tabs visible for another user:

### Step 1

# Understanding the Issue

The user currently has:


The Standard User profile does not contain access to the custom tabs and objects.


are not visible.

- Delete

inside Salesforce.

# Custom Tab Settings

Scroll down to:

```text

```

---

# Salesforce Tab Visibility Options

Each tab has a dropdown with three options.

---

## Default On

```text
Tab is visible by default.
```

Users can access the tab.

Example:

```text
Students = Default On
Teachers = Default On
Colleges = Default On
Staff Rooms = Default On
```

---

## Default Off

```text
Tab is hidden by default.
```

Users can manually add it later.

---

## Tab Hidden

```text
Tab is completely hidden.
```

Users cannot access it.

---

# Alternative Method: Permission Sets

Instead of creating a new profile, you can use:

```text
Permission Sets
```

to grant:

- Object Access
- Tab Access
- App Access

without changing the user's profile.

---

# Profile vs Permission Set

| Profile | Permission Set |
|-----------|-----------|
| Mandatory | Optional |
| One per User | Multiple per User |
| Base Access | Additional Access |
| Controls Login | Extends Permissions |

---

# Common Reasons Tabs Are Not Visible

## 1. Tab Hidden

```text
Tab Hidden
```

in Custom Tab Settings.

---

## 2. No Object Permission

User lacks:

```text
Read Access
```

on the object.

---

## 3. App Does Not Include Tab

The tab is not added to the Lightning App Navigation.

---

## 4. User Using Wrong Profile

The user has another profile assigned.

---

## 5. Permission Set Missing

Required access not granted.

✔ Never modify Standard Profiles directly

---

# Summary

When custom tabs are not visible for other users, the issue is usually related to Profile permissions. Create a custom profile by cloning the Standard User profile, enable the required tabs under Custom Tab Settings, grant object permissions, and assign the profile to the user. After saving and logging in as the user, the custom tabs such as Students, Teachers, Colleges, and Staff Rooms will become visible in the Lightning App navigation bar.
