# What are Public Groups in Salesforce? | How to Create and Use Public Groups?

## Introduction

In Salesforce, **Public Groups** are used to simplify record sharing and access management. Instead of sharing records with individual users one by one, administrators can create a Public Group and add users, roles, roles with subordinates, territories, or even other public groups to it.

Public Groups are commonly used in:

- Sharing Rules
- Manual Sharing
- Folder Access
- Report and Dashboard Access
- Queue Management
- Approval Processes

---

# Important Salesforce Security Components

Before understanding Public Groups, it is important to know the major security and sharing components available in Salesforce:

1. Users
2. Roles
3. Public Groups
4. Queues
5. Profiles
6. Page Layouts
7. Field-Level Security (FLS)
8. Permission Sets

Among these, **Public Groups** help simplify record sharing and access control.

---

# What is a Public Group in Salesforce?

## Salesforce Definition

As shown on the Public Groups page:

> A public group is a set of users. It can contain individual users, other groups, the users in a particular role or territory, or the users in a role or territory plus all of the users below that role or territory in the hierarchy.

---

## Simple Definition

A **Public Group** is a collection of users and other members that can be used for sharing records and granting access to multiple users at once.

Instead of sharing records with users individually, Salesforce allows administrators to share records with a Public Group.

---

# Why Do We Use Public Groups?

Public Groups help administrators:

- Simplify record sharing.
- Manage access for multiple users.
- Reduce maintenance efforts.
- Avoid creating multiple sharing rules.
- Group users with similar access requirements.

---

# Real-Life Example

Suppose a company has:

```text
Sales Team
├── Arun
├── Rahul
├── Priya
└── Amit
```

Instead of sharing records with all four users individually:

```text
Arun
Rahul
Priya
Amit
```

You can create:

```text
Public Group = Sales Team
```

and add all users to it.

Now any sharing rule can simply use:

```text
Sales Team Public Group
```

The following fields are available:

| Field | Description |
|---------|-------------|
| Label | Display name of the Public Group |
| Group Name | Unique API name generated automatically |
| Grant Access Using Hierarchies | Allows users higher in the role hierarchy to access records |
| Description | Purpose of the Public Group |

# Grant Access Using Hierarchies

This checkbox is enabled by default.

When checked:

- Users higher in the role hierarchy can gain access through sharing.

When unchecked:

- Hierarchy-based access is restricted for custom objects.

# Where Public Groups Are Used

## 1. Sharing Rules

Example:

```text
Share Account Records
To:
Public Group = Sales Team
```

This grants access to all members in the group.

---

## 2. Manual Sharing

Users can manually share records with Public Groups.

Example:

```text
Share Account
→ Public Group
```

---

## 3. Report and Dashboard Folder Access

Grant folder access to:

```text
Sales Team Public Group
```

instead of assigning users individually.

---

## 4. Queue Membership

Public Groups can be added to queues.

Example:

```text
Lead Queue
Case Queue
```

---

## 5. Approval Processes

Approvals can be routed to Public Groups.

Example:

```text
Finance Approval Group
HR Approval Group
```

---

# Public Group vs Role

| Public Group | Role |
|-------------|------|
| Used for sharing records | Used for hierarchy |
| Can contain users and roles | Represents reporting structure |
| Flexible membership | Fixed hierarchy |
| Used in sharing rules | Used for record visibility |

---

# Public Group vs Queue

| Public Group | Queue |
|-------------|--------|
| Used for sharing | Used for record ownership |
| Cannot own records | Can own records |
| Used in sharing rules | Used in assignment processes |

# Interview Questions

## What is a Public Group in Salesforce?

A Public Group is a collection of users, roles, roles and subordinates, territories, or other groups used for sharing records and granting access.

---

## Why are Public Groups used?

To simplify sharing and access management for multiple users.

---

## Can a Public Group contain Roles?

Yes.

A Public Group can contain:

- Users
- Roles
- Roles and Subordinates
- Territories
- Other Public Groups

---

## Can Public Groups be used in Sharing Rules?

Yes.

Public Groups are commonly used in Sharing Rules.

---

## Can Public Groups own records?

No.

Queues can own records, but Public Groups cannot.

---

## What is the difference between Public Group and Queue?

Public Groups are used for sharing, while Queues are used for record ownership and assignment.

---

## Can one Public Group contain another Public Group?

Yes.

Nested Public Groups are supported.

# Summary

A Public Group in Salesforce is a collection of users, roles, roles and subordinates, territories, or other groups that simplifies record sharing and access management. Instead of sharing records with individual users, administrators can create a Public Group and assign access to the entire group. Public Groups are widely used in Sharing Rules, Manual Sharing, Folder Access, Approval Processes, and Queue Management, making Salesforce administration easier, more scalable, and more secure.
