# Salesforce Permission Sets | Salesforce Permission Sets vs Profiles

## Introduction

In Salesforce, a **Profile** provides the base level of access for a user, while **Permission Sets** provide additional permissions without changing the user's profile.

Permission Sets are one of the most important security features in Salesforce because they allow administrators to grant extra access to selected users without creating multiple profiles.

This guide explains:

- What is a Permission Set
- Official Salesforce Definition
- Permission Set vs Profile
- Why Permission Sets are used
- Real-world example
- How to create a Permission Set
- How to assign App Access using Permission Sets
- How to assign a Permission Set to a User
- Interview Questions
- Best Practices

---

# Official Salesforce Definition

According to Salesforce:

> A permission set is a collection of settings and permissions that give users access to various tools and functions.

> Permission sets extend users' functional access without changing their profiles and are the recommended way to manage your users' permissions.

---

# What is a Permission Set?

## Definition

A Permission Set is a collection of settings and permissions that gives users access to additional Salesforce features, objects, apps, tabs, and functionality.

Permission Sets:

```text
Extend User Access
Do Not Replace Profiles
Provide Additional Permissions
```

---

# Understanding Profiles and Permission Sets

Every Salesforce user must have:

```text
1 User = 1 Profile
```

A user can also have:

```text
0 to Many Permission Sets
```

Example:

```text
User = Test3 User

Profile
--------
Standard Platform User

Permission Sets
--------
My App Access
Reports Access
Custom Object Access
```

---

# Why Use Permission Sets?

Suppose two users have the same profile:

| User | Profile |
|--------|---------|
| Test2 User | Standard Platform User |
| Test3 User | Standard Platform User |

Requirement:

```text
Provide access to My App 2
only for Test3 User
```

Changing the profile would affect all users using that profile.

Instead, assign a Permission Set only to Test3 User.

This is the recommended Salesforce approach.

---

# Profile vs Permission Set

| Feature | Profile | Permission Set |
|----------|----------|----------|
| Mandatory | ✅ Yes | ❌ No |
| Users Per Assignment | One Profile Per User | Multiple Permission Sets |
| Base Access | ✅ Yes | ❌ No |
| Additional Access | ❌ No | ✅ Yes |
| Object Permissions | ✅ Yes | ✅ Yes |
| App Permissions | ✅ Yes | ✅ Yes |
| Field Permissions | ✅ Yes | ✅ Yes |
| Recommended for Extra Access | ❌ No | ✅ Yes |

# Step 1: Verify User

Navigate to:

```text
Setup
→ Users
→ Users
```

Verify the target user.

Example:

```text
User = Test3 User
Profile = Standard User New
```

This is the user who will receive additional access.

---

# Step 2: Open Permission Sets

Navigate to:

```text
Setup
→ Search "Permission Sets"
→ Permission Sets
```

Salesforce displays:

```text
Permission Sets
```

page.

Click:

```text
New
```

---

# Step 3: Create Permission Set

Enter:

```text
Label
```

Example:

```text
Permission set for Arun Tech School
```

Salesforce automatically generates:

```text
API Name
```

Example:

```text
Permission_set_for_Arun_Tech_School
```

Optionally enter:

```text
Description
```

Click:

```text
Save
```

---

# Result

A new Permission Set is created.

Example:

```text
Permission set for Arun Tech School
```

---

Scroll to:

```text

```

# Step 5: Enable App

Click:


```text

```

```text

```

# Why Permission Sets Are Better

Instead of creating:

```text
Profile A
Profile B
Profile C
Profile D
```

you can create:

```text
1 Profile

+
Multiple Permission Sets
```

This greatly simplifies Salesforce security management.

# Interview Questions

## What is a Permission Set?

A Permission Set is a collection of settings and permissions that grants users additional access without changing their profile.

---

## Can a user have multiple Permission Sets?

```text
Yes
```

A user can have:

```text
0 to Many Permission Sets
```

---

## Can a user have multiple Profiles?

```text
No
```

A user can have:

```text
Only One Profile
```

---

## Why are Permission Sets preferred?

Because they allow administrators to grant extra permissions without creating multiple profiles.

---

## What can Permission Sets control?

Permission Sets can control:

```text
Object Access
Field Access
Tab Access
App Access
User Permissions
System Permissions
```

---

## Do Permission Sets remove permissions?

```text
No
```

Permission Sets only grant additional permissions.

They cannot restrict access already granted by a profile.

# Best Practices

✔ Keep Profiles minimal

✔ Use Permission Sets for additional access

✔ Follow the Principle of Least Privilege

✔ Avoid creating unnecessary profiles

✔ Use descriptive Permission Set names

✔ Assign only required permissions

✔ Test access using real users

✔ Regularly review assignments

---

# Summary

A Permission Set is a collection of settings and permissions that extends a user's access without changing their profile. Salesforce recommends using Permission Sets instead of creating multiple profiles for small access differences. A user can have only one profile but can have multiple Permission Sets. By creating a Permission Set, enabling app access, and assigning it to specific users, administrators can efficiently manage security while keeping profiles simple and maintainable.
