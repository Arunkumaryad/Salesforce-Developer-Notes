# What are Profiles in Salesforce? | How to Create Profile in Salesforce

## Introduction

In Salesforce, **Profiles** are used to control what users can access and what actions they can perform within the organization. Every Salesforce user must be assigned exactly one Profile.

Profiles define:

- Object Permissions
- Field-Level Security
- App Access
- Tab Visibility
- Record Type Access
- Administrative Permissions
- User Permissions
- Page Layout Assignments

Profiles are one of the most important components of Salesforce Security.

---

# What is a Profile in Salesforce?

## Salesforce Definition

> Profiles define how users access objects and data, and what they can do within the application. When you create users, you assign a profile to each one.

---

## Simple Definition

A **Profile** is a collection of settings and permissions that determines what a user can see and do in Salesforce.

Profiles control:

- Which objects users can access
- Which fields users can view or edit
- Which tabs users can see
- Which apps users can access
- Which administrative tasks users can perform

Every Salesforce user must have one Profile.

---

# Why Do We Use Profiles?

Profiles help organizations:

- Secure Salesforce data
- Restrict unauthorized access
- Control user permissions
- Manage application access
- Define field visibility
- Control record type access
- Improve overall security

# Creating a Profile

Salesforce does not allow creating a profile from scratch.

A Profile must always be created by cloning an existing Profile.

---

## Step 1: Open Profiles

Navigate to:

```text
Setup → Profiles
```

---

## Step 2: Click New Profile

Click:

```text
New Profile
```

Salesforce opens the Clone Profile page.

---

# Clone Profile Page

The Clone Profile page contains:

| Field | Description |
|---------|-------------|
| Existing Profile | Profile to clone |
| User License | Associated License |
| Profile Name | Name of new profile |

---

## Step 3: Select Existing Profile

Example:

```text
Force.com - Free User
```

Salesforce automatically displays:

```text
User License:
Force.com - Free
```

---

## Step 4: Enter Profile Name

Example:

```text
Test Profile
```

---

## Step 5: Click Save

Click:

```text
Save
```

Salesforce creates the new Profile.

# Profile vs Permission Set

| Profile | Permission Set |
|----------|---------------|
| Mandatory | Optional |
| One Profile per User | Multiple Permission Sets per User |
| Provides Base Access | Provides Additional Access |
| Assigned During User Creation | Assigned Anytime |

---

# Profile vs Role

| Profile | Role |
|----------|------|
| Controls Permissions | Controls Record Visibility |
| Security-Based | Hierarchy-Based |
| Defines What User Can Do | Defines What Records User Can See |
| Mandatory | Optional |
