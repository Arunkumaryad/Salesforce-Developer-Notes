# What is Role in Salesforce? | How to Manage Role Hierarchy in Salesforce?

## Introduction

In Salesforce, **Roles** are used to control **record-level access** through the Role Hierarchy. Roles determine which users can view, edit, and report on records owned by users below them in the hierarchy.

Role Hierarchy does **not grant object permissions**. It only controls access to records.

---

# Important Salesforce Security Components

Before understanding Roles, it is important to know the major security and sharing components in Salesforce:

1. Users

Among these, **Roles** are primarily responsible for managing record visibility.

---

# What is a Role in Salesforce?

## Definition

A **Role** in Salesforce is a mechanism used to control the visibility of records based on a hierarchy within an organization.

Users higher in the hierarchy can access records owned by users below them.

# Why Do We Use Roles?

Roles help organizations:

- Control record visibility.
- Enable managers to access subordinate records.
- Simplify data sharing.
- Support reporting hierarchy.
- Improve data security.

---

# Difference Between Role and Profile

| Profile | Role |
|----------|----------|
| Controls permissions | Controls record visibility |
| Determines what a user can do | Determines what records a user can see |
| Object-level security | Record-level security |
| Required for every user | Optional for users |

# Interview Questions

## What is a Role in Salesforce?

A Role controls record-level access through the Role Hierarchy.

---

## Does Role provide Object Permissions?

No.

Profiles and Permission Sets provide object permissions.

Roles only provide record access.

---

## Is Role mandatory for every user?

No.

A user can exist without a role.

---

## What is the purpose of Role Hierarchy?

To allow users higher in the hierarchy to access records owned by users below them.

---

## What is the difference between Role and Profile?

### Profile

Controls:

- Create
- Read
- Edit
- Delete
- Object Access
- Field Access

### Role

Controls:

- Record Visibility
- Record Sharing

---

## Can a CEO view subordinate records?

Yes.

Users higher in the hierarchy can access records owned by users below them.

---

## What does Assign do in Role Hierarchy?

Assign links a user to a specific role.

# Summary

A Role in Salesforce is used to control record visibility through a hierarchical structure. Users higher in the hierarchy automatically gain access to records owned by users below them. Roles help organizations manage record sharing, reporting relationships, and data visibility. Administrators can create a Role Hierarchy, add new roles, assign users to roles, and maintain an organizational structure that mirrors the company's reporting model while ensuring secure access to Salesforce records.
