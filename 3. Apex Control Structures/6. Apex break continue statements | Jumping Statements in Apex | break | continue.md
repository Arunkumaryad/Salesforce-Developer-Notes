# Apex break continue statements | Jumping Statements in Apex | break | continue

## Introduction

Jumping statements are used to control the execution flow of loops in Apex.

Apex supports two jumping statements:

1. break
2. continue

These statements are mainly used with:
- for loop
- while loop
- do while loop

---

# What are Jumping Statements?

## Definition

Jumping statements are statements that transfer control from one part of the program to another.

They help:
- Stop loop execution
- Skip specific iterations
- Improve loop control

---

# Types of Jumping Statements in Apex

1. break
2. continue

---

# 1. break Statement

## Definition

`break` statement is used to immediately terminate the loop.

When `break` executes:
- Loop stops completely
- Control moves outside the loop

---

# Syntax of break Statement

```apex
break;
```

---

# Example of break Statement

```apex
for(Integer i = 0; i < 5; i++) {

    if(i == 3)
        break;

    System.debug('value of i : ' + i);
}
```

---

# Output

```text
value of i : 0
value of i : 1
value of i : 2
```

---

# Explanation

Loop runs from:
```apex
0 to 4
```

When:
```apex
i == 3
```

`break` executes and loop stops immediately.

Therefore:
```text
0, 1, 2
```

are printed.

---

# Working of break Statement

```text
Loop Starts
 ↓
Condition Check
 ↓
break Found?
 ↓
Yes → Exit Loop
No  → Continue Execution
```

---

# 2. continue Statement

## Definition

`continue` statement skips current iteration and moves control to next iteration.

Loop does not stop completely.

---

# Syntax of continue Statement

```apex
continue;
```

---

# Example of continue Statement

```apex
for(Integer i = 0; i < 5; i++) {

    if(i == 3)
        continue;

    System.debug('value of i : ' + i);
}
```

---

# Output

```text
value of i : 0
value of i : 1
value of i : 2
value of i : 4
```

---

# Explanation

When:
```apex
i == 3
```

`continue` skips that iteration.

Therefore:
```text
3
```

is not printed.

Loop continues with next iteration.

---

# Difference Between break and continue

| break | continue |
|-------|-----------|
| Terminates loop completely | Skips current iteration |
| Control exits loop | Control moves to next iteration |
| Stops execution | Continues execution |

---

# break Statement in Nested Loops

## Example

```apex
for(Integer i = 0; i < 5; i++) {

    for(Integer j = 0; j < 5; j++) {

        if(j == 3)
            break;

        System.debug('value of i : ' + i +
                     ' value of j : ' + j);
    }
}
```

---

# Explanation

When:
```apex
j == 3
```

only inner loop stops.

Outer loop continues execution.

---

# continue Statement in Nested Loops

## Example

```apex
for(Integer i = 0; i < 5; i++) {

    for(Integer j = 0; j < 5; j++) {

        if(j == 3)
            continue;

        System.debug('value of i : ' + i +
                     ' value of j : ' + j);
    }
}
```

---

# Explanation

When:
```apex
j == 3
```

current iteration skips.

Remaining iterations continue normally.

---

# Advantages of break Statement

- Stops unnecessary execution
- Improves performance
- Saves processing time

---

# Advantages of continue Statement

- Skips unwanted iterations
- Cleaner loop logic
- Better readability

---

# Real-Time Uses in Salesforce

| Use Case | Statement Used |
|----------|----------------|
| Stop processing after match found | break |
| Skip invalid records | continue |
| Record filtering | continue |
| Loop optimization | break |

---

# Important Notes

- `break` exits loop completely
- `continue` skips only current iteration
- Mostly used inside conditions
- Works with all loop structures
