# Iterator Loop in LWC Salesforce | Render List Using Iterator Loop in HTML

## Overview

When working with **Lightning Web Components (LWC)**, displaying multiple records is a common requirement. LWC provides two template directives for rendering lists:

- **for:each**
- **iterator**

Both directives are used for looping through arrays. However, the **iterator** directive provides additional information about each item in the collection, such as whether it is the **first** or **last** element.

This makes the iterator directive useful when you need to apply special styling, display headers/footers, or perform custom logic for the first or last record.

---

# What is Iterator Loop in LWC?

As explained in the video:

> **Like for:each, it is used for looping. The advantage of the iterator loop is that it provides some special properties.**

Unlike `for:each`, the iterator directive provides three special properties:

- value
- first
- last

---

# Template Directive Used

LWC provides the following iterator directive:

```html
iterator:iteratorName
```

Example

```html
<template iterator:stu={students}>
```

Here,

- `students` → Array to iterate.
- `stu` → Iterator variable.

---

# Special Properties of Iterator

According to the video, the iterator object provides three special properties.

---

## 1. iteratorName.value

Returns the actual object of the current iteration.

To access a property, use:

```text
iteratorName.value.propertyName
```

Example

```html
{stu.value.firstName}
```

---

## 2. iteratorName.first

Returns **true** if the current record is the first item in the array.

Example

```html
<template if:true={stu.first}>
```

---

## 3. iteratorName.last

Returns **true** if the current record is the last item in the array.

Example

```html
<template if:true={stu.last}>
```

---

# Difference Between for:each and iterator

| for:each | iterator |
|-----------|----------|
| Used for rendering lists | Used for rendering lists |
| Faster and simpler | Slightly more powerful |
| Does not provide first/last information | Provides value, first, and last properties |
| Best for normal list rendering | Best when first/last record logic is required |

# HTML (Basic Iterator Example)

```html
<template>

    <table class="slds-table slds-table_cell-buffer slds-table_bordered">

        <thead>

            <tr>

                <th>First Name</th>

                <th>Last Name</th>

                <th>Age</th>

                <th>City</th>

            </tr>

        </thead>

        <tbody>

            <template iterator:stu={students}>

                <tr key={stu.value.id}>

                    <td>{stu.value.firstName}</td>

                    <td>{stu.value.lastName}</td>

                    <td>{stu.value.Age}</td>

                    <td>{stu.value.City}</td>

                </tr>

            </template>

        </tbody>

    </table>

</template>
```

Notice that every field is accessed using

```html
stu.value.propertyName
```

instead of

```html
stu.propertyName
```

because the iterator stores the actual record inside the **value** property.

# Printing Only the First Record

One of the biggest advantages of the iterator directive is the **first** property.

```html
<template iterator:stu={students}>

    <template if:true={stu.first}>

        <tr key={stu.value.id}>

            <td>{stu.value.firstName}</td>

            <td>{stu.value.lastName}</td>

            <td>{stu.value.Age}</td>

            <td>{stu.value.City}</td>

        </tr>

    </template>

</template>
```

Since `stu.first` becomes **true** only for the first object, only the first record is rendered.

Output

| First Name | Last Name | Age | City |
|------------|-----------|-----|------|
| Olivia | Carter | 28 | Seattle |

---

# Printing Only the Last Record

Similarly, use the **last** property.

```html
<template iterator:stu={students}>

    <template if:true={stu.last}>

        <tr key={stu.value.id}>

            <td>{stu.value.firstName}</td>

            <td>{stu.value.lastName}</td>

            <td>{stu.value.Age}</td>

            <td>{stu.value.City}</td>

        </tr>

    </template>

</template>
```

Output

| First Name | Last Name | Age | City |
|------------|-----------|-----|------|
| Liam | Turner | 41 | Denver |

# Understanding How Iterator Works

Suppose the array contains

```javascript
students = [

    {

        id:1,

        firstName:'Olivia'

    },

    {

        id:2,

        firstName:'Ethan'

    }

];
```

During the first iteration,

```text
stu.value
```

contains

```javascript
{

    id:1,

    firstName:'Olivia'

}
```

and

```text
stu.first
```

is

```text
true
```

while

```text
stu.last
```

is

```text
false
```

---

During the second iteration,

```text
stu.value
```

contains

```javascript
{

    id:2,

    firstName:'Ethan'

}
```

Now

```text
stu.first = false

stu.last = true
```

# Real-World Example

Suppose you want to highlight the first account returned from Apex.

```html
<template iterator:acc={accounts}>

    <template if:true={acc.first}>

        <p style="color:green;">

            First Account :

            {acc.value.Name}

        </p>

    </template>

</template>
```

---

Similarly,

```html
<template iterator:acc={accounts}>

    <template if:true={acc.last}>

        <p style="color:red;">

            Last Account :

            {acc.value.Name}

        </p>

    </template>

</template>
```

# When Should You Use Iterator?

Use **iterator** when you need to:

- Display only the first record.
- Display only the last record.
- Apply different styling to the first item.
- Apply different styling to the last item.
- Add separators before or after specific records.
- Build advanced list layouts.

If none of these are required, `for:each` is usually simpler.

---

# Advantages of Iterator

- Renders lists dynamically.
- Provides access to the current object through `value`.
- Identifies the first record.
- Identifies the last record.
- Useful for applying conditional styling.
- Makes advanced UI development easier.

# Rules

### Rule 1

Iterator works only with arrays.

Correct

```javascript
students = [];
```

Wrong

```javascript
student = {};
```

---

### Rule 2

Access properties using

```html
stu.value.firstName
```

Not

```html
stu.firstName
```

---

### Rule 3

Always provide a unique key.

```html
<tr key={stu.value.id}>
```

---

### Rule 4

Use

```html
stu.first
```

only when checking the first record.

---

### Rule 5

Use

```html
stu.last
```

only when checking the last record.

# Common Mistakes

## Forgetting `.value`

Wrong

```html
{stu.firstName}
```

Correct

```html
{stu.value.firstName}
```

---

## Forgetting the key

Wrong

```html
<tr>
```

Correct

```html
<tr key={stu.value.id}>
```

---

## Using Iterator for an Object

Wrong

```javascript
student = {};
```

Correct

```javascript
students = [];
```

# Iterator vs for:each

| Feature | for:each | iterator |
|----------|----------|----------|
| Render List | ✅ | ✅ |
| Access Current Record | item.property | iterator.value.property |
| Detect First Record | ❌ | ✅ |
| Detect Last Record | ❌ | ✅ |
| Simpler Syntax | ✅ | ❌ |
| Advanced Rendering | ❌ | ✅ |

# Interview Questions

### What is the iterator directive in LWC?

The iterator directive is used to iterate over an array and provides additional properties like **value**, **first**, and **last**.

---

### What is the syntax of the iterator directive?

```html
<template iterator:item={array}>
```

---

### What does `iterator.value` return?

It returns the current object being processed in the loop.

---

### What does `iterator.first` return?

It returns **true** if the current record is the first item in the array.

---

### What does `iterator.last` return?

It returns **true** if the current record is the last item in the array.

---

### What is the main difference between `for:each` and `iterator`?

`iterator` provides additional properties (`value`, `first`, and `last`) while `for:each` only renders the list without those helper properties.

---

### When should you use iterator instead of for:each?

Use **iterator** whenever you need to identify or style the first or last record, or implement advanced list-rendering logic.
