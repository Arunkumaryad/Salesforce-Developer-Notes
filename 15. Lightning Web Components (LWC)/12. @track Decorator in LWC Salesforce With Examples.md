# @track Decorator in LWC Salesforce With Examples

## Overview

The **@track** decorator is used to make a property **private** and **reactive** in Lightning Web Components (LWC). Whenever the value of a tracked property changes, the component automatically rerenders and updates the UI.

In this tutorial, you'll learn:

- What is the `@track` decorator?
- Why it is used?
- Syntax of `@track`
- How reactivity works
- Practical counter example
- Best practices
- Interview questions

---

# What is @track Decorator?

According to the video:

> **It is used to make variable private but reactive. Tracked properties are also called private reactive properties.**

In simple words,

The `@track` decorator tells LWC to observe changes in a property. Whenever the property's value changes, the HTML template automatically updates without refreshing the page.

Unlike `@api`, tracked properties are **private** and cannot be accessed from outside the component.

---

# Important Note

According to the video:

> **We can't access @track properties from outside as they are private and only accessible within its component only.**

This means:

- Parent components **cannot** access `@track` properties.
- Child components **cannot** access `@track` properties.
- They are only available inside the component where they are declared.

---

# Importing @track

According to the video:

> **To use @track we have to import it first from lwc.**

Example:

```javascript
import { LightningElement, track } from 'lwc';
```

---

# Syntax

```javascript
@track propertyName = value;
```

Example:

```javascript
@track count = 0;
```

---

# Why Do We Use @track?

We use the `@track` decorator to:

- Make variables reactive.
- Automatically update the UI.
- Store component-specific data.
- Handle counters.
- Manage forms.
- Work with objects and arrays that need reactivity.

# Example: Counter Using @track

In this example, we'll create a simple counter with **Increase (+)** and **Decrease (-)** buttons.

---

## HTML File

### trackCounterExample.html

```html
<template>

    <lightning-card>

        <div>

            Counter : {count}

        </div>

        <div>

            <lightning-button

                label="+"

                onclick={increaseCount}>

            </lightning-button>

        </div>

        <div>

            <lightning-button

                label="-"

                onclick={decrementCount}>

            </lightning-button>

        </div>

    </lightning-card>

</template>
```

---

## JavaScript File

### trackCounterExample.js

```javascript
import { LightningElement, track } from 'lwc';

export default class TrackCounterExample extends LightningElement {

    @track count = 0;

    increaseCount() {

        this.count++;

    }

    decrementCount() {

        if (this.count > 0) {

            this.count--;

        }

    }

}
```

---

## Meta XML File

### trackCounterExample.js-meta.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>

<LightningComponentBundle xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>true</isExposed>

    <targets>

        <target>lightning__AppPage</target>

        <target>lightning__HomePage</target>

        <target>lightning__RecordPage</target>

    </targets>

</LightningComponentBundle>
```

# Output

Initially

```

[ + ]

[ - ]
```

After clicking **+**

```
Counter : 1
```

Click again

```
Counter : 2
```

Click **-**

```
Counter : 1
```

The counter updates immediately because the `count` property is tracked.

# How This Example Works

### Step 1

Create a tracked property.

```javascript
@track count = 0;
```

This initializes the counter.

---

### Step 2

Display it in HTML.

```html
Counter : {count}
```

LWC automatically binds the property to the template.

---

### Step 3

Increase the value.

```javascript
this.count++;
```

Every click increments the counter.

---

### Step 4

Decrease the value.

```javascript
if(this.count > 0){

    this.count--;

}
```

The condition prevents the counter from going below zero.

---

### Step 5

LWC rerenders automatically.

Because `count` is reactive, the HTML updates without manually refreshing the page.

# @track with Objects

Example:

```javascript
@track employee = {

    firstName : 'Arun',

    city : 'Noida'

};
```

Updating a property

```javascript
this.employee.city = 'Delhi';
```

The UI automatically reflects the updated city.

---

# @track with Arrays

Example:

```javascript
@track fruits = [

    'Apple',

    'Mango',

    'Orange'

];
```

Adding an item

```javascript
this.fruits.push('Banana');
```

The updated list is displayed automatically.

# Difference Between Normal Variable and @track

### Normal Variable

```javascript
count = 0;
```

- Private
- Reactive for primitive assignments in modern LWC
- Suitable for simple primitive values

---

### @track Variable

```javascript
@track count = 0;
```

- Private
- Explicitly reactive
- Commonly used when tracking changes to object or array properties

---

# @track vs @api

| @track | @api |
|---------|------|
| Private property | Public property |
| Accessible only within the component | Accessible from parent components |
| Used for internal state | Used for Parent-to-Child communication |
| Reactive | Reactive |
| Cannot be accessed outside | Can be accessed outside |

- Simplifies state management.
- Improves code readability.
- Useful for tracking changes in complex data like objects and arrays.

# Common Mistakes

## Forgetting to Import @track

Wrong

```javascript
@track count = 0;
```

Correct

```javascript
import { LightningElement, track } from 'lwc';
```

---

## Trying to Access @track Outside the Component

Wrong

```javascript
<c-child

    count={count}>

</c-child>
```

A tracked property is private and should not be treated as a public API.

Use `@api` when you need to expose a property to another component.

---

## Forgetting to Update the Property

Wrong

```javascript
count++;
```

Correct

```javascript
this.count++;
```

Always use `this` to access component properties.

---

# Best Practices

- Use `@track` for private component state, especially when working with objects or arrays whose internal properties change.
- Use meaningful property names.
- Keep tracked properties private.
- Use `@api` instead of `@track` when data needs to come from a parent component.
- Avoid using `@track` unnecessarily for simple cases, as modern LWC automatically tracks many primitive value changes.

# Interview Questions

### What is the @track decorator?

The `@track` decorator is used to make a variable private and reactive so that changes automatically update the component's UI.

---

### Can we access @track properties outside the component?

No.

Tracked properties are private and can only be accessed inside the component.

---

### Why do we import @track?

Because it is a decorator provided by the `lwc` module.

```javascript
import { LightningElement, track } from 'lwc';
```

---

### What happens when a tracked property's value changes?

The component rerenders automatically, and the updated value is displayed in the UI.

---

### Is @track used for Parent-to-Child communication?

No.

For Parent-to-Child communication, use the `@api` decorator.

---

### Can @track be used with objects and arrays?

Yes.

It is especially useful when you want the UI to react to changes inside objects or arrays.
