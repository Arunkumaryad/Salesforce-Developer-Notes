# @wire Decorator in LWC Salesforce With Example

## Overview

The **@wire** decorator is used to read Salesforce data in Lightning Web Components (LWC). It connects your component to Salesforce data using the **Wire Service**, which automatically provisions data whenever it becomes available. When the data changes, the component automatically rerenders.

In this tutorial, you'll learn:

- What is the `@wire` decorator?
- Why it is used?
- Syntax of `@wire`
- How Wire Service works
- Reading Salesforce records using `getRecord`
- Displaying Account Name and Phone
- Best practices
- Interview questions

---

# What is @wire Decorator?

According to the video:

> **To read Salesforce data, Lightning Web Components use a reactive wire service. When the wire service provisions data, the component rerenders. Components use @wire in their JavaScript class to specify a wire adapter or an Apex method.**

In simple words,

The **@wire** decorator is used to retrieve Salesforce data without writing imperative JavaScript code. It automatically fetches data from Salesforce and refreshes the component whenever the underlying data changes.

Unlike imperative Apex calls, the Wire Service manages data fetching, caching, and refreshing automatically.

---

# Why Do We Use @wire?

We use the `@wire` decorator to:

- Read Salesforce records.
- Retrieve object metadata.
- Fetch picklist values.
- Call Apex methods.
- Automatically refresh UI when data changes.
- Reduce manual server calls.

---

# Syntax

According to the video:

```javascript
import { adapterId } from 'adapterModule';

@wire(adapterId, adapterConfig) propertyOrFunction;
```

### Explanation

### adapterId (Identifier)

The identifier of the wire adapter.

Example:

```javascript
getRecord
```

---

### adapterModule

The module that contains the wire adapter.

Example:

```javascript
lightning/uiRecordApi
```

---

### adapterConfig

Configuration required by the adapter.

Example:

```javascript
{
    recordId: '$recordId',
    fields: ['Account.Name', 'Account.Phone']
}
```

---

### propertyOrFunction

The property or function that receives the data returned by the wire service.

Example:

```javascript
accounts;
```

or

```javascript
wiredAccount(result) {

}
```

---

# Wire Service

The Wire Service is a reactive data service provided by Salesforce.

It automatically:

- Calls the server.
- Retrieves the requested data.
- Caches the response.
- Updates the component when the data changes.

Because of this, developers don't need to manually refresh the UI.

# Example: Fetch Account Name and Phone

In this example, we'll retrieve an Account record using **getRecord** and display:

- Account Name
- Account Phone

---

## HTML File

### wireExample.html

```html
<template>

    <lightning-card title="Wired example">

        <div>

            Name : {getName}

        </div>

        <div>

            Phone : {getPhone}

        </div>

    </lightning-card>

</template>
```

---

## JavaScript File

### wireExample.js

```javascript
import { getFieldValue, getRecord } from 'lightning/uiRecordApi';
import { LightningElement, api, wire } from 'lwc';

export default class WireExample extends LightningElement {

    @api recordId;

    @wire(getRecord, {

        recordId: '$recordId',

        fields: [

            'Account.Name',

            'Account.Phone'

        ]

    })

    accounts;

    get getName() {

        if (this.accounts.data) {

            return getFieldValue(

                this.accounts.data,

                'Account.Name'

            );

        }

        else if (this.accounts.error) {

            return this.accounts.error;

        }

    }

    get getPhone() {

        if (this.accounts.data) {

            return getFieldValue(

                this.accounts.data,

                'Account.Phone'

            );

        }

        else if (this.accounts.error) {

            return this.accounts.error;

        }

    }

}
```

# How This Example Works

### Step 1

Import the required modules.

```javascript
import { getRecord, getFieldValue }

from 'lightning/uiRecordApi';

import {

    LightningElement,

    api,

    wire

} from 'lwc';
```

Here,

- `getRecord` fetches the record.
- `getFieldValue` extracts a specific field value.
- `@wire` connects the component to Salesforce data.

---

### Step 2

Receive the current record Id.

```javascript
@api recordId;
```

When placed on a Record Page, Salesforce automatically provides the current record's Id.

---

### Step 3

Fetch the Account record.

```javascript
@wire(getRecord, {

    recordId: '$recordId',

    fields: [

        'Account.Name',

        'Account.Phone'

    ]

})

accounts;
```

This fetches:

- Account Name
- Account Phone

and stores the response in the `accounts` property.

---

### Step 4

Read the Name field.

```javascript
get getName() {

    if(this.accounts.data){

        return getFieldValue(

            this.accounts.data,

            'Account.Name'

        );

    }

}
```

`getFieldValue()` extracts the value from the retrieved record.

---

### Step 5

Read the Phone field.

```javascript
get getPhone() {

    if(this.accounts.data){

        return getFieldValue(

            this.accounts.data,

            'Account.Phone'

        );

    }

}
```

---

### Step 6

Display values in HTML.

```html
Name : {getName}

Phone : {getPhone}
```

Whenever Salesforce data changes, the values are automatically refreshed.

# Understanding getFieldValue()

Instead of writing

```javascript
this.accounts.data.fields.Name.value
```

Salesforce recommends

```javascript
getFieldValue(
    this.accounts.data,
    'Account.Name'
);
```

Benefits:

- Cleaner code
- Safer
- Easier to read
- Salesforce recommended approach

# Property Wire vs Function Wire

## Property Wire

```javascript
@wire(getRecord, {

    recordId: '$recordId',

    fields: ['Account.Name']

})

account;
```

The data is stored directly in the property.

---

## Function Wire

```javascript
@wire(getRecord, {

    recordId: '$recordId',

    fields: ['Account.Name']

})

wiredAccount({ data, error }) {

    if(data){

    }

    if(error){

    }

}
```

Use this approach when you want additional processing before storing or displaying the data.

# Common Mistakes

## Forgetting to Import @wire

Wrong

```javascript
@wire(getRecord)
```

Correct

```javascript
import { wire } from 'lwc';
```

---

## Forgetting the '$' Before recordId

Wrong

```javascript
recordId: recordId
```

Correct

```javascript
recordId: '$recordId'
```

The `$` makes the configuration reactive. If `recordId` changes, the wire service automatically fetches the new record.

---

## Accessing Data Without Checking

Wrong

```javascript
return this.accounts.data.fields.Name.value;
```

Correct

```javascript
if(this.accounts.data){

    return getFieldValue(

        this.accounts.data,

        'Account.Name'

    );

}
```

Always check whether data exists before accessing it.

---

## Ignoring Errors

Always handle both success and error cases.

```javascript
if(this.accounts.data){

}

else if(this.accounts.error){

}
```

---

# Best Practices

- Use `@wire` when you need reactive Salesforce data.
- Prefer `getFieldValue()` for accessing field values.
- Always check for both `data` and `error`.
- Use `@wire` with UI API adapters whenever possible instead of writing Apex.
- Use imperative Apex only when user interaction or complex server-side logic requires it.

# Interview Questions

### What is the @wire decorator?

The `@wire` decorator is used to connect an LWC component to Salesforce data using the reactive Wire Service.

---

### What is the Wire Service?

The Wire Service is a reactive service that provisions Salesforce data to Lightning Web Components and automatically refreshes the component when the data changes.

---

### Can @wire call Apex methods?

Yes.

The `@wire` decorator can be used with:

- UI API adapters
- Apex methods annotated with `@AuraEnabled(cacheable=true)`

---

### Why do we use getFieldValue()?

`getFieldValue()` safely extracts field values from records returned by the UI API.

---

### What is the purpose of '$recordId'?

The `$` makes the parameter reactive. If the `recordId` changes, the Wire Service automatically fetches the updated record.

---

### Does @wire automatically rerender the component?

Yes.

Whenever the wire service provisions new data, the component rerenders automatically.
