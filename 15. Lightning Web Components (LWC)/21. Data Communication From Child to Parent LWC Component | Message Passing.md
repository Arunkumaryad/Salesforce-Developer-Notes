# Data Communication From Child to Parent LWC Component | Message Passing

## Overview

In Lightning Web Components (LWC), **Child to Parent Communication** is used when a Child Component needs to send data or notify its Parent Component about an action. Unlike Parent to Child communication, where data is passed using the **@api** decorator, Child to Parent communication is achieved using **Custom Events**.

Whenever an event occurs in the Child Component (such as a button click or form submission), the Child dispatches a **Custom Event**, and the Parent Component listens for that event and processes the received data.

---

# Definition

**Child to Parent Communication** is the process of sending data from a Child Component to its Parent Component.

According to the screenshot:

> **It is done by Custom Event.**

A Child Component creates and dispatches a **Custom Event**, while the Parent Component listens for that event and retrieves the data sent by the Child.

---

# Communication Flow

```
Parent Component
        ▲
        │
 Custom Event
        │
Child Component
```

The Child sends data, and the Parent receives it.

---

# How Child to Parent Communication Works

The communication happens in four simple steps:

1. User performs an action in the Child Component.
2. Child Component creates a **Custom Event**.
3. Child Component dispatches the event.
4. Parent Component listens for the event and receives the data.

---

# What is a Custom Event?

A **Custom Event** is a user-defined event that allows a Child Component to send information to its Parent Component.

Syntax:

```javascript
const eventName = new CustomEvent('eventname', {
    detail: value
});

this.dispatchEvent(eventName);
```

Where:

- **eventname** → Name of the event.
- **detail** → Data sent to the Parent Component.
- **dispatchEvent()** → Sends the event.

# Step 1: Child Component

The Child Component accepts user input and dispatches a Custom Event.

## childComponent1.html

```html
<template>

    <lightning-card>

        <lightning-input
            label="Enter Name"
            onchange={handleChange}>
        </lightning-input>

        <lightning-button
            label="Click Me!"
            onclick={handleClick}>
        </lightning-button>

    </lightning-card>

</template>
```

### Explanation

The component contains:

- A **lightning-input** for entering the user's name.
- A **button** that dispatches the event when clicked.

---

## childComponent1.js

```javascript
import { LightningElement } from 'lwc';

export default class ChildComponent1 extends LightningElement {

    name = '';

    handleChange(event) {
        this.name = event.target.value;
    }

    handleClick() {

        console.log('Button Clicked');

        const newEvent = new CustomEvent('getname', {
            detail: this.name
        });

        this.dispatchEvent(newEvent);

    }

}
```

---

### Explanation

### Step 1

Initially,

```javascript
name = '';
```

stores the entered name.

---

### Step 2

Whenever the user types,

```javascript
handleChange()
```

updates

```javascript
this.name
```

using

```javascript
event.target.value
```

---

### Step 3

When the button is clicked,

```javascript
handleClick()
```

is executed.

---

### Step 4

A Custom Event is created.

```javascript
const newEvent = new CustomEvent('getname', {
    detail: this.name
});
```

Here,

Event Name

```
getname
```

Data Sent

```
this.name
```

---

### Step 5

Finally,

```javascript
this.dispatchEvent(newEvent);
```

dispatches the event to the Parent Component.

# Step 2: Parent Component

The Parent Component listens for the event dispatched by the Child Component.

---

```html
    </lightning-card>

    <lightning-card>

```

---

### Explanation

The Parent includes the Child Component.

```html

```

To listen for the event,

```html
ongetname={handleName}
```

is used.

Notice that

Child Event Name

```
getname
```

becomes

```
ongetname
```

inside the Parent HTML.

Whenever

```

```

is fired,

```

```

runs automatically.

## parentComponent1.js

```javascript
import { LightningElement } from 'lwc';

export default class ParentComponent1 extends LightningElement {

    name = '';

    handleName(event) {

        this.name = event.detail;

    }

}
```

---

### Explanation

The Parent receives the event object.

The data sent by the Child is stored inside

```javascript
event.detail
```

Therefore,

```javascript
this.name = event.detail;
```

updates the Parent Component.

The updated value is automatically displayed.

# Advantages

- Easy to implement.
- Official Salesforce communication pattern.
- Keeps components loosely coupled.
- Makes Child Components reusable.
- Supports one-way upward communication.
- Parent controls all business logic.

---

# Best Practices

- Use meaningful event names.
- Pass only the required data in `detail`.
- Keep Child Components independent.
- Use one event for one purpose.
- Let the Parent handle business logic.
- Avoid sending unnecessary objects if a simple value is enough.

# Common Mistakes

## 1. Forgetting dispatchEvent()

❌ Incorrect

```javascript
const event = new CustomEvent('getname');
```

The event is created but never dispatched.

---

✅ Correct

```javascript
this.dispatchEvent(event);
```

---

## 2. Using event.target.value in Parent

❌ Incorrect

```javascript
this.name = event.target.value;
```

The Parent should read the data from

```javascript
event.detail
```

---

✅ Correct

```javascript
this.name = event.detail;
```

---

## 3. Event Name Doesn't Match

Child

```javascript
new CustomEvent('getname');
```

Parent

```html
<c-child-component1
    onsavename={handleName}>
</c-child-component1>
```

This will not work because the event names are different.

---

✅ Correct

Child

```javascript
new CustomEvent('getname');
```

Parent

```html
<c-child-component1
    ongetname={handleName}>
</c-child-component1>
```

---

## 4. Using @api Instead of Custom Events

❌ Incorrect

Trying to send data from Child to Parent using

```javascript
@api
```

---

✅ Correct

Use

```javascript
CustomEvent
```

---

# Parent to Child vs Child to Parent

| Parent to Child | Child to Parent |
|-----------------|-----------------|
| Uses `@api` | Uses `CustomEvent` |
| Parent sends data | Child sends data |
| One-way downward | One-way upward |
| Parent controls Child | Child notifies Parent |

# Interview Questions

### What is Child to Parent Communication in LWC?

It is the process of sending data from a Child Component to its Parent Component using **Custom Events**.

---

### Which mechanism is used for Child to Parent communication?

**Custom Events**.

---

### What is dispatchEvent()?

`dispatchEvent()` sends a Custom Event from the Child Component so that the Parent Component can receive it.

---

### What is event.detail?

`event.detail` contains the data sent by the Child Component through the Custom Event.

---

### Why do we use Custom Events?

Custom Events allow Child Components to notify their Parent Components about user actions without creating tight coupling between the components.

---

### Can the Parent receive multiple values?

Yes. Multiple values can be sent by passing an object inside the `detail` property.

Example:

```javascript
new CustomEvent('save', {
    detail: {
        name: this.name,
        email: this.email,
        phone: this.phone
    }
});
```
