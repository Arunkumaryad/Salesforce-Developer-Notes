# Data Binding in LWC Salesforce | Lightning Web Components

## Overview

Data Binding is one of the most important concepts in Lightning Web Components (LWC). It allows you to display data from JavaScript directly inside the HTML template.

Instead of writing static values in HTML, you define properties in the JavaScript file and bind those properties to the HTML using **template expressions** (`{propertyName}`).

Whenever the property value changes, the UI is automatically updated.

---

# What is Data Binding?

**Definition (from the video):**

> **Data Binding is the process by which values are bounded to the properties.**

In simple words,

Data Binding is the process of connecting the values stored in the JavaScript class with the HTML template so that they can be displayed dynamically.

Instead of hardcoding text in HTML, we bind JavaScript properties to HTML elements.

---

# Why Do We Need Data Binding?

Without Data Binding:

```html
<h1>Hello Arun Kumar</h1>
```

The value is fixed (static).

If you want to change the name, you must edit the HTML.

---

With Data Binding:

```html
<h1>Hello {firstName} {lastName}</h1>
```

The values come from JavaScript.

If the values change in JavaScript, the HTML updates automatically.

---

# Syntax of Data Binding

The syntax is very simple.

```html
{propertyName}
```

Example:

```html
<h1>{firstName}</h1>
```

Here,

`firstName` is a property defined in the JavaScript file.

---

# Data Binding with Primitive Properties

Primitive values include:

- String
- Number
- Boolean

## JavaScript (helloWorld.js)

```javascript
import { LightningElement } from 'lwc';

export default class HelloWorld extends LightningElement {

    firstName = 'Arun';

    lastName = 'Kumar';

}
```

---

## HTML (helloWorld.html)

```html
<template>

    <lightning-card>

        <h1>Hello {firstName} {lastName}</h1>

    </lightning-card>

</template>
```

---

## Output

```text
Hello Arun Kumar
```

---

# How It Works

Step 1

JavaScript creates two properties.

```javascript
firstName = 'Arun';

lastName = 'Kumar';
```

↓

Step 2

HTML accesses those properties.

```html
{firstName}

{lastName}
```

↓

Step 3

Salesforce replaces them with their values.

Output

```text
Hello Arun Kumar
```

---

# Data Binding with Object Properties

Data Binding also works with JavaScript objects.

Instead of creating multiple variables, we can store everything inside one object.

---

## JavaScript

```javascript
import { LightningElement } from 'lwc';

export default class HelloWorld extends LightningElement {

    student = {

        firstName: 'Arun',

        lastName: 'Kumar',

        Age: 24,

        City: 'Noida'

    };

}
```

---

## HTML

```html
<template>

    <lightning-card>

        <h1>Hello {student.firstName} {student.lastName}</h1>

        <h2>Age : {student.Age}</h2>

        <h2>City : {student.City}</h2>

    </lightning-card>

</template>
```

---

## Output

```text
Hello Arun Kumar

Age : 24

City : Noida
```

---

# Accessing Object Properties

Suppose the object is

```javascript
student = {

    firstName: 'Arun',

    lastName: 'Kumar',

    Age: 24,

    City: 'Noida'

};
```

You can access the values like this:

| JavaScript Property | HTML Binding |
|---------------------|--------------|
| firstName | `{student.firstName}` |
| lastName | `{student.lastName}` |
| Age | `{student.Age}` |
| City | `{student.City}` |

---

```

        ▼
```

# Rules for Data Binding

### Rule 1

Always define the property in JavaScript before using it in HTML.

Correct

```javascript
name = 'Arun';
```

```html
{name}
```

---

Wrong

```html
{name}
```

(No property exists in JavaScript.)

---

### Rule 2

Use curly braces.

Correct

```html
{name}
```

Wrong

```html
name
```

---

### Rule 3

Property names are case-sensitive.

Correct

```html
{firstName}
```

Wrong

```html
{firstname}
```

These are treated as different names.

---

### Rule 4

Use dot (`.`) notation for object properties.

Correct

```html
{student.City}
```

Wrong

```html
{student->City}
```

---

# Advantages of Data Binding

- Displays dynamic data in the UI.
- Keeps HTML clean and readable.
- Separates UI from business logic.
- Automatically reflects updated property values.
- Reduces duplicate code.
- Makes components easier to maintain.

# Common Mistakes

## 1. Forgetting Curly Braces

Wrong

```html
<h1>firstName</h1>
```

Correct

```html
<h1>{firstName}</h1>
```

---

## 2. Property Doesn't Exist

JavaScript

```javascript
name = 'Arun';
```

HTML

```html
{firstName}
```

Wrong because `firstName` does not exist.

---

## 3. Wrong Object Property

JavaScript

```javascript
student = {

    City:'Noida'

}
```

Wrong

```html
{student.city}
```

Correct

```html
{student.City}
```
