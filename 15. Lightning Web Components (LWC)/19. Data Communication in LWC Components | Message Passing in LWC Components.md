# Data Communication in LWC Components | Message Passing in LWC Components

## Overview

In Lightning Web Components (LWC), multiple components often need to exchange data with one another. This process is called **Component Communication** or **Message Passing**.

For example:

- A Parent component sends data to a Child component.
- A Child component sends data back to its Parent.
- Two completely unrelated components communicate with each other.

Salesforce provides different mechanisms for each communication scenario.

---

# Definition

**Data Communication in LWC** is the process of passing data from one Lightning Web Component to another. Depending on the relationship between the components, Salesforce provides different communication techniques.

According to Salesforce:

> **There are three types of communication between LWC Components:**
>
> 1. Parent to Child Communication
> 2. Child to Parent Communication
> 3. Communication between unrelated components

---

# Types of Communication in LWC

There are **three types** of communication between LWC Components:

1. Parent to Child Communication
2. Child to Parent Communication
3. Communication between Unrelated Components

Each method is used in different situations.

---

# 1. Parent to Child Communication

## Definition

Parent to Child Communication allows a **Parent Component** to pass data to its **Child Component**.

According to the screenshot:

> **It is done by public properties using @api decorator.**

This is the most common communication technique in LWC.

---

## Flow

```
Parent Component
        │
        │  @api Property
        ▼
Child Component
```

---

## How does it work?

The Parent Component passes values to the Child Component using public properties decorated with **@api**.

Example:

Parent passes:

```
Student Name
```

Child receives:

```
Student Name
```

---

## When to use?

Use Parent to Child communication when:

- Parent sends user information
- Parent sends Account details
- Parent passes Record Id
- Parent controls Child behavior
- Parent shares configuration settings

---

## Advantages

- Simple
- Fast
- Easy to understand
- Official Salesforce approach

---

## Real-world Example

Imagine an Account page.

```
Parent Component

Account Details
```

passes

```
Account Name
Phone
Industry
```

to

```
Child Component

Account Summary
```

The Child only displays the received information.

# 2. Child to Parent Communication

## Definition

Sometimes a Child Component needs to send data back to its Parent.

According to the screenshot:

> **It is done by Custom Event.**

In LWC, this is achieved using **Custom Events**.

---

## Flow

```
Parent Component

        ▲
        │
 Custom Event
        │

Child Component
```

---

## How does it work?

The Child Component creates and dispatches a **CustomEvent**.

The Parent Component listens for that event and processes the received data.

---

## Example

Child Component

```
User clicks Save
```

↓

Child fires

```
save
```

↓

Parent receives

```
save event
```

↓

Parent performs

```
Save Record
```

---

## When to use?

Use Child to Parent communication when:

- Save button clicked
- Delete button clicked
- Form submitted
- Record selected
- Checkbox changed
- User action completed

---

## Advantages

- Loose coupling
- Reusable Child Components
- Parent controls business logic
- Standard Salesforce approach

---

## Real-world Example

```
Parent

Customer List
```

↓

Child

```
Customer Card
```

↓

User clicks

```
Delete
```

↓

Child fires

```
delete
```

↓

Parent deletes the customer.

# 3. Communication Between Unrelated Components

## Definition

Sometimes two components have **no Parent-Child relationship** but still need to exchange information.

According to the screenshot:

> **It is done by Lightning Message Service (LMS).**

Salesforce provides **Lightning Message Service (LMS)** for this purpose.

---

## Flow

```
Component A

        │
        │
Lightning Message Service
        │
        ▼

Component B
```

---

## What is Lightning Message Service (LMS)?

Lightning Message Service (LMS) is a Salesforce messaging framework that allows components with **no direct relationship** to communicate by publishing and subscribing to messages through a shared **Lightning Message Channel**.

---

## How does it work?

Component A

```
Publishes Message
```

↓

Lightning Message Channel

↓

Component B

```
Receives Message
```

---

## When to use?

Use LMS when:

- Components are unrelated
- Components are on different regions of a page
- Components don't share Parent-Child hierarchy
- Aura and LWC communicate
- Visualforce and LWC communicate
- Multiple independent components need the same data

---

## Advantages

- Decoupled communication
- Highly reusable
- Works across technologies
- Supports cross-DOM communication
- Official Salesforce messaging framework

---

## Real-world Example

Suppose a dashboard contains:

```
Component A

Customer List
```

and

```
Component B

Customer Details
```

Both are independent.

When a customer is selected in Component A,

it publishes

```
Customer Id
```

through LMS.

Component B subscribes to the message and automatically loads the selected customer's details.

# Communication Comparison

| Communication Type | Relationship | Technique Used |
|--------------------|--------------|----------------|
| Parent → Child | Parent and Child | `@api` Decorator |
| Child → Parent | Parent and Child | Custom Events |
| Unrelated Components | No relationship | Lightning Message Service (LMS) |

---

# Which Communication Should You Use?

| Situation | Recommended Method |
|-----------|--------------------|
| Parent sends data to Child | `@api` |
| Child notifies Parent | Custom Event |
| Two independent components communicate | Lightning Message Service (LMS) |

# Best Practices

- Use **@api** only for Parent to Child communication.
- Use **Custom Events** only for Child to Parent communication.
- Use **Lightning Message Service** only when components are unrelated.
- Avoid unnecessary component coupling.
- Keep components reusable and independent.
- Pass only the required data instead of entire objects whenever possible.

# Interview Questions

### What are the different types of communication in LWC?

There are three types:

- Parent to Child
- Child to Parent
- Communication between Unrelated Components

---

### Which decorator is used for Parent to Child communication?

The **@api** decorator.

---

### How does a Child Component communicate with its Parent?

By using **Custom Events**.

---

### What is Lightning Message Service?

Lightning Message Service (LMS) is a Salesforce messaging framework used to communicate between components that do not have a Parent-Child relationship.

---

### When should LMS be used?

When two components are unrelated but need to exchange information.

---

### Can Aura and LWC communicate using LMS?

Yes. Lightning Message Service supports communication between Aura Components, Lightning Web Components, and Visualforce pages.
