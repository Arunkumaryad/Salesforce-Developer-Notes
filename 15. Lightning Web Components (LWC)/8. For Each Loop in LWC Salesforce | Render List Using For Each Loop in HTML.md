# For Each Loop in LWC Salesforce | Render List Using For Each Loop in HTML

## Overview

When working with **Lightning Web Components (LWC)**, you often need to display multiple records such as Accounts, Contacts, Opportunities, or custom objects. Instead of writing the same HTML repeatedly, LWC provides **template directives** that allow you to iterate over an array and render the UI dynamically.

The **for:each** directive is used to loop through a collection (array), while **for:item** gives a name to the current item being processed in each iteration.

This makes your components cleaner, reusable, and easier to maintain.

---

# What is for:each in LWC?

**for:each** is a template directive used to iterate over an array and render HTML for every item in the collection.

Instead of manually creating HTML for each record, LWC automatically repeats the template for every element in the array.

---

# Template Directives Used

According to the video, LWC provides the following template directives for rendering lists:

1. **for:each**
2. **for:item**

---

# for:each

The **for:each** directive specifies the array that should be iterated.

### Syntax

```html
<template for:each={arrayName} for:item="item">

    <!-- HTML -->

</template>
```

Example

```html
<template for:each={students} for:item="stu">

</template>
```

Here,

- `students` → Array to iterate.
- `stu` → Represents the current object in every iteration.

---

# for:item

The **for:item** directive specifies the variable name for the current item in the loop.

Example

```html
<template for:each={students} for:item="stu">
```

Inside the loop,

```html
{stu.firstName}
```

means

> Display the **firstName** property of the current student object.

---

# Understanding the Difference

| Directive | Purpose |
|-----------|---------|
| for:each | Specifies the array to iterate |
| for:item | Represents the current object in each iteration |

# Rendering a Single Object

Before using a loop, suppose you have only one employee object.

## JavaScript

```javascript
import { LightningElement } from 'lwc';

export default class ForEachLoop extends LightningElement {

    employee = {

        firstName: 'Arun',

        lastName: 'Kumar',

        Age: 25,

        City: 'Chhattarpur'

    };

}
```

---

## HTML

```html
<template>

    <lightning-card>

        <h1 style="color:red;">
            {employee.firstName}
        </h1>

        <h1 style="color:green;">
            {employee.lastName}
        </h1>

        <h1 style="color:yellow;">
            {employee.Age}
        </h1>

        <h1 style="color:brown;">
            {employee.City}
        </h1>

    </lightning-card>

</template>
```

Output

```
Arun
Kumar
25
Chhattarpur
```

Since there is only one object, a loop is not required.

# Rendering Multiple Objects

When multiple records exist, use an array.

## JavaScript

```javascript
import { LightningElement } from 'lwc';

export default class ForEachLoop extends LightningElement {

    students = [

        {

            id: 1,

            firstName: 'Arun',

            lastName: 'Kumar',

            Age: 25,

            City: 'Chhattarpur'

        },

        {

            id: 2,

            firstName: 'Rohan',

            lastName: 'Raj',

            Age: 23,

            City: 'Ranchi'

        },

        {

            id: 3,

            firstName: 'Suddu',

            lastName: 'John',

            Age: 19,

            City: 'Kolkata'

        }

    ];

}
```

> **Note:** Each object should have a unique `id` because LWC requires a unique key while rendering lists.

# HTML Using for:each

```html
<template>

    <table class="slds-table slds-table_cell-buffer slds-table_bordered">

        <thead>

            <tr>

                <th>First Name</th>

                <th>Last Name</th>

                <th>Age</th>

                <th>City</th>

            </tr>

        </thead>

        <tbody>

            <template for:each={students} for:item="stu">

                <tr key={stu.id}>

                    <td>{stu.firstName}</td>

                    <td>{stu.lastName}</td>

                    <td>{stu.Age}</td>

                    <td>{stu.City}</td>

                </tr>

            </template>

        </tbody>

    </table>

</template>
```

# How It Works

### Step 1

Create an array in JavaScript.

```javascript
students = [

    {

        id: 1,

        firstName: 'Arun'

    },

    {

        id: 2,

        firstName: 'Rohan'

    }

];
```

---

### Step 2

Pass the array to `for:each`.

```html
<template for:each={students}>
```

---

### Step 3

Give a name to each object.

```html
for:item="stu"
```

---

### Step 4

Use the object's properties.

```html
{stu.firstName}

{stu.lastName}
```

---

### Step 5

Provide a unique key.

```html
<tr key={stu.id}>
```

LWC uses this key to efficiently update the DOM.

---

# Why is key Required?

Every repeated HTML element must have a unique identifier.

Correct

```html
<tr key={stu.id}>
```

Wrong

```html
<tr>
```

Without a key, LWC cannot efficiently track changes in the list, and the framework will throw an error.

# Real-World Example

Suppose you retrieve Account records from Apex.

JavaScript

```javascript
accounts = [

    {

        Id: '001',

        Name: 'ABC Pvt Ltd',

        Industry: 'IT'

    },

    {

        Id: '002',

        Name: 'XYZ Pvt Ltd',

        Industry: 'Manufacturing'

    }

];
```

HTML

```html
<template for:each={accounts} for:item="acc">

    <p key={acc.Id}>

        {acc.Name} - {acc.Industry}

    </p>

</template>
```

Output

```
ABC Pvt Ltd - IT

XYZ Pvt Ltd - Manufacturing
```

# Advantages of for:each

- Displays multiple records dynamically.
- Reduces duplicate HTML code.
- Makes components reusable.
- Easy to read and maintain.
- Automatically updates the UI when the array changes.
- Works with Apex records, wire adapters, and JavaScript arrays.

# Rules

### Rule 1

Use an array with `for:each`.

Correct

```javascript
students = [];
```

Wrong

```javascript
student = {};
```

---

### Rule 2

Always provide `for:item`.

```html
for:item="stu"
```

---

### Rule 3

Always use a unique key.

Correct

```html
key={stu.id}
```

Wrong

```html
key={stu.firstName}
```

(if first names are duplicated)

---

### Rule 4

The key must be unique for every record.

---

# Common Mistakes

## Forgetting key

Wrong

```html
<tr>

</tr>
```

Correct

```html
<tr key={stu.id}>
```

---

## Using an Object Instead of an Array

Wrong

```javascript
student = {};
```

Correct

```javascript
students = [];
```

---

## Using an Invalid Property

Wrong

```html
{stu.name}
```

if the object contains

```javascript
firstName
```

Correct

```html
{stu.firstName}
```

# Interview Questions

### What is `for:each` in LWC?

`for:each` is a template directive used to iterate over an array and render HTML for every item in the collection.

---

### What is `for:item`?

`for:item` represents the current object while iterating through an array.

---

### Why is the `key` attribute required?

The `key` attribute uniquely identifies each rendered element so LWC can efficiently update the DOM.

---

### Can `for:each` iterate over an object?

No.

It works only with arrays.

---

### What type of data is generally used with `for:each`?

- JavaScript arrays
- Lists returned from Apex
- Collections from wire adapters

---

### Is `key` mandatory?

Yes.

Every repeated element should have a unique key.

---

### Which property should be used as a key?

A unique value such as

```javascript
Id
```

or

```javascript
id
```
