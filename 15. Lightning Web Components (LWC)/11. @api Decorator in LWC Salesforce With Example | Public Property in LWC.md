# @api Decorator in LWC Salesforce With Example | Public Property in LWC

## Overview

The **@api** decorator is one of the most important decorators in **Lightning Web Components (LWC)**. It is used to expose a property or method so that it can be accessed by other Lightning Web Components.

The `@api` decorator is mainly used for **Parent-to-Child Communication**, where a parent component passes data to a child component through public properties.

In this tutorial, you'll learn:

- What is the `@api` decorator?
- Why it is used?
- Public properties in LWC
- Kebab Case in LWC
- Parent-to-Child Communication
- Practical examples with code

---

# What is @api Decorator?

According to the video:

> **It is used to expose a variable or functions publicly and make properties reactive.**

In simple words,

The **@api** decorator makes a property or method **public**, so it can be accessed from another Lightning Web Component.

Without using `@api`, variables and methods are **private** and can only be accessed within the same component.

---

# Why Do We Use @api?

The `@api` decorator is mainly used to:

- Expose public properties.
- Expose public methods.
- Receive data from a parent component.
- Make reusable components.
- Enable Parent-to-Child communication.

---

# Syntax

General syntax:

```javascript
@decorator propertyOrFunction;
```

Example:

```javascript
@api msg = 'Hello Parag';
```

Here,

- `@api` → Decorator
- `msg` → Public property
- `"Hello Parag"` → Default value

---

# Kebab Case in LWC

According to the video:

> **Kebab Case: Respective phrase will be transferred to all lowercase with hyphen (-) separating words.**

Example:

```javascript
headerLabel
```

becomes

```html
header-label
```

Similarly,

```javascript
studentName
```

becomes

```html
student-name
```

This conversion is important when passing values from the parent component to the child component.

# Parent-to-Child Communication Using @api

The `@api` decorator is commonly used for **Parent-to-Child communication**.

The flow is:

```
Parent Component
        │
        │ Pass Data
        ▼
Child Component
```

The parent sends values through attributes, and the child receives them using `@api`.

---

# Example 1: Simple Parent and Child Components

## Child Component

### childComponent.html

```html
<template>

    <lightning-card>

        <h1>This is Child Component.</h1>

    </lightning-card>

</template>
```

### childComponent.js-meta.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>

<LightningComponentBundle xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>true</isExposed>

    <targets>

        <target>lightning__AppPage</target>

        <target>lightning__HomePage</target>

        <target>lightning__RecordPage</target>

    </targets>

</LightningComponentBundle>
```

---

## Parent Component

### parentComponent.html

```html
<template>

    <lightning-card>

        <h1>This is Parent Component.</h1>

    </lightning-card>

</template>
```

### parentComponent.js-meta.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>

<LightningComponentBundle xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>true</isExposed>

    <targets>

        <target>lightning__AppPage</target>

        <target>lightning__HomePage</target>

        <target>lightning__RecordPage</target>

    </targets>

</LightningComponentBundle>
```

### Output

```
This is Child Component.

This is Parent Component.
```

At this stage, both components are independent.

# Example 2: Parent Includes Child Component

To display the child component inside the parent component, use the child component selector.

### parentComponent.html

```html
<template>

    <lightning-card>

        <h1>This is Parent Component.</h1>

        <c-api-child-component></c-api-child-component>

    </lightning-card>

</template>
```

### Output

```
This is Parent Component.

This is Child Component.
```

Here,

- The child component is rendered inside the parent component.
- No data is passed yet.

# Example 3: Child Component with a Variable (Without @api)

### childComponent.html

```html
<template>

    <lightning-card>

        <h1>This is Child Component.</h1>

        <div>{msg}</div>

    </lightning-card>

</template>
```

### childComponent.js

```javascript
import { LightningElement } from 'lwc';

export default class ApiChildComponent extends LightningElement {

    msg = 'Arun Kumar';

}
```

### Output

```
This is Child Component.

Arun Kumar
```

The variable `msg` is displayed only inside the child component.

---

# Example 4: Make Property Public Using @api

### childComponent.js

```javascript
import { LightningElement, api } from 'lwc';

export default class ApiChildComponent extends LightningElement {

    @api

    msg = 'Arun Kumar';

}
```

Now `msg` becomes a **public property**.

---

# Example 5: Parent Passes Data to Child

### parentComponent.html

```html
<template>

    <lightning-card>

        <h1>This is Parent Component.</h1>

        <c-api-child-component

            msg="Hello Piyush">

        </c-api-child-component>

    </lightning-card>

</template>
```

### Child Component

```javascript
import { LightningElement, api } from 'lwc';

export default class ApiChildComponent extends LightningElement {

    @api

    msg;

}
```

### childComponent.html

```html
<template>

    <lightning-card>

        <h1>This is Child Component.</h1>

        <div>{msg}</div>

    </lightning-card>

</template>
```

### Output

```
This is Parent Component.

This is Child Component.

Hello Piyush
```

The parent overrides the default value by passing a new value to the child.

---

# How Parent-to-Child Communication Works

### Step 1

Child component declares a public property.

```javascript
@api msg;
```

---

### Step 2

Parent component passes the value.

```html
<c-api-child-component

    msg="Hello Piyush">

</c-api-child-component>
```

---

### Step 3

Child receives the value.

```javascript
@api msg;
```

---

### Step 4

Display the value.

```html
<div>{msg}</div>
```

---

# Passing Multiple Values

Child

```javascript
import { LightningElement, api } from 'lwc';

export default class ChildComponent extends LightningElement {

    @api firstName;

    @api lastName;

}
```

Parent

```html
<c-child-component

    first-name="Arun"

    last-name="Kumar">

</c-child-component>
```

Child HTML

```html
<p>{firstName}</p>

<p>{lastName}</p>
```

# Public Methods Using @api

The `@api` decorator can also expose methods.

```javascript
import { LightningElement, api } from 'lwc';

export default class ChildComponent extends LightningElement {

    @api

    showMessage() {

        alert('Hello');

    }

}
```

The parent component can invoke this method using JavaScript.

# When Should You Use @api?

Use `@api` when:

- Making reusable components.
- Exposing methods publicly.
- Building configurable components.

# Common Mistakes

## Forgetting to Import @api

Wrong

```javascript
export default class Child extends LightningElement {

    @api msg;

}
```

Correct

```javascript
import { LightningElement, api } from 'lwc';
```

---

## Forgetting Kebab Case

Wrong

```html
<c-child

    firstName="Arun">

</c-child>
```

Correct

```html
<c-child

    first-name="Arun">

</c-child>
```

---

## Trying to Access Private Variables

Wrong

```javascript
msg = 'Hello';
```

The parent cannot access this variable.

Correct

```javascript
@api msg = 'Hello';
```

# Interview Questions

### What is the @api decorator?

The `@api` decorator exposes a property or method publicly so that it can be accessed by another Lightning Web Component.

---

### Why do we use @api?

It is used for:

- Parent-to-Child communication.
- Public properties.
- Public methods.

---

### Is @api reactive?

Yes.

When the parent updates the value, the child component automatically rerenders with the new value.

---

### Can @api be used on methods?

Yes.

Both properties and methods can be decorated using `@api`.

---

### What is Kebab Case?

Kebab Case converts camelCase property names into lowercase words separated by hyphens.

Example:

```javascript
studentName
```

becomes

```html
student-name
```

---

### Can the child modify an @api property?

Yes, but it is recommended that the parent controls the value and the child treats it as input to maintain predictable data flow.

---

# Best Practices

- Use `@api` only for properties or methods that need to be accessed by another component.
- Follow **camelCase** in JavaScript and **kebab-case** in HTML attributes.
- Keep public APIs minimal to make components easier to maintain.
- Use descriptive property names for better readability.
