# Use of @wire Decorator on Function in LWC Salesforce | Wire Function Example

## Overview

In Lightning Web Components (LWC), the **@wire** decorator can be used with either a **property** or a **function**. When used with a function, the wire service automatically calls the function whenever data is available or whenever the reactive parameters change.

Unlike wiring a property, wiring a function gives you more control over handling the returned data and errors before storing them in component properties.

In this tutorial, you'll learn:

- What is `@wire` on a function?
- Why do we use a wired function?
- Difference between wiring a property and a function.
- Complete example using `getRecord`.
- Code explanation.
- Advantages and best practices.
- Common mistakes.
- Interview questions.

---

# What is @wire Decorator?

According to the video:

> **To read Salesforce data, Lightning Web Components use a reactive wire service. When the wire service provisions data, the component rerenders. Components use @wire in their JavaScript class to specify a wire adapter or an Apex method.**

The same concept applies when decorating a function.

Instead of storing the response directly into a property, the wire service invokes the decorated function and passes the response (`data` and `error`) as an object.

---

# Why Use @wire on a Function?

Using `@wire` on a function allows you to:

- Perform custom processing before displaying data.
- Handle success and error responses separately.
- Store data in your own variables.
- Execute additional logic after receiving data.
- Improve code flexibility.

---

# Syntax

According to the video:

```javascript
import { adapterId } from 'adapterModule';

@wire(adapterId, adapterConfig)
propertyOrFunction;
```

When wiring a function:

```javascript
@wire(getRecord, {
    recordId: '$recordId',
    fields: ['Account.Name']
})
wiredAccount({ data, error }) {

}
```

The function receives an object containing:

- `data`
- `error`

Only one of them contains a value at a time.

# Property Wire vs Function Wire

## Property Wire

```javascript
@wire(getRecord, {
    recordId: '$recordId',
    fields: ['Account.Name']
})
account;
```

Salesforce automatically stores the response in the property.

---

## Function Wire

```javascript
@wire(getRecord, {
    recordId: '$recordId',
    fields: ['Account.Name']
})
wiredAccount({ data, error }) {

}
```

Salesforce calls the function whenever new data is available.

# Example: Using @wire on a Function

This example retrieves the current Account record and displays:

- Account Name
- Account Phone

---

## HTML File

### wireExampleWithDecorators.html

```html
<template>

    <lightning-card title="Wired Decorator with Functions">

        <template if:true={accounts}>

            <div>

                Name : {getName}

            </div>

            <div>

                Phone : {getPhone}

            </div>

        </template>

    </lightning-card>

</template>
```

---

## JavaScript File

### wireExampleWithDecorators.js

```javascript
import { getRecord } from 'lightning/uiRecordApi';

import {
    LightningElement,
    api,
    wire,
    track
} from 'lwc';

export default class WireExampleWithDecorators
extends LightningElement {

    @api recordId;

    @track accounts;

    @track error;

    @wire(getRecord, {

        recordId: '$recordId',

        fields: [

            'Account.Name',

            'Account.Phone'

        ]

    })

    wiredAccounts({ data, error }) {

        if (data) {

            this.accounts = data;

            this.error = undefined;

        }

        else if (error) {

            this.error = error;

            this.accounts = undefined;

        }

    }

    get getName() {

        return this.accounts.fields.Name.value;

    }

    get getPhone() {

        return this.accounts.fields.Phone.value;

    }

}
```

# How This Example Works

## Step 1

Import the required modules.

```javascript
import { getRecord }
from 'lightning/uiRecordApi';

import {
    LightningElement,
    api,
    wire,
    track
}
from 'lwc';
```

Here:

- `getRecord` retrieves the Salesforce record.
- `@wire` connects the component to the wire service.
- `@api` receives the current record Id.
- `@track` stores the retrieved data and errors.

---

## Step 2

Receive the Record Id.

```javascript
@api recordId;
```

When the component is placed on a Record Page, Salesforce automatically provides the current record Id.

---

## Step 3

Create variables.

```javascript
@track accounts;

@track error;
```

These variables store:

- Retrieved record data
- Error information

---

## Step 4

Decorate the function.

```javascript
@wire(getRecord, {

    recordId: '$recordId',

    fields: [

        'Account.Name',

        'Account.Phone'

    ]

})

wiredAccounts({ data, error }) {

}
```

Whenever the wire service retrieves data, this function is executed automatically.

---

## Step 5

Handle successful response.

```javascript
if(data){

    this.accounts = data;

    this.error = undefined;

}
```

If the request succeeds:

- Store the record.
- Clear previous errors.

---

## Step 6

Handle errors.

```javascript
else if(error){

    this.error = error;

    this.accounts = undefined;

}
```

If an error occurs:

- Store the error.
- Remove old data.

---

## Step 7

Create Getter for Name.

```javascript
get getName(){

    return this.accounts.fields.Name.value;

}
```

This getter extracts the Account Name.

---

## Step 8

Create Getter for Phone.

```javascript
get getPhone(){

    return this.accounts.fields.Phone.value;

```

This getter extracts the Account Phone.

---

## Step 9

Display values.

```html

Phone : {getPhone}
```

The HTML automatically updates whenever the wire service provides new data.

# Why Use a Wired Function?

Property wiring is simple:

```javascript
@wire(getRecord)

account;
```

However, function wiring lets you:

- Modify data before displaying it.
- Filter records.
- Combine multiple responses.
- Handle errors more cleanly.
- Execute additional logic.

For complex applications, function wiring is often the preferred approach.

---

# Advantages

- Better control over returned data.
- Easier error handling.
- Custom data processing.
- Cleaner business logic.
- Reactive updates.
- Automatic rerendering.

# Common Mistakes

## Forgetting `$` in recordId

Wrong

```javascript
recordId: recordId
```

Correct

```javascript
recordId: '$recordId'
```

---

## Not Checking for Data

Wrong

```javascript
this.accounts.fields.Name.value;
```

Correct

```javascript
if(data){

    this.accounts = data;

}
```

Always verify that data exists before accessing it.

---

## Not Handling Errors

Always write:

```javascript
if(data){

}

else if(error){

}
```

Ignoring errors can cause runtime issues.

---

## Forgetting to Import `wire`

Wrong

```javascript
@wire(getRecord)
```

Correct

```javascript
import { wire } from 'lwc';
```

---

## Accessing Undefined Data

Avoid accessing:

```javascript
this.accounts.fields.Name.value;
```

before the wire service has returned data. Ensure the HTML or JavaScript checks for available data first.

---

# Best Practices

- Use function wiring when additional processing is required.
- Handle both `data` and `error`.
- Keep the wired function focused on data handling.
- Use getters to simplify template expressions.
- Use reactive parameters (such as `$recordId`) whenever appropriate.

> **Note:** In modern LWC, primitive fields are reactive by default. The `@track` decorator is mainly needed when you want to observe changes to nested properties of objects or arrays. For simple examples like this, `@track` is often optional.

# Interview Questions

### What is the difference between wiring a property and wiring a function?

- Property wiring stores the response directly in a property.
- Function wiring invokes a function with `{ data, error }`, allowing custom processing before storing or displaying the data.

---

### When should we use function wiring?

Use function wiring when:

- Additional processing is required.
- You need custom error handling.
- You want to transform the returned data.
- You need to execute extra logic after receiving data.

---

### What arguments does a wired function receive?

A wired function receives an object containing:

```javascript
{ data, error }
```

Only one of them contains a value at a given time.

---

### Why is `$recordId` used?

The `$` makes the parameter reactive. If `recordId` changes, the wire service automatically executes the function again with the updated value.

---

### Does a wired function rerender the component?

Yes. When the wire service provisions new data, the component rerenders automatically.

---

### Can a wired function call Apex methods?

Yes. A wired function can use:

- Lightning Data Service adapters (such as `getRecord`)
- Apex methods annotated with `@AuraEnabled(cacheable=true)`
