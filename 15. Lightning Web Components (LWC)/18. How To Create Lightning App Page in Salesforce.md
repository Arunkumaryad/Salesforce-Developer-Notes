# How To Create Lightning App Page in Salesforce

## Overview

A **Lightning App Page** is a custom page created using **Lightning App Builder**. It allows you to design a page by simply dragging and dropping Lightning components without writing code.

Lightning App Pages are commonly used to:

- Display custom Lightning Web Components (LWC)
- Show dashboards and reports
- Display related records
- Combine multiple components on a single page
- Build custom business applications

Unlike Record Pages or Home Pages, an **App Page** is independent and can be added as a navigation tab inside any Lightning App.

---

# What is Lightning App Builder?

**Lightning App Builder** is a drag-and-drop tool provided by Salesforce that enables administrators and developers to create custom pages for Lightning Experience and the Salesforce Mobile App.

Using Lightning App Builder, you can create:

- App Pages
- Home Pages
- Record Pages

without writing any Apex or Visualforce code.

---

# What is a Lightning App Page?

A **Lightning App Page** is a custom page created in Lightning App Builder that displays Lightning components in a layout of your choice.

You can place:

- Lightning Web Components (LWC)
- Aura Components
- Standard Salesforce Components
- Reports
- Dashboards
- Related Lists
- Rich Text
- Charts

on a single page.

---

# Prerequisites

Before creating a Lightning App Page, make sure:

- Lightning Experience is enabled.
- You have the **Customize Application** permission.
- Your Lightning Web Components are deployed.
- Your component is exposed using:

```xml
<isExposed>true</isExposed>
```

and proper targets are added.

Example

```xml
<targets>

    <target>lightning__AppPage</target>

</targets>
```


Example

```

```

# Difference Between Lightning Pages

| Lightning App Page | Home Page | Record Page |
|--------------------|-----------|-------------|
| Independent page | Default home screen | Opens with a record |
| Can contain LWCs | Can contain LWCs | Can contain LWCs |
| Added as Navigation Item | Default Home | Record-specific layout |
| Used for custom applications | Used for dashboards | Used for Account, Contact, Lead etc. |

# Best Practices

- Give meaningful page names.
- Use responsive layouts.
- Group related components together.
- Avoid placing too many components on one page.
- Keep the page lightweight for better performance.
- Activate the page only after testing.
- Add the page to the required Lightning App navigation.

---

# Common Mistakes

## Forgetting to Expose the LWC

Wrong

```xml
<isExposed>false</isExposed>
```

Correct

```xml
<isExposed>true</isExposed>
```

---

## Missing App Page Target

Wrong

```xml
<targets>

    <target>lightning__RecordPage</target>

</targets>
```

Correct

```xml
<targets>

    <target>lightning__AppPage</target>

</targets>
```

---

## Not Activating the Page

Saving does **not** automatically make the page available.

Always click

```
Activate
```

after saving.

---

## Forgetting Navigation

Many beginners successfully create an App Page but cannot find it later because it was not added to the app's navigation menu.

Always:

```
Edit Navigation

↓

Add More Items

↓

Select Your Page

↓

Save
```

# Interview Questions

### What is Lightning App Builder?

Lightning App Builder is a drag-and-drop tool used to create custom Lightning Pages without writing code.

---

### What types of pages can Lightning App Builder create?

- App Page
- Home Page
- Record Page

---

### What is a Lightning App Page?

A Lightning App Page is a standalone custom page that can display Lightning components and can be added as a navigation tab inside a Lightning App.

---

### Can Lightning Web Components be added to an App Page?

Yes.

As long as the component is exposed and includes:

```xml
<target>lightning__AppPage</target>
```

---

### What happens after clicking Save?

The page is saved but is **not visible** until it is activated.

---

### Why do we activate a Lightning Page?

Activation makes the page available to users and allows Salesforce to use it within the selected app or navigation.

---

### How do you make an App Page appear in the navigation bar?

Open the Lightning App → Edit Navigation → Add More Items → Select the App Page → Save.

---

### What is the difference between an App Page and a Record Page?

- **App Page:** Standalone page accessed from the app navigation.
- **Record Page:** Opens when viewing a specific record (Account, Contact, Opportunity, etc.).
