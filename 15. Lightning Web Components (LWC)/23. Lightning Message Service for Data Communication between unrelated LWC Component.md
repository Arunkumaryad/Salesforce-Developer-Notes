# Lightning Message Service (LMS) for Data Communication between Unrelated LWC Components

## Overview

In Lightning Web Components (LWC), components are not always related to each other. Sometimes two components are placed on the same Lightning App Page but have **no Parent-Child relationship**. In such cases, they cannot communicate using `@api` or `CustomEvent`.

Salesforce provides **Lightning Message Service (LMS)** to enable communication between unrelated components.

Lightning Message Service allows one component to **publish** data and another component to **subscribe** to receive that data.

---

# What is Lightning Message Service (LMS)?

**Lightning Message Service (LMS)** is a Salesforce feature that enables communication between unrelated Lightning Web Components (LWC), Aura Components, and Visualforce Pages without requiring a Parent-Child relationship.

It works using a **Message Channel**, where one component publishes a message and another component subscribes to receive it.

Communication between unrelated components is done using **Lightning Message Service (LMS).**

# Components Used in LMS

There are **three parts** involved in Lightning Message Service.

## 1. Publisher

**Definition**

A **Publisher** is the component that sends data to other components.

It uses the following method:

```javascript
publish(this.messageContext, MessageChannel, payload);
```

---

## 2. Subscriber

**Definition**

A **Subscriber** is the component that receives data published by the Publisher.

It uses the following method:

```javascript
subscribe(
    this.messageContext,
    MessageChannel,
    (message) => {
        // Statements
    }
);
```

---

## 3. Message Channel

A **Message Channel** acts as the communication bridge between the Publisher and Subscriber.

## publisherComponent.js

```javascript
import { LightningElement, wire } from 'lwc';

import {
    MessageContext,
    publish
} from 'lightning/messageService';

import ComrevoChannel
from '@salesforce/messageChannel/ComrevoChannel__c';

export default class PublisherComponent extends LightningElement {

    name = '';

    @wire(MessageContext)
    messageContext;

    handleChange(event) {

        this.name = event.target.value;

    }

    handleClick() {

        let payload = {

            name: this.name

        };

        publish(
            this.messageContext,
            ComrevoChannel,
            payload
        );

    }

}
```

## subscriberComponent.js

```javascript
import { LightningElement, wire } from 'lwc';

import {

MessageContext,

subscribe,

unsubscribe

}

from 'lightning/messageService';

import ComrevoChannel

from '@salesforce/messageChannel/ComrevoChannel__c';

export default class SubscriberComponent extends LightningElement {

    name = '';

    subscription = null;

    @wire(MessageContext)

    messageContext;

    connectedCallback() {

        this.handleSubscribe();

    }

    disconnectedCallback() {

        this.handleUnSubscribe();

    }

    handleSubscribe() {

        if (!this.subscription) {

            this.subscription = subscribe(

                this.messageContext,

                ComrevoChannel,

                (parameter) => {

                    this.name = parameter.name;

                }

            );

        }

    }

    handleUnSubscribe() {

        unsubscribe(this.subscription);

        this.subscription = null;

    }

}
```

# LMS vs Parent-Child Communication

| Feature | Parent-Child | Child-Parent | Lightning Message Service |
|----------|--------------|--------------|---------------------------|
| Related Components | Yes | Yes | No |
| Uses @api | Yes | No | No |
| Uses CustomEvent | No | Yes | No |
| Uses Message Channel | No | No | Yes |
| Works Across Independent Components | No | No | Yes |

# Advantages

- Allows communication between unrelated components.
- Supports LWC, Aura, and Visualforce.
- Components remain loosely coupled.
- Easy to maintain.
- Real-time message delivery.
- Recommended Salesforce solution for cross-component communication.

# Best Practices

- Create one Message Channel per business purpose.
- Use meaningful channel names.
- Always unsubscribe in `disconnectedCallback()`.
- Keep payloads lightweight.
- Publish only required data.

# Interview Questions

### What is Lightning Message Service (LMS)?

Lightning Message Service is a Salesforce feature that enables communication between unrelated Lightning Web Components, Aura Components, and Visualforce Pages using a Message Channel.

---

### When should you use LMS?

Use LMS when components do not have a Parent-Child relationship but still need to exchange data.

---

### What are the main parts of LMS?

- Publisher
- Subscriber
- Message Channel

---

### What does `publish()` do?

It sends data from the Publisher Component to all subscribed components through the Message Channel.

---

### What does `subscribe()` do?

It listens for messages on a specific Message Channel and executes a callback whenever a new message is received.

---

### Why is `unsubscribe()` important?

It stops listening for messages when the component is destroyed, preventing unnecessary processing and memory leaks.
