# How To Run Apex Code in VS Code | How To Create Apex Class in Visual Studio Code

## Overview

Visual Studio Code (VS Code) is one of the most popular development environments for Salesforce developers. Using VS Code, developers can:

- Create Apex Classes
- Edit Apex Code
- Execute Anonymous Apex
- Retrieve Metadata from Salesforce Org
- Deploy Code to Salesforce Org
- Enable Auto Deploy
- Manage Salesforce Projects

This guide covers the complete setup and workflow required to work with Apex code in VS Code.

---

# Development Environment Setup

### Actual Definition (From Notes)

To work with Apex in VS Code, install:

1. Salesforce CLI
2. VS Code
3. Salesforce Extension Pack

---

# Required Software

## 1. Visual Studio Code (VS Code)

VS Code is a lightweight source code editor developed by Microsoft.

Features:

- Syntax Highlighting
- Debugging Support
- Git Integration
- Extensions Support

---

## 2. Salesforce CLI

### Terminology

#### SFDX

### Actual Definition (From Notes)

> SFDX stands for Salesforce Developer Experience.

> It is a set of tools and processes developed by Salesforce to make it easier for developers to build and manage applications on the Salesforce platform.

### Purpose

Salesforce CLI allows developers to:

- Create Projects
- Authorize Orgs
- Deploy Metadata
- Retrieve Metadata
- Run Apex Code
- Execute Tests

---

## 3. Salesforce Extension Pack

Salesforce provides official VS Code extensions.

Features:

- Apex Language Support
- LWC Support
- SOQL Query Support
- Deployment Tools
- Org Browser

# How to Create a Salesforce Project

## Step 1

Open VS Code.

---

## Step 2

Open Command Palette.

Shortcut:

```text
CTRL + SHIFT + P
```

---

## Step 3

Search for:

```text
SFDX: Create Project
```

---

## Step 4

Select:

```text
Standard Project
```

---

## Step 5

Enter Project Name.

Example:

```text
SalesforceProject
```

---

## Step 6

Select Project Location.

Project is created successfully.

---

# How to Authorize Salesforce Org

Before running Apex code, connect VS Code with Salesforce Org.

---

## Step 1

Open Command Palette.

```text
CTRL + SHIFT + P
```

---

## Step 2

Search:

```text
SFDX: Authorize an Org
```

---

## Step 3

Select Environment:

```text
Production
```

or

```text
Sandbox
```

---

## Step 4

Login to Salesforce.

---

## Step 5

Grant Access.

Your org is now connected to VS Code.

# How to Run Anonymous Apex Code

### Actual Definition (From Notes)

## Step 1

Go to Command Palette.

Shortcut:

```text
CTRL + SHIFT + P
```

Search:

```text
SFDX: Create Project
```

Follow the project creation steps.

---

## Step 2

Again open Command Palette.

Shortcut:

```text
CTRL + SHIFT + P
```

Search:

```text
SFDX: Authorize an Org
```

Follow the authorization steps.

---

## Step 3

Navigate to:

```text
scripts
   ↓
apex
   ↓
hello.apex
```

---

## Step 4

Write Apex Code.

Example:

```apex
System.debug('Hello Salesforce');
```

---

## Step 5

Open Command Palette.

Search:

```text
SFDX: Execute Anonymous Apex with Editor Contents
```

---

## Example

```apex
Account acc = new Account(
    Name = 'Demo Account'
);

insert acc;

System.debug(
    'Account Created Successfully'
);
```

# How to Create an Apex Class

## Method 1: Using Command Palette

Open:

```text
CTRL + SHIFT + P
```

Search:

```text
SFDX: Create Apex Class
```

Enter:

```text
AccountUtility
```

VS Code creates:

```text
AccountUtility.cls
AccountUtility.cls-meta.xml
```

---

## Example Apex Class

```apex
public class AccountUtility {

    public static void createAccount() {

        Account acc =
        new Account(
            Name = 'Test Account'
        );

        insert acc;
    }
}
```

---

# How to Edit Apex Classes and Deploy Back to Org

### Actual Definition (From Notes)

### Step 1

Go to:

```text
Org Browser
```

(Cloud Icon)

Then:

```text
Apex Classes
```

Click:

```text
Retrieve Source from Org
```

(Cloud and Arrow Icon)

---

### Step 2

Go to Explorer.

```text
force-app
   ↓
main
   ↓
default
   ↓
classes
```

---

### Step 3

Select Apex Class.

Edit Code.

Save:

```text
CTRL + S
```

---

### Step 4

To Deploy:

Open Command Palette.

Select:

```text
SFDX: Deploy This Source to Org
```

---

# Example Deployment Workflow

```text
Retrieve Class
       ↓
Edit Class
       ↓
Save File
       ↓
Deploy Source
       ↓
Changes Available in Salesforce Org
```

---

# How to Retrieve Apex Classes from Org

## Method

Open:

```text
Org Browser
```

Select:

```text
Apex Classes
```

Choose Class.

Click:

```text
Retrieve Source from Org
```

The class is downloaded to your local project.

---

# How to Activate Auto Deploy

### Actual Definition (From Notes)

## Step 1

Go to:

```text
File
   ↓
Preferences
   ↓
Settings
```

---

## Step 2

Search:

```text
Auto Deploy
```

---

## Step 3

Enable Checkbox:

```text
salesforcedx-vscode-core
    ↓
push-or-deploy-on-save
    ↓
Enabled
```

---

# What Happens After Enabling Auto Deploy?

Whenever you save:

```text
CTRL + S
```

VS Code automatically deploys changes to Salesforce Org.

---

# Benefits of Auto Deploy

- Faster Development
- No Manual Deployment
- Immediate Synchronization
- Better Productivity
