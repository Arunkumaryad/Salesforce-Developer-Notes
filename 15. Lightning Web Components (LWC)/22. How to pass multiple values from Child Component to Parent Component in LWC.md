# How to Pass Multiple Values from Child Component to Parent Component in LWC

## Overview

In Lightning Web Components (LWC), there are many situations where a Child Component needs to send **multiple pieces of data** to its Parent Component. Instead of sending only a single value, we can send an **object** inside the `detail` property of a **Custom Event**.

This approach is commonly used when a Child Component collects multiple input values (such as First Name, Last Name, Email, Phone, etc.) and passes them together to the Parent Component.

---

# Definition

Passing multiple values from a Child Component to a Parent Component means sending an **object containing multiple properties** through the `detail` property of a **CustomEvent**.

Instead of writing:

```javascript
detail: this.firstName
```

we write:

```javascript
detail: {
    FirstName: this.firstName,
    LastName: this.lastName
}
```

The Parent Component can then access each value individually using:

```javascript
event.detail.FirstName
event.detail.LastName
```

---

# Why Do We Need to Pass Multiple Values?

Imagine a registration form.

The Child Component collects:

- First Name
- Last Name
- Email
- Mobile Number

Instead of sending four separate events, we send **one object** containing all values.

This makes the code:

- Cleaner
- Faster
- Easier to maintain
- More scalable

---

# Step 1: Child Component

The Child Component collects the First Name and Last Name from the user and sends both values to the Parent Component.

---

## childComponent1.html

```html
<template>

    <lightning-card title="Child to Parent Communication">

        <lightning-input
            label="Enter First Name"
            onchange={handleFirstName}>
        </lightning-input>

        <lightning-input
            label="Enter Last Name"
            onchange={handleLastName}>
        </lightning-input>

        <lightning-button
            label="Click Me!"
            onclick={handleClick}>
        </lightning-button>

    </lightning-card>

</template>
```

---

### Explanation

This component contains:

- One input for First Name.
- One input for Last Name.
- One button to send both values to the Parent Component.

---

## childComponent1.js

```javascript
import { LightningElement } from 'lwc';

export default class ChildComponent1 extends LightningElement {

    firstName = '';
    lastName = '';

    handleFirstName(event) {
        this.firstName = event.target.value;
    }

    handleLastName(event) {
        this.lastName = event.target.value;
    }

    handleClick() {

        console.log('Button Clicked');

        const newEvent = new CustomEvent('getname', {

            detail: {

                FirstName: this.firstName,
                LastName: this.lastName

            }

        });

        this.dispatchEvent(newEvent);

    }

}
```

---

```javascript

lastName = '';
```

These variables store the user input.


## Step 3

Whenever the user enters the Last Name,

```javascript

```

runs.

```javascript

```

Example

```

↓
```

## Step 4

When the button is clicked,

```javascript

```

```javascript

        FirstName: this.firstName,
```

Instead of sending one value,

we send an object.

The object contains

```

    FirstName : "Arun",

```

## Step 5

Finally,

```javascript

```

sends the event to the Parent Component.

# Step 2: Parent Component

The Parent Component receives the object and displays both values.

---

## parentComponent1.html

```html
<template>

    <lightning-card>

        <c-child-component1
            ongetname={handleName}>
        </c-child-component1>

    </lightning-card>

    <lightning-card>

        <div>
            FirstName : {firstName}
        </div>

        <div>
            LastName : {lastName}
        </div>

    </lightning-card>

```

---

### Explanation

The Parent includes the Child Component.

```html
<c-child-component1>
```

The Parent listens for the event.

```html
ongetname={handleName}
```

When the Child dispatches

```
getname
```

the Parent automatically calls

```
handleName()
```

---

## parentComponent1.js

```javascript
import { LightningElement } from 'lwc';

export default class ParentComponent1 extends LightningElement {

    firstName = '';
    lastName = '';

    handleName(event) {

        this.firstName = event.detail.FirstName;
        this.lastName = event.detail.LastName;

    }

}
```

---

# Code Explanation

The Parent receives the event.

The data is available inside

```javascript
event.detail
```

Since the Child sent an object,

```javascript
event.detail
```

contains

```javascript
{

    FirstName: "Arun",

    LastName: "Kumar"

}
```

Therefore,

```javascript
this.firstName = event.detail.FirstName;
```

stores

```
Arun
```

and

```javascript
this.lastName = event.detail.LastName;
```

stores

```
Kumar
```

The UI automatically updates because LWC is reactive.

# Advantages

- Sends multiple values using a single event.
- Makes the code cleaner and more readable.
- Reduces unnecessary event dispatches.
- Easy to extend by adding more properties.
- Improves component communication.
- Follows Salesforce best practices.

# Best Practices

- Send related values together in one object.
- Use meaningful property names.
- Keep event names descriptive.
- Let the Parent handle business logic.
- Send only the required data.

# Common Mistakes

## 1. Sending Multiple Separate Events

❌ Incorrect

```javascript
this.dispatchEvent(event1);

this.dispatchEvent(event2);
```

---

✅ Correct

```javascript
this.dispatchEvent(

    new CustomEvent('getname', {

        detail: {

            FirstName,

            LastName

        }

    })

);
```

---

## 2. Accessing Wrong Property

❌ Incorrect

```javascript
event.FirstName
```

---

✅ Correct

```javascript
event.detail.FirstName
```

---

## 3. Typing Incorrect Property Name

Child

```javascript
detail: {

    FirstName: this.firstName

}
```

Parent

```javascript
event.detail.firstname
```

This returns

```
undefined
```

because JavaScript is case-sensitive.

---

✅ Correct

```javascript
event.detail.FirstName
```

---

## 4. Forgetting dispatchEvent()

Creating the event is not enough.

You must dispatch it.

❌ Incorrect

```javascript
const event = new CustomEvent('getname');
```

---

✅ Correct

```javascript
this.dispatchEvent(event);
```

# Interview Questions

### How do you pass multiple values from a Child Component to a Parent Component?

By passing an object inside the `detail` property of a **CustomEvent**.

---

### Why use an object instead of multiple events?

Using an object allows multiple related values to be sent together in a single event, making the code cleaner, easier to maintain, and more efficient.

---

### Where are the values stored in the Parent Component?

The values are stored in the `event.detail` object.

Example:

```javascript
event.detail.FirstName

event.detail.LastName
```

---

### Can we send arrays or objects using detail?

Yes. The `detail` property can contain:

- Strings
- Numbers
- Booleans
- Objects
- Arrays

---

### Can we send five or more values?

Yes. You can send as many related values as needed by adding more properties to the object.

Example:

```javascript
detail: {

    name,

    email,

    phone,

    city,

    state,

    country

}
```
