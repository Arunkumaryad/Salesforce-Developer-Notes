# Salesforce Lightning Web Components (LWC) Introduction

## Overview

Lightning Web Components (LWC) is Salesforce's modern framework for building fast, lightweight, and reusable user interface components.

LWC is based on modern web standards such as:

- HTML
- JavaScript
- CSS
- Web Components

It was introduced by Salesforce in **2019** as a modern alternative to Aura Components.

---

# What is LWC?

## Actual Definition (From Notes)

### 1. LWC stands for Lightning Web Components.

### 2. It is a programming model.

### 3. It is used to develop Lightning Components.

### 4. Lightning Web Components (2019) and Aura Components (2014) both are used to develop Lightning Components.

### 5. Lightning Web Components are custom HTML elements built using HTML and modern JavaScript. To admins and end users, they both appear as Lightning Components.

### 6. Lightning Web Components (LWC) is available in Enterprise, Performance, Unlimited, and Developer editions.

### 7. Lightning Web Components is lightweight and delivers exceptional performance. Most of the code you write is standard JavaScript and HTML.

---

# Simple Definition

Lightning Web Components (LWC) is Salesforce's modern UI development framework that allows developers to build reusable Lightning Components using standard web technologies like HTML, JavaScript, and CSS.

---

# What is a Lightning Component?

A Lightning Component is a reusable UI block that can be placed on:

- Record Pages
- App Pages
- Home Pages
- Experience Cloud Sites
- Utility Bars
- Quick Actions

Examples:

- Account Details Component
- Contact List Component
- Product Catalog Component
- Dashboard Widget
- Custom Form

# Why Salesforce Introduced LWC?

## Actual Definition (From Notes)

### Why Salesforce launched LWC when Aura Components was already there?

LWC is built on modern web standards and leverages native browser capabilities, resulting in better performance compared to Aura components.

LWC has a smaller runtime footprint, faster rendering, and optimized event handling, leading to improved overall performance.

---

# Simple Explanation

Aura Components work well, but Salesforce wanted a framework that:

- Is faster
- Uses modern JavaScript
- Uses browser-native features
- Has better performance
- Is easier to learn for web developers

Therefore Salesforce introduced Lightning Web Components.

# Architecture of LWC

```text
Lightning Web Component
        │
        ├── HTML
        │
        ├── JavaScript
        │

        │
        └── XML Metadata
```

---

# LWC Component Structure

Example Component:

```text
helloWorld
```

Files created:

```text
helloWorld.html
helloWorld.js
helloWorld.js-meta.xml
helloWorld.css (optional)
```

---

# File Explanation

## 1. HTML File

Contains UI markup.

Example:

```html
<template>
    <h1>Hello World</h1>
</template>
```

---

## 2. JavaScript File

Contains component logic.

Example:

```javascript
import { LightningElement } from 'lwc';

export default class HelloWorld
extends LightningElement {

}
```

---

## 3. Meta XML File

Controls where the component can be used.

Example:

```xml
<?xml version="1.0" encoding="UTF-8"?>

<LightningComponentBundle
xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>true</isExposed>

    <targets>

        <target>
            lightning__RecordPage
        </target>

        <target>
            lightning__AppPage
        </target>

        <target>
            lightning__HomePage
        </target>

    </targets>

</LightningComponentBundle>
```
