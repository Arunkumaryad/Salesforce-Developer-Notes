# Conditional Rendering in HTML in LWC | Apply if-else in Lightning Web Components

## Overview

Conditional Rendering is a feature in **Lightning Web Components (LWC)** that allows you to display or hide HTML elements based on a condition.

Instead of always displaying every element on the page, you can render specific content only when a condition is **true** or **false**.

This helps create dynamic and interactive user interfaces.

---

# What is Conditional Rendering?

Conditional Rendering means showing or hiding HTML content depending on a Boolean value.

For example,

- Show user details only if a checkbox is selected.
- Display a success message after a record is saved.
- Show an error message when validation fails.

Instead of displaying everything, LWC decides what should appear on the screen.

---

# Template Directives

According to the video, LWC provides two template directives for conditional rendering:

1. **if:true**
2. **if:false**

These directives render HTML based on the Boolean property's value.

---

# if:true

The `if:true` directive displays its content **only when the Boolean value is true**.

### Syntax

```html
<template if:true={propertyName}>

    <!-- HTML Content -->

</template>
```

---

### Example

```html
<template if:true={isChecked}>

    <h1>You are inside True Block</h1>

</template>
```

If

```javascript
isChecked = true;
```

Output

```text
You are inside True Block
```

If

```javascript
isChecked = false;
```

Nothing is displayed.

---

# if:false

The `if:false` directive displays its content **only when the Boolean value is false**.

### Syntax

```html
<template if:false={propertyName}>

    <!-- HTML Content -->

</template>
```

---

### Example

```html
<template if:false={isChecked}>

    <h1>You are inside False Block</h1>

</template>
```

If

```javascript
isChecked = false;
```

Output

```text
You are inside False Block
```

# Example Used in the Video

## JavaScript (conditionalRendering.js)

```javascript
import { LightningElement } from 'lwc';

export default class ConditionalRendering extends LightningElement {

    isChecked = false;

    handleCheckbox(event) {

        this.isChecked = event.target.checked;

    }

}
```

---

## HTML (conditionalRendering.html)

```html
<template>

    <lightning-card>

        <lightning-input
            type="checkbox"
            label="Show Details"
            onchange={handleCheckbox}>
        </lightning-input>

        <template if:true={isChecked}>

            <h1 style="color: green;">
                You are currently in True Block.
            </h1>

        </template>

        <template if:false={isChecked}>

            <h1 style="color: red;">
                You are currently in False Block.
            </h1>

        </template>

    </lightning-card>

</template>
```

# How the Example Works

### Step 1

Initially,

```javascript
isChecked = false;
```

---

### Step 2

The HTML checks

```html
if:false={isChecked}
```

Since the value is false,

```
You are currently in False Block.
```

is displayed.

---

### Step 3

The user checks the checkbox.

The `onchange` event is fired.

```html
onchange={handleCheckbox}
```

---

### Step 4

The JavaScript method executes.

```javascript
handleCheckbox(event) {

    this.isChecked = event.target.checked;

}
```

`event.target.checked` returns

```
true
```

---

### Step 5

Now

```javascript
isChecked = true;
```

---

### Step 6

LWC automatically re-renders the template.

The following block becomes visible.

```html
<template if:true={isChecked}>
```

The following block is hidden.

```html
<template if:false={isChecked}>
```

---

# Understanding event.target.checked

For a checkbox,

```javascript
event.target.checked
```

returns

| Checkbox | Value |
|----------|-------|
| Checked | true |
| Unchecked | false |

Example

```javascript
this.isChecked = event.target.checked;
```

This updates the Boolean property automatically.

# Real-World Example

Suppose you want to display order details only after clicking a checkbox.

JavaScript

```javascript
showOrder = false;

handleChange(event) {

    this.showOrder = event.target.checked;

}
```

HTML

```html
<lightning-input
    type="checkbox"
    label="Show Order"
    onchange={handleChange}>
</lightning-input>

<template if:true={showOrder}>

    <p>Order Number : ORD-1001</p>

    <p>Status : Delivered</p>

</template>
```

Initially, the order details are hidden.

When the checkbox is selected, the order information becomes visible.

# Rules

### Rule 1

The property used in `if:true` or `if:false` should be a Boolean.

Correct

```javascript
isChecked = true;
```

---

### Rule 2

Use template directives.

Correct

```html
<template if:true={isChecked}>
```

---

### Rule 3

Use the property name inside curly braces.

Correct

```html
if:true={isChecked}
```

Wrong

```html
if:true="isChecked"
```

---

### Rule 4

Update the Boolean value whenever the condition changes.

Example

```javascript
this.isChecked = event.target.checked;
```

---

# Common Mistakes

## Using String Instead of Boolean

Wrong

```javascript
isChecked = "true";
```

Correct

```javascript
isChecked = true;
```

---

## Forgetting Curly Braces

Wrong

```html
if:true="isChecked"
```

Correct

```html
if:true={isChecked}
```

---

## Forgetting to Update the Value

Wrong

```javascript
handleCheckbox() {

}
```

Correct

```javascript
handleCheckbox(event) {

    this.isChecked = event.target.checked;

}
```
