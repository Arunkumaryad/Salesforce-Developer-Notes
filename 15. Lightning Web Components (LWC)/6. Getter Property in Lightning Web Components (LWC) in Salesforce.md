# Getter Property in Lightning Web Components (LWC) in Salesforce

## Overview

Getter properties are a powerful feature in **Lightning Web Components (LWC)** that allow you to calculate values dynamically instead of storing them directly in variables.

A getter is a JavaScript method that behaves like a property. It performs some logic or calculations and returns the result whenever it is accessed from the HTML template.

This makes your code cleaner, reusable, and easier to maintain.

---

# What is a Getter Property?

### Definition (from the video)

> **Getter properties are used to perform some calculations and return values.**

In simple words,

A Getter Property is a special JavaScript method that calculates a value and returns it whenever the HTML template requests it.

Instead of storing calculated values in variables, we calculate them dynamically using a getter.

---

# Why Do We Need Getter Properties?

Suppose you have a student's age stored in JavaScript.

```javascript
Age = 45;
```

Now you want to display the student's rank based on age.

Rules:

- Age ≥ 50 → One
- Age ≥ 30 → Two
- Otherwise → Three

Instead of creating another variable like:

```javascript
rank = 'Two';
```

you can calculate it automatically using a getter.

This keeps the data consistent and avoids duplicate variables.

---

# Syntax of Getter Property

```javascript
get propertyName() {

    // logic

    return value;

}
```

Example

```javascript
get fullName() {

    return this.firstName + ' ' + this.lastName;

}
```

In HTML

```html
{fullName}
```

Notice that you **do not use parentheses** when accessing a getter.

---

# Example Used in the Video

## JavaScript (helloWorld.js)

```javascript
import { LightningElement } from 'lwc';

export default class HelloWorld extends LightningElement {

    student = {

        firstName: 'Arun',

        lastName: 'Kumar',

        Age: 45,

        City: 'Noida'

    };

    get studentRank() {

        const rank =
            this.student.Age >= 50
                ? 'One'
                : this.student.Age >= 30
                ? 'Two'
                : 'Three';

        return rank;

    }

}
```

---

## HTML (helloWorld.html)

```html
<template>

    <lightning-card>

        <h1>Hello {student.firstName} {student.lastName}</h1>

        <h2>Age : {student.Age}</h2>

        <h2>City : {student.City}</h2>

        <h2>Rank : {studentRank}</h2>

    </lightning-card>

</template>
```

---

## Output

```text
Hello Arun Kumar

Age : 45

City : Noida

Rank : Two
```

Since the student's age is **45**, the getter returns **Two**.

---

# Understanding the Getter Logic

The getter uses the **ternary operator**.

```javascript
this.student.Age >= 50
    ? 'One'
    : this.student.Age >= 30
    ? 'Two'
    : 'Three';
```

This is equivalent to:

```javascript
if (this.student.Age >= 50) {

    return 'One';

}
else if (this.student.Age >= 30) {

    return 'Two';

}
else {

    return 'Three';

}
```

# Accessing Getter in HTML

Correct

```html
{studentRank}
```

Wrong

```html
{studentRank()}
```

A getter behaves like a property, **not a method**.

---

# Getter vs Normal Method

## Normal Method

JavaScript

```javascript
calculateRank() {

    return 'Two';

}
```

HTML

```html
{calculateRank()}
```

❌ Not allowed in LWC templates.

---

## Getter

JavaScript

```javascript
get studentRank() {

    return 'Two';

}
```

HTML

```html
{studentRank}
```

✔ Correct.

# When Should You Use Getter Properties?

Use getters when:

- You need calculated values.
- You want to combine multiple properties.
- You want to format data before displaying it.
- You want to avoid creating unnecessary variables.
- You need reusable logic for the template.

---

# Advantages of Getter Properties

- Calculates values dynamically.
- Reduces duplicate code.
- Keeps JavaScript clean and readable.
- Makes components easier to maintain.
- Automatically updates when dependent data changes.
- Improves code reusability.

---

# Rules for Getter Properties

### Rule 1

Always use the `get` keyword.

Correct

```javascript
get fullName() {

}
```

---

### Rule 2

A getter must return a value.

Correct

```javascript
get fullName() {

    return 'Arun Kumar';

}
```

---

### Rule 3

Do not use parentheses in HTML.

Correct

```html
{fullName}
```

Wrong

```html
{fullName()}
```

---

### Rule 4

Use `this` to access component properties.

Correct

```javascript
return this.firstName;
```

Wrong

```javascript
return firstName;
```

---

# Common Mistakes

## Forgetting the `get` Keyword

Wrong

```javascript
studentRank() {

    return 'One';

}
```

Correct

```javascript
get studentRank() {

    return 'One';

}
```

---

## Forgetting to Return a Value

Wrong

```javascript
get studentRank() {

    const rank = 'One';

}
```

Correct

```javascript
get studentRank() {

    return 'One';

}
```

---

## Calling Getter Like a Function

Wrong

```html
{studentRank()}
```

Correct

```html
{studentRank}
```

# Real-World Example

Suppose an employee receives a bonus based on salary.

JavaScript

```javascript
employee = {

    Name: 'Rahul',

    Salary: 70000

};

get bonus() {

}
```

HTML

```html
<template>

    <p>Name : {employee.Name}</p>

    <p>Salary : {employee.Salary}</p>

    <p>Bonus : {bonus}</p>

</template>
```

Output

```text
Name : Rahul

Salary : 70000

Bonus : 7000
```

The bonus is calculated dynamically whenever it is displayed.
