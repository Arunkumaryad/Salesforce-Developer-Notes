# Data Communication From Parent to Child LWC Component | Message Passing

## Overview

In Lightning Web Components (LWC), **Parent to Child Communication** is one of the most commonly used communication methods. It allows a Parent Component to send data to its Child Component using **public properties** decorated with the **@api** decorator.

Whenever the value in the Parent Component changes, the Child Component automatically receives the updated value and re-renders.

---

# Definition

**Parent to Child Communication** is the process of passing data from a Parent Component to its Child Component.

According to the screenshot:

> **It is done by public properties using @api decorator.**

The Child Component exposes a **public property** using the **@api** decorator, and the Parent Component passes a value to that property through an HTML attribute.

---

# Communication Flow

```
Parent Component
        │
        │
        ▼
Child Component
```

The Parent sends data, and the Child receives it.

---

# How Parent to Child Communication Works

The communication happens in three simple steps:

1. Create a public property in the Child Component using **@api**.
2. Pass a value from the Parent Component using the Child Component tag.
3. The Child Component displays or uses the received value.

---

# Understanding @api

The **@api** decorator makes a property or method **public**.

A public property can be accessed from its Parent Component.

Example:

```javascript
@api fullName;
```

Here,

```
fullName
```

becomes a public property.

The Parent Component can now assign a value to it.

# Step 1: Child Component

The Child Component creates a public property using **@api**.

## childComponent.html

```html
<template>
    <lightning-card>
        Hello {fullName}!
    </lightning-card>
</template>
```

### Explanation

```
{fullName}
```

displays the value received from the Parent Component.

---

## childComponent.js

```javascript
import { LightningElement, api } from 'lwc';

export default class ChildComponent extends LightningElement {

    @api fullName = 'John Doe';

}
```

### Explanation

Import

```javascript
api
```

from

```javascript
lwc
```

The property

```javascript
fullName
```

is decorated using

```javascript
@api
```

which makes it public.

The default value is

```
John Doe
```

If the Parent passes another value, it replaces this default value.

---

## childComponent.js-meta.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>

<LightningComponentBundle xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>false</isExposed>

</LightningComponentBundle>
```

### Explanation

The Child Component is used only inside the Parent Component.

Therefore,

```xml
<isExposed>false</isExposed>
```

is enough because it doesn't need to appear in App Builder.

---

# Step 2: Parent Component

The Parent Component stores the value that will be sent to the Child Component.

---

## parentComponent.js

```javascript
import { LightningElement } from 'lwc';

export default class ParentComponent extends LightningElement {

    name = 'Madhur ji';

}
```

### Explanation

The Parent Component contains a variable named

```javascript
name
```

whose value is

```
Madhur ji
```

This value will be passed to the Child Component.

---

## parentComponent.html

```html
<template>

    <c-child-component full-name={name}></c-child-component>

</template>
```

### Explanation

The Parent uses the Child Component tag:

```html
<c-child-component>
```

and passes the value

```html
full-name={name}
```

Here,

```
name
```

is the Parent variable.

```
full-name
```

is the Child Component's public property.

---

## parentComponent.js-meta.xml

```xml
<?xml version="1.0" encoding="UTF-8"?>

<LightningComponentBundle xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>true</isExposed>

    <targets>
        <target>lightning__AppPage</target>
        <target>lightning__RecordPage</target>
        <target>lightning__HomePage</target>
    </targets>

</LightningComponentBundle>
```

### Explanation

The Parent Component is exposed so it can be added to:

- App Page
- Home Page
- Record Page

The Child Component is automatically loaded inside the Parent Component.

---

The Parent Component sends:

```
Madhur ji
```

The Child Component receives:

# Understanding Kebab Case

According to the screenshot:

> **Kebab Case:** Respective phrase will be transferred to all lowercase with hyphen (-) separating words.

Example:

```
headerLabel
```

becomes

```
header-label
```

---

## Why Kebab Case?

In JavaScript, property names use **camelCase**.

Example:

```javascript
@api fullName;
```

In HTML, LWC automatically converts camelCase into **kebab-case**.

So,

```javascript
fullName
```

becomes

```html
full-name
```

---

# Advantages

- Very simple to implement.
- Official Salesforce communication method.
- Automatic re-rendering.
- Clean and reusable components.
- Ideal for one-way data flow.
- Improves component modularity.

---

# Best Practices

- Use **@api** only when a property needs to be accessed by the Parent.
- Keep Child Components reusable.
- Pass only the required data.
- Use meaningful property names.
- Follow camelCase in JavaScript.
- Use kebab-case in HTML attributes.
- Avoid exposing unnecessary properties.

---

# Common Mistakes

## 1. Forgetting @api

❌ Incorrect

```javascript
fullName;
```

The Parent cannot access this property.

---

✅ Correct

```javascript
@api fullName;
```

---

## 2. Using camelCase in HTML

❌ Incorrect

```html
<c-child-component fullName={name}>
</c-child-component>
```

---

✅ Correct

```html
<c-child-component full-name={name}>
</c-child-component>
```

---

## 3. Using a Different Property Name

Child

```javascript
@api fullName;
```

Parent

```html
<c-child-component user-name={name}>
```

This will not work because the property names do not match.

---

## 4. Exposing Child Component Unnecessarily

If the Child Component is only used inside the Parent, keep

```xml
<isExposed>false</isExposed>
```

There is no need to expose it in App Builder.

# Interview Questions

### What is Parent to Child Communication in LWC?

It is the process of sending data from a Parent Component to a Child Component using public properties decorated with the **@api** decorator.

---

### Which decorator is used for Parent to Child Communication?

The **@api** decorator.

---

### Why do we use @api?

The **@api** decorator makes a property or method public so that it can be accessed by the Parent Component.

---

### What is Kebab Case?

Kebab Case converts a camelCase property name into lowercase words separated by hyphens.

Example:

```
fullName
```

becomes

```
full-name
```

---

### Why is kebab-case used?

HTML attributes in LWC use **kebab-case**, while JavaScript properties use **camelCase**.

---

### Can a Child Component change the Parent's data directly?

No.

The Child Component only receives data from the Parent. To send data back, it must use **Custom Events**.
