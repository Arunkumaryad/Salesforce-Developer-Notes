# Create First Lightning Web Component & Deploy to Salesforce Org

## Overview

Lightning Web Components (LWC) are Salesforce's modern programming model for building fast, reusable, and lightweight user interfaces. In this guide, you will learn how to create your first Lightning Web Component using Visual Studio Code and deploy it to your Salesforce Org.

---

# Topics Covered

This guide includes:

- Creating your first Lightning Web Component (LWC)
- Understanding the files created in an LWC bundle
- Deploying the component to Salesforce
- Making the component available as a Lightning Tab
- Displaying static text
- Displaying dynamic values
- Styling the component using CSS

---

# Prerequisites

Before creating your first Lightning Web Component, make sure you have completed the following setup:

- Install Visual Studio Code
- Install Salesforce CLI
- Install Salesforce Extension Pack
- Create a Salesforce Project
- Authorize your Salesforce Org

---

# Step 1: Create Your First Lightning Web Component

Open **Visual Studio Code**.

Open the **Command Palette** using:

```text
CTRL + SHIFT + P
```

Search for:

```text
SFDX: Create Lightning Web Component
```

Enter the component name.

Example:

```text
helloWorld
```

Select the default location:

```text
force-app/main/default/lwc
```

Your first Lightning Web Component is created successfully.

---

# Component Structure

After creating the component, VS Code generates the following folder structure:

```text
lwc
└── helloWorld
    ├── __tests__
    ├── helloWorld.html
    ├── helloWorld.js
    └── helloWorld.js-meta.xml
```

---

# Understanding Each File

## 1. helloWorld.html

Contains the HTML template of the component.

This file defines what users will see on the screen.

---

## 2. helloWorld.js

Contains the JavaScript logic.

This file manages:

- Variables
- Functions
- Event Handling
- Business Logic

---

## 3. helloWorld.js-meta.xml

Contains metadata for the component.

It defines:

- API Version
- Visibility
- Supported Targets

---

## 4. __tests__ Folder

Contains Jest test files used for unit testing Lightning Web Components.

---

# Step 2: Make the Component Available in Salesforce

Open:

```xml
helloWorld.js-meta.xml
```

Modify the file as follows:

```xml
<?xml version="1.0" encoding="UTF-8"?>
<LightningComponentBundle
    xmlns="http://soap.sforce.com/2006/04/metadata">

    <apiVersion>67.0</apiVersion>

    <isExposed>true</isExposed>

    <targets>

        <target>lightning__AppPage</target>

        <target>lightning__HomePage</target>

        <target>lightning__Tab</target>

    </targets>

</LightningComponentBundle>
```

---

# Explanation

## isExposed

```xml
<isExposed>true</isExposed>
```

Makes the component available inside Salesforce.

If set to:

```xml
false
```

the component cannot be used on Lightning Pages or Tabs.

---

## Targets

Targets specify where the component can be used.

### App Page

```xml
<target>lightning__AppPage</target>
```

Allows the component to be added to Lightning App Pages.

---

### Home Page

```xml
<target>lightning__HomePage</target>
```

Allows the component to be placed on the Salesforce Home Page.

---

### Lightning Tab

```xml
<target>lightning__Tab</target>
```

Allows the component to be used inside a custom Lightning Tab.

# Display Dynamic Values

Instead of displaying fixed text, LWC can display JavaScript variables.

---

## HTML

```html
<template>

    <lightning-card>

        <h1>

            Hello {firstName} {lastName}

        </h1>

    </lightning-card>

</template>
```

---

## JavaScript

```javascript
import { LightningElement } from 'lwc';

export default class HelloWorld extends LightningElement {

    firstName = 'Arun';

    lastName = 'Kumar';

}
```

---

## Output

```text
Hello Arun Kumar
```

---

# How Data Binding Works

The values inside curly braces:

```html
{firstName}

{lastName}
```

are automatically replaced with the values from the JavaScript class.

Example:

```javascript
firstName = 'Arun';
```

becomes

```text
Hello Arun
```

---

# Styling Using CSS

Create:

```text
helloWorld.css
```

Write:

```css
h1{

    color:blue;

}
```

No need to import CSS manually.

Salesforce automatically loads the CSS file if it has the same name as the component.

---

## Output

```text
Hello Arun Kumar
```

Displayed in blue color.

# Important Points

- Every Lightning Web Component is created inside the `lwc` folder.
- Every component consists of at least three required files:
  - HTML
  - JavaScript
  - Metadata XML
- Set `<isExposed>true</isExposed>` to make the component available in Salesforce.
- Add appropriate `<target>` values to specify where the component can be used.
- CSS files are automatically scoped to the component.
- Data binding is achieved using curly braces `{}`.
- Deploy changes using **SFDX: Deploy This Source to Org**.
