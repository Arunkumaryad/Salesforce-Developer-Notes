# Decorators in LWC Salesforce Introduction | @api, @track, @wire in LWC

## Overview

Decorators are one of the most important concepts in **Lightning Web Components (LWC)**. They provide additional functionality to JavaScript properties and methods, making components more powerful, reusable, and reactive.

In LWC, decorators are represented using the **@** symbol and are used to expose properties, make variables reactive, and retrieve Salesforce data.

---

# What are Decorators in LWC?

As explained in the video:

> **The Lightning Web Components programming model has three decorators that add functionality to a property or function.**

The three decorators available in LWC are:

- **@api**
- **@track**
- **@wire**

Each decorator serves a different purpose.

---

# Decorator Syntax

General syntax:

```javascript
@decorator propertyOrFunction;
```

Example:

```javascript
@api msg = 'Hello Parag';
```

Here,

- `@api` → Decorator
- `msg` → Property
- `"Hello Parag"` → Initial value

---

# Types of Decorators in LWC

| Decorator | Purpose |
|-----------|----------|
| @api | Makes a property or method public |
| @track | Makes private properties reactive |
| @wire | Retrieves Salesforce data reactively |

---

# 1. @api Decorator

## Definition

According to the video:

> **It is used to expose a variable or functions publicly and make properties reactive.**

The **@api** decorator allows a property or method to be accessed from another Lightning Web Component.

It is mainly used for **parent-to-child communication**.

---

## Example

Child Component

```javascript
import { LightningElement, api } from 'lwc';

export default class ChildComponent extends LightningElement {

    @api message = 'Hello from Child';

}
```

Parent HTML

```html
<c-child-component message="Hello Parent"></c-child-component>
```

Here,

- `message` is public.
- Parent component can pass values to it.

---

## @api on Methods

The `@api` decorator can also expose methods.

```javascript
import { LightningElement, api } from 'lwc';

export default class ChildComponent extends LightningElement {

    @api

    showMessage() {

        alert('Hello');

    }

}
```

The parent component can invoke this method.

---

## Use Cases

Use `@api` when:

- Passing data from Parent to Child.
- Exposing methods to another component.
- Creating reusable components.

---

# 2. @track Decorator

## Definition

According to the video:

> **It is used to make variable private but reactive. Tracked properties are also called private reactive properties.**

The `@track` decorator makes a private property reactive.

When the value changes, the component automatically rerenders.

---

## Note (From the Video)

> **We can't access @track properties from outside as they are private and only accessible within its component only.**

---

## Example

```javascript
import { LightningElement, track } from 'lwc';

export default class DemoComponent extends LightningElement {

    @track

    student = {

        firstName: 'Arun',

        city: 'Noida'

    };

}
```

If the `student` object changes, the UI updates automatically.

---

## Why was @track Needed?

Earlier, `@track` was required to observe changes inside objects and arrays.

For example,

```javascript
student.city = 'Delhi';
```

Without tracking, the UI would not update in older versions.

---

## Current LWC Behavior

In modern versions of LWC:

- Primitive values (`String`, `Number`, `Boolean`) are reactive by default.
- Objects and arrays are also reactive when reassigned.

Therefore, `@track` is required much less often than before.

Example

```javascript
this.student = {

    ...this.student,

    city: 'Delhi'

};
```

---

## Use Cases

Use `@track` when:

- Working with complex object mutations in older code.
- Maintaining legacy LWC components.
- Explicitly indicating reactive state.

---

# 3. @wire Decorator

## Definition

According to the video:

> **To read Salesforce data, Lightning Web Components use a reactive wire service. When the wire service provisions data, the component rerenders. Components use @wire in their JavaScript class to specify a wire adaptor or an Apex method.**

The `@wire` decorator connects a component to Salesforce data.

Whenever the underlying data changes, the component updates automatically.

---

# @wire Syntax

According to the video:

```javascript
import { adapterId } from 'adapterModule';

@wire(adapterId, adapterConfig)

propertyOrFunction;
```

---

# Understanding the Syntax

## adapterId

According to the video:

> **The identifier of the wire adapter.**

Example

```javascript
getRecord
```

---

## adapterModule

According to the video:

> **The identifier of the module that contains the wire adapter function.**

Example

```javascript
lightning/uiRecordApi
```

---

## adapterConfig

Configuration object.

Example

```javascript
{

    recordId: '$recordId',

    fields: FIELDS

}
```

---

## propertyOrFunction

The destination where the returned data is stored.

It can be

- Property

or

- Function

---

# Example Using a Property

```javascript
import { LightningElement, wire } from 'lwc';

import { getRecord } from 'lightning/uiRecordApi';

export default class AccountDemo extends LightningElement {

    @wire(getRecord, {

        recordId: '$recordId',

        fields: ['Account.Name']

    })

    account;

}
```

The returned record is stored inside

```javascript
account
```

---

# Example Using a Function

```javascript
import { LightningElement, wire } from 'lwc';

import getAccounts from '@salesforce/apex/AccountController.getAccounts';

export default class Demo extends LightningElement {

    @wire(getAccounts)

    wiredAccounts({ data, error }) {

        if (data) {

            console.log(data);

        }

        else if (error) {

            console.error(error);

        }

    }

}
```

---

# Property vs Function

Property

```javascript
@wire(getAccounts)

accounts;
```

Stores the returned data automatically.

---

Function

```javascript
@wire(getAccounts)

wiredAccounts({data,error})
```

Provides greater control over the response.

---

# When Should You Use @wire?

Use `@wire` when:

- Reading Salesforce records.
- Calling cacheable Apex methods.
- Automatically refreshing data.
- Using Lightning Data Service.

---

# Comparison of Decorators

| Feature | @api | @track | @wire |
|----------|------|---------|--------|
| Public Property | ✅ | ❌ | ❌ |
| Private Property | ❌ | ✅ | ❌ |
| Reactive | ✅ | ✅ | ✅ |
| Parent-to-Child Communication | ✅ | ❌ | ❌ |
| Fetch Salesforce Data | ❌ | ❌ | ✅ |
| Calls Apex | ❌ | ❌ | ✅ |
| Calls LDS | ❌ | ❌ | ✅ |

# Advantages

## @api

- Makes components reusable.
- Enables parent-child communication.
- Exposes public methods.

---

## @track

- Keeps UI synchronized.
- Makes properties reactive.
- Useful for complex objects.

---

## @wire

- Automatically fetches Salesforce data.
- Reactive data updates.
- Less JavaScript code.
- Built-in caching support.
- Easy integration with Apex and Lightning Data Service.

---

# Common Mistakes

## Forgetting to Import the Decorator

Wrong

```javascript
@api

message;
```

Correct

```javascript
import { api } from 'lwc';
```

---

## Using @wire with a Non-cacheable Apex Method

Wrong

```apex
@AuraEnabled

public static List<Account> getAccounts()
```

Correct

```apex
@AuraEnabled(cacheable=true)

public static List<Account> getAccounts()
```

---

## Accessing @track from Another Component

Wrong

```javascript
child.student;
```

`@track` properties are private.

# Interview Questions

### What are decorators in LWC?

Decorators are special annotations that add additional functionality to JavaScript properties or methods.

---

### How many decorators are available in LWC?

There are three decorators:

- `@api`
- `@track`
- `@wire`

---

### What is the purpose of `@api`?

It exposes a property or method publicly so it can be accessed by another Lightning Web Component.

---

### What is the purpose of `@track`?

It makes private properties reactive so that UI updates when the property changes.

---

### What is the purpose of `@wire`?

It retrieves Salesforce data reactively using Lightning Data Service or Apex methods.

---

### Can @wire call Apex?

Yes.

It can call Apex methods annotated with

```apex
@AuraEnabled(cacheable=true)
```

---

### Which decorator is used for Parent-to-Child communication?

```javascript
@api
```

---

### Which decorator is used for Salesforce data?

```javascript
@wire
```

---

### Which decorator makes properties reactive?

```javascript
@track
```

(Although modern LWC automatically handles many reactive scenarios.)

---

# Best Practices

- Use **@api** only for properties or methods that must be accessed from another component.
- Avoid using **@track** unless it is required for your use case or maintaining legacy code.
- Use **@wire** for reading Salesforce data reactively.
- Use imperative Apex calls instead of `@wire` when you need to perform DML operations such as insert, update, or delete.
