# Lightning Message Service (LMS) for Passing Multiple Values Between Components

## Overview

In Lightning Web Components (LWC), sometimes we need to send **more than one value** between two unrelated components. Since these components do not have a Parent-Child relationship, we cannot use `@api` or `CustomEvent`.

Salesforce provides **Lightning Message Service (LMS)** to exchange multiple values between unrelated components by sending an object (payload) through a **Message Channel**.

In this example, the Publisher Component sends both **First Name** and **Last Name**, and the Subscriber Component receives and displays both values.

---

# What is Lightning Message Service (LMS)?

**Lightning Message Service (LMS)** is a Salesforce feature that enables communication between unrelated Lightning Web Components (LWC), Aura Components, and Visualforce Pages.

Instead of passing a single value, LMS can also pass **multiple values** by using a JavaScript object called a **payload**.

---

# Definition

**Lightning Message Service (LMS)** allows unrelated components to communicate with each other by publishing and subscribing to messages through a Message Channel.

When multiple values need to be transferred, they are sent together inside the payload object.

## publisherComponent.js

```javascript
import { LightningElement, wire } from 'lwc';

import {
    MessageContext,
    publish
} from 'lightning/messageService';

import ComrevoChannel
from '@salesforce/messageChannel/ComrevoChannel__c';

export default class PublisherComponent extends LightningElement {

    firstName = '';

    lastName = '';

    @wire(MessageContext)
    messageContext;

    handleChangeFirstName(event) {

        this.firstName = event.target.value;

    }

    handleChangeLastName(event) {

        this.lastName = event.target.value;

    }

    handleClick() {

        let payload = {

            FirstName: this.firstName,

            LastName: this.lastName

        };

        publish(
            this.messageContext,
            ComrevoChannel,
            payload
        );

    }

}
```

## subscriberComponent.js

```javascript
import { LightningElement, wire } from 'lwc';

import {

MessageContext,

subscribe,

unsubscribe

}

from 'lightning/messageService';

import ComrevoChannel

from '@salesforce/messageChannel/ComrevoChannel__c';

export default class SubscriberComponent extends LightningElement {

    firstName = '';

    lastName = '';

    subscription = null;

    @wire(MessageContext)

    messageContext;

    connectedCallback() {

        this.handleSubscribe();

    }

    disconnectedCallback() {

        this.handleUnSubscribe();

    }

    handleSubscribe() {

        if (!this.subscription) {

            this.subscription = subscribe(

                this.messageContext,

                ComrevoChannel,

                (parameter) => {

                    this.firstName = parameter.FirstName;

                    this.lastName = parameter.LastName;

                }

            );

        }

    }

    handleUnSubscribe() {

        unsubscribe(this.subscription);

        this.subscription = null;

    }

}
```

---

## Subscribe

```javascript

ComrevoChannel,

}
)
```

Listens for messages published on the Message Channel.

## Receive Last Name

Receives the Last Name from the payload.

# Advantages

- Pass multiple values in a single message.
- Works between unrelated components.
- Supports LWC, Aura, and Visualforce.
- Reduces component dependency.
- Easy to maintain.
- Supports scalable applications.

---

# Best Practices

- Use descriptive Message Channel names.
- Group related data into a single payload object.
- Always unsubscribe in `disconnectedCallback()`.
- Send only the required fields.
- Keep payload objects lightweight.

---

# Common Mistakes

## Incorrect Field Names

❌ Publisher

```javascript
{

firstName : this.firstName

}
```

Subscriber

```javascript
parameter.FirstName
```

The names do not match.

---

✅ Correct

Publisher

```javascript
{

FirstName : this.firstName

}
```

Subscriber

```javascript
parameter.FirstName
```

---

## Forgetting unsubscribe()

Always unsubscribe.

```javascript
unsubscribe(this.subscription);
```

---

## Forgetting MessageContext

Always use

```javascript
@wire(MessageContext)

messageContext;
```

before calling `publish()` or `subscribe()`.

# Interview Questions

### What is Lightning Message Service (LMS)?

Lightning Message Service is a Salesforce feature used to communicate between unrelated Lightning Web Components, Aura Components, and Visualforce Pages using a Message Channel.

---

### Can LMS pass multiple values?

Yes. Multiple values are sent together inside a JavaScript object called a **payload**.

Example:

```javascript
{

FirstName: "Arun",

LastName: "Kumar"

}
```

---

### What is a payload?

A payload is a JavaScript object that contains the data being transferred from the Publisher to the Subscriber.

---

### What is the role of a Message Channel?

A Message Channel acts as the communication bridge between the Publisher and Subscriber components.

---

### Why is `unsubscribe()` important?

It stops listening for messages when the component is removed, preventing memory leaks and unnecessary processing.
