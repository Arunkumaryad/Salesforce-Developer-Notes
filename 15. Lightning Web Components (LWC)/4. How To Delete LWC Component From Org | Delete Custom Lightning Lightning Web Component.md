# How To Delete LWC Component From Org | Delete Custom Lightning Web Component

## Overview

During Salesforce development, you may create Lightning Web Components (LWCs) that are no longer needed. Instead of leaving unused components in your Salesforce Org, you can permanently delete them.

One of the easiest ways to delete a Lightning Web Component directly from a Salesforce Org is by using the **Developer Console** along with the **Tooling API**.

In this guide, you'll learn how to:

- Open the Developer Console
- Find all Lightning Web Components in your Org
- Query a specific component
- Delete a Lightning Web Component
- Understand the `LightningComponentBundle` object

---

# What is LightningComponentBundle?

`LightningComponentBundle` is a Salesforce Tooling API object that stores metadata information about every Lightning Web Component (LWC) in your Salesforce Org.

Each LWC you create is stored as a record in this object.

Using the Tooling API, you can:

- View all Lightning Web Components
- Search for a component
- Delete a component
- Retrieve component information

---

# Prerequisites

Before deleting an LWC, make sure:

- You have access to the Salesforce Org.
- You have the required permissions.
- The component is not actively being used in Lightning Pages, Apps, or Tabs.
- You have access to the **Developer Console**.

---

# Step 1: Open Developer Console

Log in to your Salesforce Org.

Click the **⚙️ Gear Icon** in the top-right corner.

Select:

```text
Developer Console
```

The Developer Console window will open.

---

# Step 2: Open Query Editor

Inside the Developer Console:

Navigate to:

```text
Query Editor
```

The Query Editor allows you to execute SOQL queries and perform operations using the Tooling API.

---

# Step 3: Write the Query

Enter the following query:

```sql
SELECT Id, DeveloperName
FROM LightningComponentBundle
```

This query retrieves:

- Component Id
- Component Developer Name

---

# Understanding the Query

```sql
SELECT Id, DeveloperName
FROM LightningComponentBundle
```

### Id

Returns the unique Salesforce ID of the Lightning Web Component.

Example:

```text
0RbgL000003sXwXSAU
```

---

### DeveloperName

Returns the API Name of the Lightning Web Component.

Example:

```text
helloWorld
```

---

### LightningComponentBundle

This Tooling API object stores metadata for all Lightning Web Components in your Salesforce Org.

---

# Step 4: Enable Tooling API

Before executing the query, enable the following checkbox:

```text
✔ Use Tooling API
```

**Why is this required?**

`LightningComponentBundle` is a **Tooling API object**, not a standard Salesforce object.

Without enabling **Use Tooling API**, the query will fail because Salesforce won't recognize the object.

---

# Step 5: Execute the Query

Click:

```text
Execute
```

Salesforce returns all Lightning Web Components.

Example:

| Id | DeveloperName |
|----|---------------|
| 0RbgL000003sXwXSAU | helloWorld |
| 0RbgL0000011x3tSAA | demoLWC |
| 0RbgL0000011xWvSAI | firstLWC |

---

# Step 6: Find the Component You Want to Delete

If you know the component ID, run:

```sql
SELECT Id, DeveloperName
FROM LightningComponentBundle
WHERE Id = '0RbgL000003sXwXSAU'
```

Example Result

| Id | DeveloperName |
|----|---------------|
| 0RbgL000003sXwXSAU | helloWorld |

This confirms that the correct component has been selected.

---

# Step 7: Delete the Component

Select the row returned by the query.

Click:

```text
Delete Row
```

Salesforce displays a confirmation dialog.

Click:

```text
Yes
```

The Lightning Web Component is permanently deleted from your Salesforce Org.

---

# What Happens After Deletion?

After deleting the component:

- The component is removed from the Salesforce Org.
- It will no longer appear in Lightning App Builder.
- It cannot be added to Lightning Pages.
- Associated metadata is deleted from the Org.

If the component still exists in your local Visual Studio Code project, you must also delete it locally or retrieve the latest metadata from the Org to keep both environments in sync.

# Important Notes

- Always enable **Use Tooling API** before querying `LightningComponentBundle`.
- Verify the `DeveloperName` before deleting to avoid removing the wrong component.
- Deletion is **permanent** in the Salesforce Org.
- If the component is used in Lightning Pages or Tabs, remove those references before deletion.
- If the component exists in your local VS Code project, delete it locally as well or retrieve metadata again to synchronize.

# Common Errors

## Query Doesn't Work

**Reason**

The **Use Tooling API** checkbox is not selected.

**Solution**

Enable:

```text
✔ Use Tooling API
```

and execute the query again.

---

## Component Not Found

**Reason**

The component does not exist in the Org, or the `DeveloperName`/`Id` is incorrect.

**Solution**

Run the query to list all components and verify the correct name or ID.

---

## Unable to Delete

**Reason**

You may not have sufficient permissions, or the component is still referenced in the Org.

**Solution**

Ensure you have the required permissions and remove any dependencies before attempting deletion.

### Can Lightning Web Components be deleted using SOQL?

No. SOQL is used only to retrieve the records. The actual deletion is performed using the **Delete Row** option in the Developer Console (with the Tooling API).

---

### Is deleting an LWC permanent?

Yes. Once confirmed, the component is permanently removed from the Salesforce Org.
