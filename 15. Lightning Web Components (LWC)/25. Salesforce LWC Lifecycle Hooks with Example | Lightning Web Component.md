# Salesforce LWC Lifecycle Hooks with Example | Lightning Web Component

## Overview

Every Lightning Web Component (LWC) goes through different stages during its lifetime, such as creation, rendering, updating, and removal. Salesforce provides **Lifecycle Hooks** that allow developers to execute custom code at these different stages.

Lifecycle hooks are useful for performing initialization, fetching data, updating the UI, cleaning up resources, and handling runtime errors.

---

# What is a Lifecycle Hook?

**Definition (as shown in the screenshot):**

> **A lifecycle hook is a callback method triggered at a specific phase of a component instance's lifecycle.**

In simple words, a lifecycle hook is a predefined JavaScript method that Salesforce automatically calls when a component reaches a particular stage of its life.

---

# LWC Lifecycle Hooks

LWC provides the following lifecycle hooks:

1. `constructor()`
2. `connectedCallback()`
3. `renderedCallback()`
4. `render()`
5. `disconnectedCallback()`
6. `errorCallback(error, stack)`

---

# Lifecycle Flow

```
Component Created
        │
        ▼
constructor()
        │
        ▼
connectedCallback()
        │
        ▼
render()
        │
        ▼
renderedCallback()
        │
        │
 (Component Updates)
        │
        ▼
render()
        │
        ▼
renderedCallback()
        │
        ▼
disconnectedCallback()

If any child component throws an error

        │
        ▼
errorCallback(error, stack)
```

# Lifecycle Hooks Explanation

## 1. constructor()

### Definition

`constructor()` is called when the component instance is created.

It is the first lifecycle hook that executes.

### When is it called?

- When the component object is created.
- Before the component is inserted into the DOM.

### Description (Screenshot)

> Called when the component is created.

---

### Common Uses

- Initialize variables
- Set default values
- Call `super()`
- Basic initialization

---

### Example

```javascript
constructor() {
    super();

    console.log('Component Created');
}
```

---

### Important Notes

- Always call `super()` first.
- Do not access template elements here.
- Do not use `this.template`.

---

# 2. connectedCallback()

## Definition

`connectedCallback()` executes when the component is inserted into the DOM (Document Object Model).

### When is it called?

After the constructor and before rendering.

### Description (Screenshot)

> Called when the element is inserted into a document.

---

### Common Uses

- Call Apex methods
- Fetch records
- Initialize LMS
- Register event listeners
- Start timers

---

### Example

```javascript
connectedCallback() {

    console.log('Component Connected');

}
```

---

### Execution Order

```
constructor()

↓

connectedCallback()

↓

render()
```

---

# 3. renderedCallback()

## Definition

`renderedCallback()` executes after the component has been completely rendered on the UI.

### Description (Screenshot)

> Called when a component is completely rendered on UI.

> This hook is called after the component gets rendered every time. Hence, it should be called cautiously.

---

### Important

Unlike `connectedCallback()`, this method executes:

- Initial rendering
- Every re-render
- Every UI update

---

### Common Uses

- DOM manipulation
- Third-party library initialization
- Chart initialization
- Element measurement

---

### Example

```javascript
renderedCallback() {

    console.log('Component Rendered');

}
```

---

### Warning

Avoid changing tracked properties directly inside `renderedCallback()` because it can cause an infinite rendering loop.

❌ Incorrect

```javascript
renderedCallback() {

    this.name = 'Arun';

}
```

---

# 4. render()

## Definition

`render()` is used to override the standard rendering behavior of a component.

### Description (Screenshot)

> This hook is used to override the standard rendering functionality in Lightning Web Components & to update the UI.

> This hook is not technically a lifecycle hook.

> It is a protected method on the LightningElement class.

---

### When to use?

Rarely.

Mostly when you want to render different templates dynamically.

---

### Example

```javascript
render() {

    return templateOne;

}
```

---

### Example Use Case

```javascript
if(isAdmin)

↓

Admin Template

Else

↓

User Template
```

---

# 5. disconnectedCallback()

## Definition

`disconnectedCallback()` executes when the component is removed from the DOM.

### Description (Screenshot)

> Called when the element is removed from a document or DOM.

---

### Common Uses

- Stop timers
- Remove event listeners
- Unsubscribe LMS
- Release resources

---

### Example

```javascript
disconnectedCallback() {

    console.log('Component Removed');

}
```

---

### Example

```javascript
window.removeEventListener(...);

clearInterval(timer);

unsubscribe(subscription);
```

---

# 6. errorCallback(error, stack)

## Definition

`errorCallback(error, stack)` is called when one of the child components throws a JavaScript error during its lifecycle.

### Description (Screenshot)

> Called when the component throws error in one of the lifecycle hooks.

> error and stack are the two arguments.

> error is JavaScript native error object whereas stack is a string which gives error information.

---

### Parameters

### error

JavaScript Error Object

Example

```javascript
TypeError

ReferenceError

SyntaxError
```

---

### stack

Contains stack trace.

Example

```
at ChildComponent.js

Line 25

Column 12
```

---

### Example

```javascript
errorCallback(error, stack) {

    console.log(error);

    console.log(stack);

}
```

# Parent Component Example

## lifecycleParent.html

```html
<template>

    <lightning-card>

        <c-lifecycle-child>

        </c-lifecycle-child>

        <template if:true={error}>

            <b>Error Occured</b>

        </template>

        <template if:false={error}>

            <b>Error did not Occured</b>

        </template>

    </lightning-card>

</template>
```

---

### Explanation

The Parent Component contains:

- Child Component
- Error message if an error occurs
- Success message if no error occurs

---

## lifecycleParent.js

```javascript
import { LightningElement } from 'lwc';

export default class LifecycleParent extends LightningElement {

    error;
    stack;

    constructor() {

        super();

        console.log('In Parent Constructor');

    }

    connectedCallback() {

        console.log('In Parent connectedCallback method');

    }

    renderedCallback() {

        console.log('In Parent renderedCallback method');

    }

    disconnectedCallback() {

        console.log('In Parent disconnectedCallback method');

    }

    errorCallback(error, stack) {

        console.log('In Parent errorCallback method');

        this.error = error;

        this.stack = stack;

    }

}
```

---

### Explanation

This component demonstrates all major lifecycle hooks.

Whenever a child throws an error, `errorCallback()` is automatically invoked.

# Lifecycle Hook Order

For Parent and Child Components, the execution order is:

```
Parent Constructor

↓

Child Constructor

↓

Parent connectedCallback()

↓

Child connectedCallback()

↓

Parent render()

↓

Child render()

↓

Child renderedCallback()

↓

Parent renderedCallback()
```

---

# When Components are Removed

```
Parent removes Child

↓

Child disconnectedCallback()

↓

Parent disconnectedCallback()
```

---

# Lifecycle Hook Summary

| Lifecycle Hook | When it is Called | Common Use |
|----------------|-------------------|------------|
| constructor() | Component instance is created | Initialize variables |
| connectedCallback() | Component inserted into DOM | Load data, call Apex, register events |
| render() | Before rendering | Override rendering logic |
| renderedCallback() | After UI rendering | DOM manipulation, initialize libraries |
| disconnectedCallback() | Component removed from DOM | Cleanup, unsubscribe, remove listeners |
| errorCallback(error, stack) | Child throws an error | Error handling and logging |

# Best Practices

- Always call `super()` inside `constructor()`.
- Use `connectedCallback()` for initialization and data loading.
- Avoid updating reactive properties inside `renderedCallback()` to prevent infinite loops.
- Always clean up resources in `disconnectedCallback()`.
- Use `errorCallback()` to gracefully handle errors from child components.
- Use `render()` only when you need custom rendering logic.

---

# Common Mistakes

### ❌ Forgetting `super()`

```javascript
constructor() {

}
```

✔ Correct

```javascript
constructor() {

    super();

}
```

---

### ❌ Updating properties in `renderedCallback()`

```javascript
renderedCallback() {

    this.count++;

}
```

This can cause an infinite render loop.

---

### ❌ Not unsubscribing from LMS

Always unsubscribe inside:

```javascript
disconnectedCallback()
```

---

### ❌ Accessing DOM in `constructor()`

```javascript
this.template.querySelector()
```

This is not allowed because the template has not been rendered yet.

# Interview Questions

### What is a Lifecycle Hook in LWC?

A lifecycle hook is a callback method that Salesforce automatically invokes at different stages of a component's lifecycle, such as creation, rendering, updating, and destruction.

---

### Which lifecycle hook is called first?

`constructor()`

---

### Which hook is used to call Apex?

`connectedCallback()`

---

### Which hook runs after the component is rendered?

`renderedCallback()`

---

### Which hook is used to clean up resources?

`disconnectedCallback()`

---

### Which hook handles child component errors?

`errorCallback(error, stack)`

---

### Is `render()` a lifecycle hook?

No. Although it participates in the rendering process, it is a protected method of `LightningElement` and is not technically considered a lifecycle hook.
