# Use querySelector() to Fetch Data in LWC | HTML Template Data Binding

## Overview

In Lightning Web Components (LWC), the `querySelector()` method is used to access an HTML element inside the component's template using a CSS selector. It is useful when you want to read values from input fields, modify elements, set focus, or perform DOM operations.

In this tutorial, you'll learn:

- What is `querySelector()`?
- CSS selectors supported by `querySelector()`.
- Fetch data using Tag Name.
- Fetch data using Class Name.
- Fetch data using Tag + Class Name.
- Complete code examples.
- Best practices.
- Common mistakes.
- Interview questions.

---

# What is querySelector()?

According to the video:

> **The querySelector() method returns the first element that matches a CSS selector.**

In LWC, `querySelector()` is called on the component template.

```javascript
this.template.querySelector(selector);
```

It searches only inside the current component's HTML template.

---

# Syntax

```javascript
this.template.querySelector('CSS Selector');
```

Returns:

- First matching element
- `null` if no matching element is found

---

# CSS Selectors Supported

According to the video, `querySelector()` supports the following CSS selectors.

## 1. By Tag

```javascript
querySelector('tagname')
```

Example

```javascript
this.template.querySelector('lightning-input');
```

Returns the first `<lightning-input>` element.

---

## 2. By Class Name

```javascript
querySelector('.classname')
```

Example

```javascript
this.template.querySelector('.inputSection');
```

Returns the first element having the class `inputSection`.

---

## 3. By Tag and Class Name

```javascript
querySelector('tagname.classname')
```

Example

```javascript
this.template.querySelector('lightning-input.inputSection');
```

Returns the first `lightning-input` having the class `inputSection`.

# Example 1: Using querySelector() by Tag Name

This example reads the value entered in a `lightning-input` and displays it on the page.

---

## HTML File

### querySelectorExamples.html

```html
<template>

    <lightning-card>

        <div>
            Name : {userName}
        </div>

        <div>

            <lightning-input
                label="Enter Your Name"
                value="Test">
            </lightning-input>

        </div>

        <div>

            <lightning-button
                label="Click Me"
                onclick={handleUserName}>
            </lightning-button>

        </div>

    </lightning-card>

</template>
```

---

## JavaScript File

### querySelectorExamples.js

```javascript
import { LightningElement } from 'lwc';

export default class QuerySelectorExamples extends LightningElement {

    userName = 'Arun';

    handleUserName() {

        this.userName =
            this.template.querySelector('lightning-input').value;

    }

}
```

# Example 2: Using querySelector() by Class Name

Sometimes multiple input fields exist.

Instead of selecting by tag, assign a CSS class.

---

## HTML

```html
<template>

    <lightning-card>

        <div>

            Name : {userName}

        </div>

        <div>

            <lightning-input
                label="Enter Your Name"
                value="Test"
                class="inputSection">
            </lightning-input>

        </div>

        <div>

            <lightning-button
                label="Click Me"
                onclick={handleUserName}>
            </lightning-button>

        </div>

    </lightning-card>

</template>
```

---

## JavaScript

```javascript
import { LightningElement } from 'lwc';

export default class QuerySelectorExamples extends LightningElement {

    userName = 'Arun';

    handleUserName() {

        this.userName =
            this.template.querySelector('.inputSection').value;

    }

}
```

Here,

```javascript
'.inputSection'
```

selects the first element having the class `inputSection`.

---

# Example 3: Using Tag Name + Class Name

You can also combine both selectors.

```javascript
this.template.querySelector(
    'lightning-input.inputSection'
).value;
```

This means:

- Find a `lightning-input`
- That has the class `inputSection`

This is more specific than using only the class name.

---

# CSS Example

```css
.inputSection{

    color:aqua;

}
```

Although this example only changes the input styling, the class can also be used with `querySelector()`.

# Why Use querySelector()?

It is commonly used to:

- Read input values.
- Focus an input field.
- Reset fields.
- Access child HTML elements.
- Modify DOM properties.
- Call public methods on child components (using component selectors).

---

# querySelector() vs querySelectorAll()

| querySelector() | querySelectorAll() |
|-----------------|--------------------|
| Returns first matching element | Returns all matching elements |
| Returns a single element | Returns a NodeList |
| Most commonly used | Used for multiple elements |

Example

```javascript
this.template.querySelector('lightning-input');
```

Example

```javascript
this.template.querySelectorAll('lightning-input');
```

# Common Mistakes

## Forgetting `this.template`

Wrong

```javascript
querySelector('lightning-input');
```

Correct

```javascript
this.template.querySelector('lightning-input');
```

---

## Missing Dot for Class Selector

Wrong

```javascript
querySelector('inputSection');
```

Correct

```javascript
querySelector('.inputSection');
```

---

## Accessing a Non-Existing Element

Wrong

```javascript
this.template.querySelector('.demo').value;
```

If no element exists with class `demo`, `querySelector()` returns `null`, which causes an error when accessing `.value`.

A safer approach is:

```javascript
const input = this.template.querySelector('.demo');

if (input) {
    this.userName = input.value;
}
```

---

## Expecting Multiple Elements

Remember,

```javascript
querySelector()
```

returns only the **first** matching element.

If multiple elements need to be selected, use:

```javascript
querySelectorAll()
```

---

# Best Practices

- Use `this.template.querySelector()` only for elements inside the current component.
- Prefer `event.target.value` inside input event handlers when possible, as it is simpler and avoids an extra DOM lookup.
- Use meaningful class names.
- Check whether the returned element is `null` before accessing its properties.
- Use `querySelectorAll()` when working with multiple matching elements.

# Interview Questions

### What is querySelector() in LWC?

`querySelector()` is a DOM method that returns the first element inside the component template that matches a specified CSS selector.

---

### What does querySelector() return?

It returns:

- The first matching element.
- `null` if no element matches the selector.

---

### Can querySelector() access elements outside the component?

No.

It only searches within the current component's template using:

```javascript
this.template.querySelector()
```

---

### How do you select an element by class?

```javascript
this.template.querySelector('.inputSection');
```

---

### How do you select an element by tag?

```javascript
this.template.querySelector('lightning-input');
```

---

### How do you select an element by tag and class together?

```javascript
this.template.querySelector(
    'lightning-input.inputSection'
);
```

---

### What is the difference between querySelector() and querySelectorAll()?

- `querySelector()` returns the first matching element.
- `querySelectorAll()` returns all matching elements as a NodeList.
