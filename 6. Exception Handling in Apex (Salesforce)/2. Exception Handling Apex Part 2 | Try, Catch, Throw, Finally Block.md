# Exception Handling Apex Part 2 | Try, Catch, Throw, Finally Block

# Introduction

In the previous part, we learned about different types of exceptions such as:

- MathException
- DmlException
- NullPointerException

An unhandled exception immediately terminates the execution of the program.

To prevent this behavior, Apex provides the following exception handling keywords:

- try
- catch
- throw
- finally

These keywords help us gracefully handle runtime errors and continue program execution.

---

# What Happens Without Exception Handling?

Consider the following code:

```apex
System.debug('Before Exception');

Double result = 10 / 0;

System.debug('After Exception');
```

---

## Output

```text
System.MathException: Divide by 0
```

---

## Explanation

Execution Flow:

1. First debug statement executes.
2. Apex attempts to divide 10 by 0.
3. A MathException occurs.
4. Program execution immediately stops.
5. The second debug statement never executes.

Because the exception is not handled, Salesforce terminates the transaction.

---

# What is a Try Block?

## Definition

A **try block** contains the code that may generate an exception.

Syntax:

```apex
try {
    // Risky code
}
```

Any statement that might fail should be placed inside the try block.

---

# What is a Catch Block?

## Definition

A **catch block** is used to handle an exception generated inside the try block.

Syntax:

```apex
try {

}
catch(Exception e) {

}
```

The catch block receives the exception object and allows us to process the error without stopping the program.

---

# Example 1: Handling MathException

```apex
System.debug('Before Exception');

try {

    Double result = 10 / 0;

}
catch(MathException e) {

    System.debug(e);

}

System.debug('After Exception');
```

---

## Output

```text
Before Exception

System.MathException: Divide by 0

After Exception
```

---

## Explanation

Execution Flow:

1. "Before Exception" is printed.
2. Division by zero occurs.
3. Apex throws a MathException.
4. Catch block handles the exception.
5. Error message is printed.
6. Program continues executing.
7. "After Exception" is printed.

Without try-catch, execution would stop immediately.

---

# Multiple Catch Blocks

Apex allows multiple catch blocks for handling different exception types.

Example:

```apex
System.debug('Before Exception');

try {

    Double result = 10 / 0;

}
catch(NullPointerException e) {

    System.debug(e);

}
catch(MathException e) {

    System.debug(e);

}
catch(DMLException e) {

    System.debug(e);

}

System.debug('After Exception');
```

---

## Output

```text
Before Exception

System.MathException: Divide by 0

After Exception
```

---

## Explanation

Although there are multiple catch blocks:

```apex
catch(NullPointerException e)
catch(MathException e)
catch(DMLException e)
```

Only the matching exception block executes.

Since division by zero generates:

```text
MathException
```

Only this block executes:

```apex
catch(MathException e)
```

The remaining catch blocks are skipped.

---

# Generic Exception Catch Block

Sometimes we do not know which exception might occur.

In such cases we can catch the parent Exception class.

Example:

```apex
try {

    Double result = 10 / 0;

}
catch(Exception e) {

    System.debug(e);

}
```

---

## Why Use Exception?

All Apex exception classes inherit from:

```apex
Exception
```

Therefore:

```apex
catch(Exception e)
```

can handle:

- MathException
- DmlException
- NullPointerException
- QueryException
- StringException
- Any other Apex exception

---

# Important Rule

Specific exceptions should always be written before the generic Exception block.

Correct:

```apex
catch(MathException e)

catch(DmlException e)

catch(Exception e)
```

Incorrect:

```apex
catch(Exception e)

catch(MathException e)
```

The generic catch block would already catch everything.

---

# What is a Finally Block?

## Definition

A **finally block** contains code that always executes, whether an exception occurs or not.

Syntax:

```apex
try {

}
catch(Exception e) {

}
finally {

}
```

---

# Example: Finally Block with Exception

```apex
System.debug('Before Exception');

try {

    Double result = 10 / 0;

}
catch(NullPointerException e) {

    System.debug(e);

}
catch(MathException e) {

    System.debug(e);

}
catch(DMLException e) {

    System.debug(e);

}
catch(Exception e) {

    System.debug(e);

}
finally {

    System.debug('In Finally Block');

}

System.debug('After Exception');
```

---

## Output

```text
Before Exception

System.MathException: Divide by 0

In Finally Block

After Exception
```

---

## Explanation

Execution Flow:

1. Try block executes.
2. Exception occurs.
3. Matching catch block executes.
4. Finally block executes.
5. Remaining code executes.

The finally block always runs.

---

# Example: Finally Block Without Exception

```apex
System.debug('Before Exception');

try {

    Double result = 10 / 2;

}
catch(NullPointerException e) {

    System.debug(e);

}
catch(MathException e) {

    System.debug(e);

}
catch(DMLException e) {

    System.debug(e);

}
catch(Exception e) {

    System.debug(e);

}
finally {

    System.debug('In Finally Block');

}

System.debug('After Exception');
```

---

## Output

```text
Before Exception

In Finally Block

After Exception
```

---

## Explanation

No exception occurs because:

```apex
10 / 2
```

is valid.

Therefore:

- No catch block executes.
- Finally block still executes.
- Remaining code executes normally.

---

# What is Throw?

## Definition

The **throw** keyword is used to manually generate an exception.

Syntax:

```apex
throw new ExceptionType();
```

Example:

```apex
throw new NullPointerException();
```

---

## Output

```text
System.NullPointerException:
Script-thrown exception
```

---

## Why Use Throw?

Developers use throw when business logic detects an invalid condition.

Example:

```apex
if(score < 0) {

    throw new MathException();

}
```

This allows custom validation and controlled error handling.

---

# Complete Flow of Exception Handling

```text
Try Block
      ↓
Exception Occurs?
      ↓
   Yes
      ↓
Matching Catch Block
      ↓
 Finally Block
      ↓
Remaining Code
```

If no exception occurs:

```text
Try Block
      ↓
Finally Block
      ↓
Remaining Code
```

# Real-Time Salesforce Examples

## Example 1: DML Operation

```apex
try {

    insert accountRecord;

}
catch(DmlException e) {

    System.debug(e.getMessage());

}
```

Used when records may fail validation rules.

---

## Example 2: SOQL Query

```apex
try {

    Account acc =
    [
        SELECT Id
        FROM Account
        WHERE Name='Test'
    ];

}
catch(QueryException e) {

    System.debug(e.getMessage());

}
```

Used when queries may return unexpected results.

---

## Example 3: Null Object Access

```apex
try {

    String name;

    System.debug(name.toUpperCase());

}
catch(NullPointerException e) {

    System.debug(e);

}
```

Prevents application crashes caused by null references.

# Interview Questions

## What is a try block?

A try block contains code that may generate an exception.

---

## What is a catch block?

A catch block handles exceptions generated inside the try block.

---

## What is a finally block?

A finally block always executes whether an exception occurs or not.

---

## What is the purpose of throw?

The throw keyword is used to manually generate an exception.

---

## Can multiple catch blocks be used?

Yes. Apex allows multiple catch blocks to handle different exception types.

---

## Which catch block executes?

Only the first matching catch block executes.

---

## Does finally execute if no exception occurs?

Yes.

The finally block executes regardless of whether an exception occurs.

---

## Can Exception catch all exceptions?

Yes.

```apex
catch(Exception e)
```

can catch any Apex exception because Exception is the parent class.

---

# Key Points for Revision

- Unhandled exceptions terminate execution.
- Risky code should be placed inside a try block.
- Catch blocks handle runtime exceptions.
- Multiple catch blocks can be used.
- Only the matching catch block executes.
- Exception is the parent class of all Apex exceptions.
- Finally always executes.
- Finally executes even when no exception occurs.
- Throw is used to manually generate exceptions.
- Proper exception handling improves application stability and user experience.

---

# Summary

Exception handling in Apex is implemented using **try, catch, throw, and finally** blocks. The try block contains code that may generate an exception, catch blocks handle specific exceptions, throw is used to manually create exceptions, and finally contains code that always executes. By using proper exception handling, Salesforce developers can prevent application crashes, display meaningful error messages, and build reliable enterprise applications.
