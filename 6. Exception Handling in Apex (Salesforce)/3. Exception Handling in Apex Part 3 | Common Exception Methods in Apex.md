# Exception Handling in Apex Part 3 | Common Exception Methods in Apex

# Introduction

In the previous parts, we learned:

- What exceptions are
- How to use try, catch, throw, and finally blocks
- How to handle different exception types in Apex

In this part, we will learn about some commonly used methods available in the Apex Exception class. These methods help us retrieve detailed information about an exception such as:

- Error Message
- Line Number
- Stack Trace
- Exception Type
- Cause of Exception

These details are extremely useful while debugging Apex code.

---

# Exception Class Methods

The Exception class provides several built-in methods:

| Method | Description |
|----------|-------------|
| `getCause()` | Returns the underlying cause of the exception |
| `getMessage()` | Returns the exception message |
| `getLineNumber()` | Returns the line number where the exception occurred |
| `getStackTraceString()` | Returns the complete stack trace |
| `getTypeName()` | Returns the exception class name |

---

# Example Program

```apex
System.debug('Before Exception');

try {

    Double result = 10 / 0;

}
catch(MathException e) {

    System.debug('Cause : ' + e.getCause());

    System.debug('Message : ' + e.getMessage());

    System.debug('Line Number : ' + e.getLineNumber());

    System.debug('Stack Trace : ' + e.getStackTraceString());

    System.debug('Type Name : ' + e.getTypeName());

}
finally {

    System.debug('In Finally Block');

}

System.debug('After Exception');
```

---

# Output

```text
Before Exception

Cause : null

Message : Divide by 0

Line Number : 3

Stack Trace : AnonymousBlock: line 3, column 1

Type Name : System.MathException

In Finally Block

After Exception
```

# Execution Flow

### Step 1

The following statement executes successfully:

```apex
System.debug('Before Exception');
```

Output:

```text
Before Exception
```

---

### Step 2

A division by zero operation is performed:

```apex
Double result = 10 / 0;
```

This generates:

```text
System.MathException
```

---

### Step 3

The exception is caught by:

```apex
catch(MathException e)
```

Now the exception object `e` contains all details about the error.

---

### Step 4

Different Exception methods are executed.

---

# getCause()

## Definition

Returns the underlying cause of the exception.

Syntax:

```apex
e.getCause()
```

Example:

```apex
System.debug('Cause : ' + e.getCause());
```

Output:

```text
Cause : null
```

---

## Explanation

The divide-by-zero exception was generated directly by Apex.

Since there is no parent exception responsible for creating it:

```text
Cause : null
```

is returned.

---

# getMessage()

## Definition

Returns the message associated with the exception.

Syntax:

```apex
e.getMessage()
```

Example:

```apex
System.debug('Message : ' + e.getMessage());
```

Output:

```text
Message : Divide by 0
```

---

## Explanation

This method provides a human-readable description of the error.

For our example:

```apex
10 / 0
```

Apex generates:

```text
Divide by 0
```

---

# getLineNumber()

## Definition

Returns the line number where the exception occurred.

Syntax:

```apex
e.getLineNumber()
```

Example:

```apex
System.debug('Line Number : ' + e.getLineNumber());
```

Output:

```text
Line Number : 3
```

---

## Explanation

The exception occurred on:

```apex
Double result = 10 / 0;
```

which is located on line 3.

Therefore Apex returns:

```text
3
```

---

# getStackTraceString()

## Definition

Returns the complete stack trace of the exception.

Syntax:

```apex
e.getStackTraceString()
```

Example:

```apex
System.debug('Stack Trace : ' + e.getStackTraceString());
```

Output:

```text
Stack Trace : AnonymousBlock: line 3, column 1
```

---

## Explanation

A stack trace shows:

- Where the exception occurred
- Which class generated the exception
- Which method generated the exception
- The execution path that led to the error

This is one of the most useful methods for debugging complex applications.

---

# getTypeName()

## Definition

Returns the exact exception type.

Syntax:

```apex
e.getTypeName()
```

Example:

```apex
System.debug('Type Name : ' + e.getTypeName());
```

Output:

```text
Type Name : System.MathException
```

---

## Explanation

This method tells us exactly which exception was generated.

Examples:

```text
System.MathException
System.DmlException
System.NullPointerException
System.QueryException
```

In our example the generated exception is:

```text
System.MathException
```

---

# Why These Methods Are Important

While developing Salesforce applications, exceptions may occur in:

- Apex Classes
- Triggers
- Batch Apex
- Queueable Apex
- Future Methods
- Web Service Integrations

Using Exception methods helps developers quickly identify:

- What went wrong
- Where it happened
- Which exception occurred
- What caused the failure

This significantly reduces debugging time.

---

# Real-Time Example

Instead of:

```apex
catch(Exception e) {

    System.debug(e);

}
```

Use:

```apex
catch(Exception e) {

    System.debug('Message : ' + e.getMessage());

    System.debug('Line Number : ' + e.getLineNumber());

    System.debug('Type Name : ' + e.getTypeName());

}
```

This provides much more useful information for troubleshooting.
