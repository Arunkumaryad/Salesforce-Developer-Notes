# Exception Handling in Apex Part 1 | Salesforce Development

# Introduction

While executing an Apex program, unexpected errors may occur. These errors can stop the execution of the program and prevent the expected output from being generated.

To handle such situations, Salesforce provides **Exception Handling**.

---

# What is an Exception?

## Definition

> **An exception is an event, which occurs during the execution of a program, that disrupts the normal flow of the program's instructions.**

In simple words, an exception is a runtime error that occurs while the program is executing.

Examples:

- Dividing a number by zero
- Inserting a record that violates a validation rule
- Accessing a null object
- Calling a method on a null variable
- Invalid type conversion

---

# Why Do We Need Exception Handling?

Exception handling helps developers:

- Prevent program crashes
- Identify runtime errors
- Display meaningful error messages
- Continue program execution when possible
- Improve application reliability

---

# Example 1: Normal Program Execution

The following code executes successfully because division by 2 is valid.

```apex
Double result = 10 / 2;

System.debug('Result : ' + result);
```

## Output

```text
Result : 5.0
```

### Explanation

- 10 is divided by 2.
- The result is stored in a Double variable.
- No exception occurs.
- Output is displayed successfully.

---

# Example 2: Math Exception

The following code attempts to divide a number by zero.

```apex
Double result = 10 / 0;

System.debug('Result : ' + result);
```

## Error

```text
System.MathException: Divide by 0
```

### Explanation

Division by zero is mathematically invalid.

Salesforce throws a:

```text
System.MathException
```

because Apex cannot calculate a value when the denominator is zero.

---

# What is MathException?

## Definition

**MathException occurs when an invalid mathematical operation is performed.**

Examples:

- Divide by zero
- Invalid mathematical calculations

Example:

```apex
Double result = 10 / 0;
```

Output:

```text
System.MathException: Divide by 0
```

---

# Example 3: DML Exception

Consider the following code:

```apex
// DML Exception

Student__c std = new Student__c();

insert std;
```

Suppose a Validation Rule exists on the object requiring the field:

```text
Entrance_Score__c
```

to be mandatory.

---

## Error

```text
System.DmlException:
Insert failed.

First exception on row 0;

FIELD_CUSTOM_VALIDATION_EXCEPTION,

Entrance Score of Student's Should not be blank.

[Entrance_Score__c]
```

---

## Explanation

The record is being inserted without providing the required value.

Because the Validation Rule fails, Salesforce throws:

```text
System.DmlException
```

---

# What is DmlException?

## Definition

**DmlException occurs when a DML operation fails.**

DML operations include:

- insert
- update
- delete
- undelete
- upsert

Examples:

- Validation rule failure
- Required field missing
- Duplicate rule failure
- Field value errors

---

# Example 4: Null Pointer Exception

```apex
// Null Pointer Exception

String str;

System.debug(str.toLowerCase());
```

---

## Error

```text
System.NullPointerException:
Attempt to de-reference a null object
```

---

## Explanation

The variable:

```apex
String str;
```

is declared but not initialized.

Its value is:

```text
null
```

When Apex tries to execute:

```apex
str.toLowerCase();
```

it attempts to call a method on a null object.

Therefore Salesforce throws:

```text
System.NullPointerException
```

---

# What is NullPointerException?

## Definition

**NullPointerException occurs when we try to access a method, property, or object reference that is null.**

Example:

```apex
String str;

str.toLowerCase();
```

Output:

```text
System.NullPointerException:
Attempt to de-reference a null object
```

---

# Manually Throwing an Exception

Developers can also throw exceptions manually.

Example:

```apex
throw new NullPointerException();
```

---

## Error

```text
System.NullPointerException:
Script-thrown exception
```

---

## Explanation

In this case the exception is not generated by Salesforce.

The developer explicitly throws the exception using:

```apex
throw new NullPointerException();
```

# Common Apex Exceptions

Salesforce provides many built-in exception classes.

Some commonly used exceptions are:

| Exception | Description |
|------------|------------|
| AssertException | Thrown when a System.assert() statement fails. |
| AuraException | Legacy Aura-related exception. |
| AuraHandledException | Used to return custom errors to Lightning Components. |
| AsyncException | Occurs during asynchronous processing. |
| BigObjectException | Occurs while working with Big Objects. |
| CalloutException | Occurs during external web service callouts. |
| DmlException | Occurs when a DML operation fails. |
| EmailException | Occurs while sending emails. |
| JSONException | Occurs during JSON parsing or generation. |
| ListException | Occurs while working with Lists. |
| MathException | Occurs due to invalid mathematical operations. |
| NullPointerException | Occurs when accessing a null reference. |
| QueryException | Occurs when a SOQL query fails. |
| SObjectException | Occurs while accessing invalid SObject fields. |
| StringException | Occurs due to invalid string operations. |
| TypeException | Occurs during invalid type conversion. |
| VisualforceException | Occurs in Visualforce processing. |

- Accessing a null object causes a NullPointerException.
- Developers can manually throw exceptions using the throw keyword.

# Summary

Exception Handling is one of the most important concepts in Apex programming. Runtime errors such as divide-by-zero errors, failed DML operations, and null references can stop code execution. Salesforce provides built-in exception classes like MathException, DmlException, and NullPointerException to identify these issues. Understanding exceptions is the first step before learning try-catch-finally blocks, which are used to handle these exceptions gracefully in real-world Salesforce applications.
