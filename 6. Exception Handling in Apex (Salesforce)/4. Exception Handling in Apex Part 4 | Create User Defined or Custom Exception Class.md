Apex provides many built-in exceptions such as:

However, sometimes business requirements demand custom error handling. In such cases, Salesforce allows us to create our own exception classes known as **User Defined Exceptions** or **Custom Exceptions**.

---

# What is a Custom Exception?

## Definition

A **Custom Exception** is a user-created exception class that extends the Apex `Exception` class.

It allows developers to:

- Create meaningful business-specific errors.
- Improve code readability.
- Handle custom validation scenarios.
- Display customized exception messages.

---

# Syntax

```apex
public class CustomExceptionName extends Exception {

}
```

Example:

```apex
public class NewException extends Exception {

}
```

---

# Creating a Custom Exception Class

Create a new Apex Class:

```apex
public class NewException extends Exception {

}
```

---

## Explanation

Here:

```apex
extends Exception
```

means:

- NewException inherits all functionality of the Apex Exception class.
- It can be thrown using the `throw` keyword.
- It can be caught using a `catch` block.
- It supports exception methods such as:
  - getMessage()
  - getCause()
  - getLineNumber()
  - getTypeName()

---

# Example 1: Using a Custom Exception

## Anonymous Window Code

```apex
System.debug('Before Exception');

Integer a = 10,
        b = 0;

try {

    if(b == 0) {

        throw new NewException();

    }
    else {

        Double result = a / b;

    }

}
catch(NewException e) {

    System.debug('Cause : ' + e.getCause());

    System.debug('Message : ' + e.getMessage());

    System.debug('Line Number : ' + e.getLineNumber());

    System.debug('Stack Trace : ' + e.getStackTraceString());

    System.debug('Type Name : ' + e.getTypeName());

}
finally {

    System.debug('In Finally Block');

}

System.debug('After Exception');
```

---

# Output

```text
Before Exception

Cause : null

Message : Script-thrown exception

Line Number : 5

Stack Trace : AnonymousBlock: line 5, column 1

Type Name : NewException

In Finally Block

After Exception
```

# Why Message Shows "Script-thrown exception"?

Output:

```text
Message : Script-thrown exception
```

---

## Reason

The custom exception class does not define its own message.

Current class:

```apex
public class NewException extends Exception {

}
```

Therefore Apex automatically generates the default message:

```text
Script-thrown exception
```

---

# Customizing Exception Messages

We can override the `getMessage()` method to provide our own custom message.

---

# Modified Exception Class

```apex
public class NewException extends Exception {

    override public String getMessage() {

        return 'Divide by Zero Exception';

    }

}
```

---

# Explanation

Here we override:

```apex
getMessage()
```

and return a custom error message:

```apex
return 'Divide by Zero Exception';
```

Now Apex will display our message instead of the default:

```text
Script-thrown exception
```

# Real-Time Business Example

Suppose a company has a rule:

> A student's entrance score cannot be less than zero.

Instead of throwing a generic exception:

```apex
throw new Exception();
```

Create a custom exception:

```apex
public class InvalidScoreException extends Exception {

}
```

Usage:

```apex
if(score < 0) {

    throw new InvalidScoreException();

}
```

This makes the code much easier to understand.

# Key Points for Revision

- Custom exceptions are user-defined exception classes.
- They are created using the `extends Exception` keyword.
- Custom exceptions can be thrown using `throw`.
- Custom exceptions can be handled using `catch`.
- If no custom message is defined, Apex displays:

# Summary

Apex allows developers to create their own exception classes by extending the built-in `Exception` class. These custom exceptions help implement business-specific error handling, improve code readability, and make debugging easier. By overriding methods like `getMessage()`, developers can provide meaningful error messages instead of generic system-generated messages. Custom exceptions are a best practice when implementing complex business validations in Salesforce applications.
