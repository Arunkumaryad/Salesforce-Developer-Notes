# Schema Builder in Salesforce

# Introduction

Schema Builder is a Salesforce tool used to:

- View Objects
- Create Objects
- Create Fields
- Create Relationships
- Manage Database Structure visually

It provides a graphical view of all objects and relationships inside Salesforce.

---

# Schema Builder

# Definition

Schema Builder is:

```text
A dynamic environment where we can view and modify Objects and Relationships in App.
```

It is:

```text
The graphical representation of Objects and Relationships in App.
```

---

# Why Use Schema Builder?

Schema Builder helps developers and administrators to:

- Understand object relationships visually
- Create objects quickly
- Add fields easily
- Create relationships using drag-and-drop
- Manage schema structure efficiently

---

# Features of Schema Builder

| Feature | Description |
|---|---|
| Visual Representation | Displays objects and relationships graphically |
| Drag and Drop | Easy field creation |
| Relationship View | Shows object connections |
| Object Creation | Create custom objects |
| Field Creation | Add custom fields |
| Relationship Creation | Create Lookup and Master-Detail relationships |

---

# Step-by-Step Implementation of Schema Builder

# Step 1: Open Salesforce Setup

Click:

```text
Gear Icon → Setup
```

---

# Step 2: Search Schema Builder

Inside Setup Search Box:

```text
Schema Builder
```

Click:

```text
Schema Builder
```

---

# Step 3: Open Schema Builder

Now Salesforce opens:

```text
Schema Builder Workspace
```

This workspace displays:

- Objects
- Fields
- Relationships
- Graphical Schema Structure

---

# Relationship Representation

# Lookup Relationship

Displayed using:

```text
Blue Colored Line
```

Example:

```text
Staff Room → Teacher
```

This represents:

- One Staff Room can have many Teachers.
- Relationship is optional.

---

# Master-Detail Relationship

Displayed using:

```text
Red Colored Line
```

Example:

```text
Teacher → Student Teacher
```

This represents:

- Strong parent-child relationship
- Child record depends on parent

---

# Required Field Representation

Required fields are displayed using:

```text
Red Mark
```

Meaning:

```text
Field value is mandatory
```
