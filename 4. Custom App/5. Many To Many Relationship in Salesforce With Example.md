# Many To Many Relationship in Salesforce With Example

# Introduction

A Many-to-Many Relationship in Salesforce is used when:

- One record can be connected with multiple records.
- And the second object can also be connected with multiple records.

Salesforce does not directly support Many-to-Many relationships.

So, we create it using:

# Junction Object

A Junction Object is a custom object that contains:

- Two Master-Detail Relationships

This object acts as a bridge between two parent objects.

---

# What is Many-to-Many Relationship?

# Definition

Many-to-Many Relationship is a relationship where:

- One record can be related to many records.
- And many records can also be related back to one record.

Salesforce creates Many-to-Many relationships using:

```text
Junction Object
```

with:

```text
Two Master-Detail Relationships
```

---

# Real-Time Example

```text
Student ↔ Teacher
```

- One Student can study from many Teachers.
- One Teacher can teach many Students.

---

# Objects Used

| Object | Type |
|---|---|

# Relationship Structure

| Relationship | Type | Field |
|---|---|---|
| College → Student | One to Many | Master-Detail |
| Staff Room → Teacher | One to Many | Lookup |
| Student ↔ Teacher | Many to Many | Junction Object |

---

# Parent and Child Objects

## Parent Objects

- Student
- Teacher

## Child Object

- Student Teacher (Junction Object)

---

# Important Note

## Relationship fields should always be created in the Junction Object.

Example:

```text
Student ← Student Teacher → Teacher
```

- Student = Parent Object
- Teacher = Parent Object
- Student Teacher = Child Object

---

# Junction Object

# Definition

A Junction Object is a custom object used to create a Many-to-Many relationship between two objects using two Master-Detail relationships.

---

# Architecture of Many-to-Many Relationship

## 1. Master Detail Relationship

| Parent | Child |
|---|---|
| Student | Student Teacher |

---

## 2. Master Detail Relationship

| Parent | Child |
|---|---|
| Teacher | Student Teacher |

# Step-by-Step Implementation of Many-to-Many Relationship

# Step 1: Open Salesforce Setup

Click:

```text
Gear Icon → Setup
```

---

# Step 2: Open Object Manager

Navigate to:

```text
Setup → Object Manager
```

Click:

```text
Create → Custom Object
```

---

# Step 3: Create Junction Object

Create a custom object using the following details:

| Field | Value |
|---|---|
| Label | Student Teacher |
| Plural Label | Student Teacher |
| Object Name | Student_Teacher |

---

Enter:

| Field | Value |
|---|---|
| Record Name | Student Teacher Name |
| Data Type | Auto Number |
| Display Format | Junc-{0000} |

# Difference Between One-to-Many and Many-to-Many

| One-to-Many | Many-to-Many |
|---|---|
| One parent, many child records | Many records connected both sides |
| Single relationship field | Junction Object required |
| Simpler structure | More flexible |
| Example: College → Student | Example: Student ↔ Teacher |

# How Many Master-Detail Relationships are required?

```text
Two Master-Detail Relationships
```

# Can Salesforce directly create Many-to-Many Relationship?

```text
No
```

Salesforce uses Junction Objects to create Many-to-Many relationships.

# Which object becomes Child Object?

```text
Junction Object
```
