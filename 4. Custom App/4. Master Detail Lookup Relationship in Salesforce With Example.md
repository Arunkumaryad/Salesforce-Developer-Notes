# Master Detail & Lookup Relationship in Salesforce With Example

# Introduction

Salesforce provides Object Relationships to connect one object with another object.

Relationships help:
- Connect related records
- Reduce duplicate data
- Improve data management
- Maintain proper database structure

# What is Object Relationship?

## Definition

Object Relationship is used to connect two objects together in Salesforce.

Relationships help in:
- Sharing related data
- Managing connected records
- Avoiding data redundancy
- Improving data consistency

---

# Types of Relationships Used

## 1. Master Detail Relationship
## 2. Lookup Relationship

---

# Parent and Child Objects

## Parent Object
The object that controls another object.

## Child Object
The object that depends on the parent object.

---

# Important Note

## Relationship field should always be created in the Child Object.

Example:

```text
College → Student
```

- College = Parent Object
- Student = Child Object

---

# Master Detail Relationship

# Definition

Master Detail Relationship creates a special type of parent-child relationship between this object (the child, or "detail") and another object (the parent, or "master") where:

- The relationship field is required on all detail records.
- The ownership and sharing of a detail record are determined by the master record.
- When a user deletes the master record, all detail records are deleted.
- You can create rollup summary fields on the master record to summarize the detail records.

---

# Real-Time Example

```text
College → Student
```

One College can have many Students.

---

# Features of Master Detail Relationship

| Feature | Description |
|----------|-------------|
| Strong Relationship | Child depends on Parent |
| Mandatory Relationship Field | Child must contain Parent |
| Cascade Delete | Child records deleted automatically |
| Roll-Up Summary | Supported |
| Sharing & Ownership | Controlled by Parent |

---

Click:

```text
Gear Icon → Setup
```

Navigate to:

```text

```

```text

```

Inside Student Object:





- College = Parent (Master)

# Lookup Relationship

# Definition

Lookup Relationship creates a relationship that links this object to another object.

The relationship field allows users to click on a lookup icon to select a value from a popup list.

The other object is the source of the values in the list.

---

# Real-Time Example

```text
Staff Room → Teacher
```

One Staff Room can contain many Teachers.

---

# Features of Lookup Relationship

| Feature | Description |
|----------|-------------|
| Loose Relationship | Child can exist independently |
| Optional Relationship Field | Not mandatory |
| No Cascade Delete | Child records are not deleted automatically |
| Independent Objects | Parent and Child independent |
| Roll-Up Summary | Not supported directly |

# Difference Between Master Detail and Lookup Relationship

| Master Detail | Lookup |
|---------------|---------|
| Strong Relationship | Loose Relationship |
| Child depends on Parent | Child independent |
| Relationship field mandatory | Relationship field optional |
| Cascade delete supported | No cascade delete |
| Roll-Up Summary supported | Not supported directly |
| Ownership controlled by Parent | Independent ownership |

# Interview Questions

# What is Master Detail Relationship?

Master Detail Relationship is a strong parent-child relationship where child record depends completely on parent record.

---

# What is Lookup Relationship?

Lookup Relationship is a loose relationship where child record can exist independently.

---

# What is Cascade Delete?

When parent record is deleted, all related child records are automatically deleted.

This feature is available in:
```text
Master Detail Relationship
```

---

# What is Roll-Up Summary Field?

Roll-Up Summary Field is used to summarize child records on parent object.

Example:
- Count total students in a college
- Sum total fees
