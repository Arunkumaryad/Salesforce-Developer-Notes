# Object Relationships in Salesforce | Master Detail, Lookup, Many to Many

# Introduction

In Salesforce, data is stored inside objects.

Sometimes multiple objects are connected with each other.  
To connect objects, Salesforce provides Object Relationships.

Relationships help:
- Avoid Data Redundancy
- Avoid Data Inconsistency
- Maintain proper data structure
- Connect related records together

---

# Why Relationships are Required?

## Objects are divided into smaller objects to avoid:

1. Data Redundancy  
2. Data Inconsistency

---

# What is Data Redundancy?

## Definition

Data Redundancy means storing the same data multiple times unnecessarily.

### Example

If the same college information is stored repeatedly for every student record, it creates duplicate data.

---

# What is Data Inconsistency?

## Definition

Data Inconsistency means different versions of the same data exist in different places.

### Example

One record shows:
```text
College = PICT
```

Another record shows:
```text
College = Pune Institute of Computer Technology
```

Both represent the same college but data is inconsistent.

---

# Relationships Between Two Objects

There are mainly 4 types of relationships between objects:

1. One to One  
2. One to Many  
3. Many to One  
4. Many to Many

---

# Relationships Used in Salesforce

In Salesforce mainly these relationships are used:

1. One to Many
   - Master Detail Relationship
   - Lookup Relationship

2. Many to Many

---

# Important Note

## Relationship field should always be created in the Child Object.

---

# Parent and Child Objects

## Parent Object
The object that controls another object.

## Child Object
The object that depends on the parent object.

---

# Example

| Parent Object | Child Object |
|---------------|--------------|
| College | Student |
| Staff Room | Teacher |

---

# Master Detail Relationship

## Definition

Master Detail Relationship creates a strong parent-child relationship between two objects.

- Master = Parent Object
- Detail = Child Object

---

# Structure

```text
Master (Parent) → Detail (Child)
```

---

# Example

```text
College → Student
```

One College can contain many Student records.

---

# Features of Master Detail Relationship

## 1. Relationship Field is Mandatory

The relationship field is required on all detail (child) records.

Example:
Every Student must belong to a College.

---

## 2. Child Record Depends on Parent

The child object cannot exist independently.

Example:
Student record depends on College record.

---

## 3. Ownership and Sharing Controlled by Parent

The ownership and sharing settings of child records are controlled by the parent object.

---

## 4. Cascade Delete

When parent record is deleted, all child records are automatically deleted.

### Example

If College is deleted:
- All related Student records are also deleted.

---

## 5. Roll-Up Summary Fields Supported

Salesforce allows creation of Roll-Up Summary fields on the parent object.

### Example

Count total students in a college.

---

# Real-Time Example

| Parent Object | Child Object | Relationship |
|---------------|--------------|--------------|
| College | Student | Master Detail |

# Lookup Relationship

# Definition

Lookup Relationship creates a loose connection between two objects.

Child object can exist independently.

---

# Example

```text
Staff Room → Teacher
```

One Staff Room can contain many Teachers.

---

# Staff Room (Parent)

| Name | Dept | Area |
|------|------|------|
| A304 | CE | 35 |
| A301 | IT | 40 |
| A305 | ENTC | 41 |

---

# Teacher (Child)

| Name | Dept | Teacher ID | Staff Room |
|------|------|-------------|-------------|
| Parag Jambhulkar | CE | 123 | A304 |
| Mayur Chavan | CE | 234 | A301 |
| Yogesh Handge | IT | 421 | A304 |

---

# Features of Lookup Relationship

## 1. Relationship Field is Optional

Child record can exist without parent record.

---

## 2. No Cascade Delete

Deleting parent record does not delete child records automatically.

---

## 3. Independent Objects

Both parent and child can exist independently.

---

## 4. Roll-Up Summary Not Supported Directly

Roll-Up Summary fields are not directly available in Lookup Relationship.

---

# Real-Time Example

| Parent Object | Child Object | Relationship |
|---------------|--------------|--------------|
| Staff Room | Teacher | Lookup |

---

# Many to Many Relationship

# Definition

Many records of one object can be related to many records of another object.

---

# Example

```text
Student ↔ Teacher
```

- One Student can study with many Teachers
- One Teacher can teach many Students

---

# How Many to Many Relationship Works?

Salesforce creates Many to Many Relationship using:

```text
Junction Object
```

---

# What is Junction Object?

## Definition

A Junction Object is a custom object that contains two Master Detail Relationships.

---

# Example

| Object 1 | Junction Object | Object 2 |
|-----------|----------------|-----------|
| Student | Student_Teacher | Teacher |

---

# Comparison Table

| Feature | Master Detail | Lookup |
|----------|---------------|---------|
| Strong Relationship | Yes | No |
| Child Depends on Parent | Yes | No |
| Cascade Delete | Yes | No |
| Roll-Up Summary | Supported | Not Supported |
| Relationship Field Required | Yes | No |

---

# Important Interview Questions

# What is Object Relationship in Salesforce?

Object Relationship is used to connect two objects together.

---

# What is Master Detail Relationship?

Master Detail Relationship is a strong parent-child relationship where child record depends on parent record.

---

# What is Lookup Relationship?

Lookup Relationship is a loose relationship where child object can exist independently.

---

# What is Many to Many Relationship?

Many to Many Relationship allows multiple records of one object to connect with multiple records of another object using a Junction Object.

---

# What is Junction Object?

A Junction Object is a custom object that contains two Master Detail Relationships.

---

# Difference Between Master Detail and Lookup

| Master Detail | Lookup |
|---------------|---------|
| Strong relationship | Loose relationship |
| Cascade delete available | No cascade delete |
| Roll-up summary supported | Not supported |
| Child depends on parent | Independent objects |
