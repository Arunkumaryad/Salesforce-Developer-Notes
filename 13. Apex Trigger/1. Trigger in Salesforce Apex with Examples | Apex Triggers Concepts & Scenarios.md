# What is a Trigger in Salesforce?

## Definition

A database trigger is a special set of operations that runs automatically when specific actions occur within a database.

In Salesforce, a Trigger is Apex code that executes automatically before or after specific events occur on a Salesforce object.

### Trigger executes on the following operations:

- Insert
- Update
- Delete
- Merge
- Upsert
- Undelete

---

## Trigger Definition (Actual Concept)

> A Trigger is Apex code that executes before or after the following types of operations on objects:
>
> - Insert
> - Update
> - Delete
> - Merge
> - Upsert
> - Undelete

---

# What is the Use of a Trigger?

Apex Triggers allow developers to perform custom actions before or after records are saved in Salesforce.

Examples:

- Auto-populate fields
- Perform validations
- Update related records
- Call external systems
- Prevent invalid data from being saved
- Automate business processes

---

## Main Purpose

> Triggers are used for automating business processes.

---

# Why Do We Need Apex Triggers?

### 1. Automation

Triggers provide endless possibilities for automating tasks.

Example:

```text
Automatically update Status when a record is created.
```

---

### 2. Complex Validations

Validation Rules have limitations.

Triggers can implement advanced business validations.

Example:

```text
Student age must be greater than 18 only if course = MBA.
```

---

### 3. Integration with Third-Party Systems

Triggers can communicate with external systems whenever Salesforce data changes.

Example:

```text
Send data to SAP, ERP, or external APIs.
```

---

### 4. Faster Than Legacy Automation Tools

Triggers are generally faster than:

- Workflow Rules
- Process Builder
- Visual Workflow

because Apex code runs directly within the transaction.

# Example 1: Before Insert Trigger

## Requirement

If Branch is blank while creating a Student record:

```text
Branch = Others
```

---

## Trigger Code

```apex
trigger UpdateStudentField on Student1__c (
    before insert
) {

    for(Student1__c std : Trigger.new) {

        if(std.Branch__c == NULL) {

            std.Branch__c = 'Others';

        }

    }

}
```

---

## Explanation

### Trigger.new

Contains newly created records.

```apex
Trigger.new
```

returns:

```text
List<Student1__c>
```

---

### Before Insert

Runs before the record is saved.

Because record is not saved yet:

```apex
std.Branch__c = 'Others';
```

works without DML.

---

## Output

Input:

```text
Student Name = Radha
Branch = Blank
```

Result:

```text
Branch = Others
```

automatically populated before saving.

# Difference Between Before and After Triggers

## Before Trigger

### Definition

Before Trigger is used to update or validate record values before they are saved into the database.

The trigger executes first and then Salesforce saves the record.

### Common Uses

- Validation
- Default values
- Field updates
- Data formatting

Example:

```apex
before insert
before update
```

---

## After Trigger

### Definition

After Trigger is used when you need access to record IDs or system-generated values after the record has been saved.

It is commonly used to update related records.

### Common Uses

- Create related records
- Update parent records
- Send notifications
- Integration logic

Example:

```apex
after insert
after update
```

---

## Comparison Table

| Before Trigger | After Trigger |
|---------------|--------------|
| Executes before save | Executes after save |
| Can modify Trigger.new records | Cannot directly modify Trigger.new |
| Used for validations | Used for related record operations |
| Faster | Slightly slower |
| No extra DML needed | Usually requires DML |

# Validation and Throwing Errors

Sometimes we want to stop a record from being saved.

Use:

```apex
addError()
```

method.

---

## Example

Requirement:

```text
Branch cannot be blank.
```

---

## Trigger Code

```apex
trigger UpdateStudentField on Student1__c (
    before insert,
    before update
) {

    for(Student1__c std : Trigger.new) {

        if(std.Branch__c == NULL) {

            std.addError(
                'Branch is Required. Please Enter the value'
            );

        }

    }

}
```

---

## Explanation

```apex
addError()
```

prevents the record from being saved.

Salesforce displays the error message to the user.

---

## Output

If Branch is empty:

```text
Branch is Required. Please Enter the value
```

Record will not be saved.

# Multiple Triggers

Salesforce allows multiple triggers on the same object.

Example:

```text
Student Trigger 1

Student Trigger 2

Student Trigger 3
```

All can exist simultaneously.

---

## Important Points

### 1. Multiple Triggers Are Allowed

You can create multiple triggers on the same object.

---

### 2. Execution Order Is Not Guaranteed

Salesforce does not guarantee which trigger runs first.

Example:

```text
Trigger A

Trigger B

Trigger C
```

Any trigger may execute before another.

---

### 3. Shared Governor Limits

All triggers on the same transaction share:

- SOQL Limits
- DML Limits
- CPU Limits
- Heap Limits

---

## Example Problem

Trigger A:

```apex
100 SOQL queries
```

Trigger B:

```apex
20 SOQL queries
```

Total:

```text
120 Queries
```

Governor Limit:

```text
100 Queries
```

Result:

```text
System.LimitException
```

---

# Trigger Best Practices

## 1. One Trigger Per Object

Recommended:

```text
AccountTrigger

ContactTrigger

OpportunityTrigger
```

Avoid:

```text
AccountTrigger1
AccountTrigger2
AccountTrigger3
```

---

## 2. Use Trigger Handler Pattern

Trigger:

```apex
trigger AccountTrigger on Account(
    before insert,
    before update
){
    AccountTriggerHandler.run();
}
```

Business logic should be inside Apex classes.

---

## 3. Bulkify Your Trigger

Never write:

```apex
for(Account acc : Trigger.new){

    insert new Contact();

}
```

Instead process records in collections.

---

## 4. Avoid SOQL Inside Loops

Bad:

```apex
for(Account acc : Trigger.new){

    Account a = [
        SELECT Id
        FROM Account
        WHERE Id = :acc.Id
    ];

}
```

---

Good:

```apex
Set<Id> ids = new Set<Id>();
```

Query once.

---

## 5. Avoid DML Inside Loops

Bad:

```apex
for(Account acc : Trigger.new){

    update acc;

}
```

---

Good:

```apex
List<Account> accountsToUpdate =
    new List<Account>();

update accountsToUpdate;
```
