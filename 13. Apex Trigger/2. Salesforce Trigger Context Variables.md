# Salesforce Trigger Context Variables

## Overview

Trigger Context Variables are predefined variables provided by Salesforce that help determine:

- Which operation triggered the execution
- Whether the trigger is running before or after an event
- Which records are involved
- Old and new values of records
- Current operation type
- Total records being processed

These variables are available only inside Apex Triggers.

---

# What are Trigger Context Variables?

Trigger Context Variables are built-in variables that Salesforce automatically makes available when a trigger executes.

They provide information about:

- Trigger execution context
- Event type
- Record data
- Old values
- New values
- Trigger operation

These variables help write dynamic and reusable trigger logic.

---

# Trigger Context Variables Table

| Variable | Data Type | Description |
|-----------|-----------|-------------|
| Trigger.isInsert | Boolean | Returns true if the trigger was fired due to an insert operation. |
| Trigger.isUpdate | Boolean | Returns true if the trigger was fired due to an update operation. |
| Trigger.isDelete | Boolean | Returns true if the trigger was fired due to a delete operation. |
| Trigger.isUndelete | Boolean | Returns true if the trigger was fired after a record has been restored from the Recycle Bin. |
| Trigger.isBefore | Boolean | Returns true if the trigger is executing before records are saved. |
| Trigger.isAfter | Boolean | Returns true if the trigger is executing after records are saved. |
| Trigger.new | List\<SObject> | Returns a list of new versions of the records. |
| Trigger.newMap | Map\<Id, SObject> | Returns a map of IDs to new versions of records. |
| Trigger.old | List\<SObject> | Returns a list of old versions of records. |
| Trigger.oldMap | Map\<Id, SObject> | Returns a map of IDs to old versions of records. |
| Trigger.isExecuting | Boolean | Returns true if the current Apex code is running in a trigger context. |
| Trigger.size | Integer | Returns the total number of records in the trigger invocation. |
| Trigger.operationType | Enum | Returns the current trigger operation type. |

# Trigger Events Example

Consider the following trigger:

```apex
trigger UpdateStudentField on Student1__c (
    before insert,
    before update,
    before delete
) {

    if(Trigger.isBefore && Trigger.isInsert) {

        System.debug(
            'Trigger fired due to insert operation and Before event.'
        );

    }

    if(Trigger.isBefore && Trigger.isUpdate) {

        System.debug(
            'Trigger fired due to update operation and Before event.'
        );

    }

    if(Trigger.isBefore && Trigger.isDelete) {

        System.debug(
            'Trigger fired due to delete operation and Before event.'
        );

    }

}
```

# Trigger.new

## Definition

Returns the new versions of records involved in the trigger.

### Data Type

```apex
List<SObject>
```

---

## Example

```apex
trigger UpdateStudentField on Student1__c (
    after update
) {

    for(Student1__c std : Trigger.new) {

        System.debug(
            'New Score : ' + std.Score__c
        );

    }

}
```

---

## Output

If score changes:

```text
New Score : 54
```

# Comparing Trigger.new and Trigger.old

## Example

```apex
trigger UpdateStudentField on Student1__c (
    after update
) {

    if(Trigger.isAfter && Trigger.isUpdate) {

        for(Student1__c std : Trigger.new) {

            System.debug(
                'New Score : ' + std.Score__c
            );

        }

        for(Student1__c std : Trigger.old) {

            System.debug(
                'Old Score : ' + std.Score__c
            );

        }

    }

}
```

---

## Output

```text

Old Score : 98
```

---

# Real-Time Example Using oldMap

## Requirement

If new score is less than old score, keep the old score.

---

## Trigger Code

```apex
trigger UpdateStudentField on Student1__c (
    before update
) {

    if(Trigger.isBefore &&
       Trigger.isUpdate) {

        for(Student1__c std :
            Trigger.new) {

            Decimal oldScore =
                Trigger.oldMap
                .get(std.Id)
                .Score__c;

            if(std.Score__c < oldScore) {

                std.Score__c = oldScore;

            }

        }

    }

}
```

---

## Example

Current Score:

```text
98
```

User tries to update:

```text
12
```

Trigger executes:

```apex
if(12 < 98)
```

Result:

```text
Score remains 98
```

# Validation Example Using Trigger.new

## Requirement

Score should not be empty.

---

## Trigger Code

```apex
trigger UpdateStudentField on Student1__c (
    before update
) {

    if(Trigger.isBefore &&
       Trigger.isUpdate) {

        for(Student1__c std :
            Trigger.new) {

            if(std.Score__c == NULL) {

                std.addError(
                    'Score can not be empty.'
                );

            }

        }

    }

}
```

---

## Output

If user clears the score field:

```text
Score can not be empty.
```

Record will not be saved.

# Interview Questions

## What are Trigger Context Variables?

Built-in variables that provide information about the trigger execution environment.

---

## What is the difference between Trigger.new and Trigger.old?

### Trigger.new

Contains updated/new records.

### Trigger.old

Contains previous versions of records.

---

## Can Trigger.old be used in Insert Trigger?

No.

Because there is no old record available during insertion.

---

## Can Trigger.new be modified in After Trigger?

No.

Trigger.new is read-only in after triggers.

---

## What is Trigger.size?

Returns the number of records currently processed by the trigger.

---

## What is Trigger.operationType?

Returns the current trigger event as an enum value.

Example:

```text
BEFORE_UPDATE
AFTER_INSERT
AFTER_DELETE
```

---

# Best Practices

### Use Trigger.newMap and Trigger.oldMap whenever possible

Faster lookup than loops.

---

### Avoid SOQL inside loops

Use collections and maps.

---

### Use Trigger.size for bulk processing

Helps handle multiple records efficiently.

---

### Use Trigger.operationType for cleaner code

Instead of multiple nested conditions.
