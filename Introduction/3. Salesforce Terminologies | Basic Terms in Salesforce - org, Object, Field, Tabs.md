# Salesforce Terminologies

## Introduction

Salesforce uses several important terminologies that every Salesforce Administrator and Developer should understand.

These terms help us understand how data and applications are organized inside Salesforce.

---

# Basic Salesforce Terminologies

1. Org
2. Object
3. Field
4. Record
5. Tab
6. App

---

# 1. Org

## Definition

A deployment of Salesforce with a defined set of licensed users.

An organization is the virtual space provided to an individual customer of Salesforce.

Your organization includes all of your data and applications, and is separate from all other organizations.

---

# Types of Org

1. Production Org
2. Development Org

---

# 1. Production Org

Production orgs are designed to be used with live customer data and to actively run your business.

---

## Four Editions of Production Org

1. Essentials
2. Professional
3. Enterprise
4. Unlimited

---

# 2. Development Org

Development org is used for building and testing new features and customisations before they are released into the production org.

There are several types of development instance, each serving different purposes and with different sets of features and customisations available.

---

## Common Types of Development Org

1. Full Copy Sandbox
2. Partial Copy Sandbox
3. Developer Pro Org
4. Developer Edition Org
5. Scratch Org

---

# 2. Object

## Definition

Object is Table of Data distributed among Rows and Columns.

Objects are used to store data in Salesforce.

---

## Examples of Standard Objects

- Account
- Contact
- Lead
- Opportunity

---

## Types of Objects

### 1. Standard Object
Objects provided by Salesforce by default.

Examples:
- Account
- Contact
- Lead

---

### 2. Custom Object
Objects created by users based on business requirements.

Example:
- Student
- Employee
- Property

---

# 3. Field

## Definition

Every Column is called as Field.

Fields store specific information about a record.

---

## Examples of Fields

| Field Name | Data |
|------------|------|
| Name | Arun Kumar |
| Email | arun@gmail.com |
| Phone | 9876543210 |

---

## Types of Fields

- Text
- Number
- Date
- Checkbox
- Picklist
- Email
- Phone

---

# 4. Record

## Definition

Every Row in Object (i.e. Table) represents a Record.

A record stores complete information about a particular entry.

---

## Example

| Name | Email | Phone |
|------|-------|-------|
| Arun | arun@gmail.com | 9876543210 |

This complete row is called a Record.

---

# 5. Tab

## Definition

Tabs in Salesforce are reference links that redirect us to display data of Objects/Web Page/Visualforce Page/Lightning Component.

Tabs help users easily access different Salesforce features and records.

---

## Types of Tabs

- Standard Object Tab
- Custom Object Tab
- Web Tab
- Visualforce Tab
- Lightning Component Tab

---

# 6. App

## Definition

Number of items that work together to serve a particular functionality is termed as App.

These items include:
- Standard Objects
- Custom Objects
- Visualforce Tabs
- Web Tabs
- Lightning Component Tabs

Salesforce app can be:
- Classic App
- Lightning App

---

# Example of Salesforce Apps

- Sales App
- Service App
- Marketing App

---

# Difference Between Object, Field, and Record

| Terminology | Meaning |
|-------------|----------|
| Object | Table |
| Field | Column |
| Record | Row |

---

# Why Salesforce Terminologies are Important

Understanding Salesforce terminologies helps in:
- Learning Salesforce easily
- Understanding data structure
- Working with applications
- Development and customization
- Interview preparation

# Interview Questions

## What is an Org in Salesforce?
An Org is a virtual Salesforce environment provided to customers for storing data and applications.

---

## What is a Production Org?
Production Org is used with live customer data to run business operations.

---

## What is a Development Org?
Development Org is used for building and testing new features before deployment to production.

---

## What is an Object in Salesforce?
Object is a table of data distributed among rows and columns.

---

## What is a Field?
Every column in an object is called a Field.

---

## What is a Record?
Every row in an object represents a Record.

---

## What is a Tab in Salesforce?
Tabs are reference links used to display data of objects, pages, or components.

---

## What is an App in Salesforce?
A collection of items working together to serve a specific functionality is called an App.

---

# Key Revision Points

- Org = Salesforce environment
- Object = Table
