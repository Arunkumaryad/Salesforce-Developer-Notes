# Workflow Rules in Salesforce with Example

## Introduction

Workflow Rules are one of Salesforce's automation tools that help automate standard business processes without writing code.

A Workflow Rule evaluates records based on specified criteria and automatically performs actions when those criteria are met.

Workflow Rules are commonly used to:

- Update fields automatically
- Send email notifications
- Create tasks
- Send outbound messages to external systems

> **Note:** Salesforce recommends using Flow Builder for new automation, but Workflow Rules are still important for understanding legacy Salesforce implementations and interview preparation.

---

# What are Workflow Rules?

## Definition

Workflow Rules allow you to automate standard internal procedures and processes to save time across your Salesforce organization.

A Workflow Rule is the main container for a set of workflow instructions.

These instructions can always be summarized using an:

```text
IF condition is true
THEN perform action
```

statement.

---

# Workflow Rule Structure

A Workflow Rule consists of two main parts:

## 1. Workflow Rule Criteria

The **IF** part of the statement.

It defines the conditions that must be true before Salesforce executes any actions.

Example:

```text
Student Score >= 40
```

---

## 2. Workflow Rule Actions

The **THEN** part of the statement.

It defines what Salesforce should do when the criteria are satisfied.

Workflow can perform the following actions:

### 1. Task

Assign a task to a user, role, or record owner.

### 2. Email Alert

Send an email notification to one or more recipients.

### 3. Field Update

Update a field value automatically.

### 4. Outbound Message

Send a secure XML message to an external system.

---

# Workflow Rule Architecture

```text
Record Created/Updated
          ↓
 Workflow Rule Criteria
          ↓
  Criteria Met?
     ↓      ↓
   Yes      No
    ↓        ↓
 Workflow Actions
    ↓
 Task / Email Alert /
 Field Update /
 Outbound Message
```

# Example Scenario

## Problem Statement

Suppose we have a custom object called:

```text
Student
```

with a field:

```text
Score
```

### Requirement

If:

```text
Score >= 40
```

then Salesforce should:

### Action 1

Mark:

```text
Exam Passed = True
```

### Action 2

Send an email alert to:

```text
Object Owner / Creator
```

### Action 3

Assign a task to:

```text
Test3 User
```

to review the student's score.

### Action 4

Send an outbound message containing student details.

# Immediate vs Time-Dependent Workflow Actions

## Immediate Workflow Action

Executes instantly when criteria are met.

Examples:

```text
Email Alert
Task
Field Update
Outbound Message
```

---

## Time-Dependent Workflow Action

Executes after a specified time.

Examples:

```text
2 days after creation
7 days before due date
30 minutes after update
```

# Advantages of Workflow Rules

### No Coding Required

Business users can automate processes easily.

### Saves Time

Reduces manual effort.

### Improves Accuracy

Minimizes human errors.

### Automates Notifications

Automatically informs users.

### Supports External Integrations

Through Outbound Messages.

---

# Limitations of Workflow Rules

### Limited Logic

Cannot handle complex branching.

### No Record Creation

Cannot create records directly.

### No Loops

Cannot iterate through collections.

### Being Replaced by Flow

Salesforce recommends Flow Builder for future automation.

---

# Interview Questions

## What is a Workflow Rule?

A Workflow Rule is a Salesforce automation tool that evaluates records based on specified criteria and automatically performs actions when those criteria are met.

---

## What are the two parts of a Workflow Rule?

1. Workflow Rule Criteria
2. Workflow Rule Actions

---

## What actions can Workflow Rules perform?

1. Task
2. Email Alert
3. Field Update
4. Outbound Message

---

## What is an Immediate Workflow Action?

An action that executes immediately when the workflow criteria are satisfied.

---

## What is a Time-Dependent Workflow Action?

An action scheduled to execute at a future date or time after the workflow criteria are met.

---

## Can Workflow Rules create records?

No.

Workflow Rules cannot create records. Flow Builder should be used for such requirements.

---

# Summary

Workflow Rules are Salesforce automation tools that follow an **IF-THEN** model. In this example, a Workflow Rule was created on the Student object with the condition **Score >= 40**. When the condition is met, Salesforce automatically performs four actions: creates a task for **Test3 User**, sends an email alert, updates the **Exam Passed** field to **True**, and sends an outbound message containing student information. Although Workflow Rules are being replaced by Flow Builder, they remain important for maintaining legacy Salesforce implementations and for Salesforce interviews.
