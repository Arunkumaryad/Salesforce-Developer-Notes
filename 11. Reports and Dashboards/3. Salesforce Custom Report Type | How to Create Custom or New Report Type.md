# Salesforce Custom Report Type | How to Create Custom or New Report Type

## Overview

### What is a Custom Report Type?

Custom Report Types define the criteria from which users can run and create reports.

When you create a report type, you specify the objects, relationships, and fields that users can select for their reports.

Unlike Standard Report Types, Custom Report Types allow administrators to control:

- Which objects are included in reports
- How objects are related
- Which records appear in reports
- Which fields are available to report users

---

## Why Use Custom Report Types?

Custom Report Types are useful when:

- Standard report types do not meet business requirements.
- You need reports across multiple related objects.
- You want to control which records appear in reports.
- You need custom relationships between objects.
- You want additional fields available in reports.

### Example

Suppose you want a report that displays:

- Account Name
- Case Number
- Case Status

A standard report type may not provide the exact combination you need.

You can create a Custom Report Type:

**Accounts With Cases**

and generate reports using both Account and Case information.

# Navigation

```text
Setup
→ Analytics
→ Reports & Dashboards
→ Report Types
```

Click:

```text
New Custom Report Type
```

---

# Step 1: Define the Custom Report Type

## Select Primary Object

Choose the main object for reporting.

Example:

```text
Primary Object = Accounts
```

The primary object becomes the base object of the report.

---

## Enter Details

Fill the following fields:

| Field | Value |
|---------|---------|
| Display Label | Account With Cases |
| API Name | Account_With_Cases |
| Description | Account With Cases |
| Store in Category | Accounts & Contacts |

### Description Example

```text
Custom report type that allows users to generate reports using Accounts and related Cases.
```

---

# Step 2: Set Availability

Salesforce provides two deployment options.

## In Development

- Visible only to administrators.
- Not available to end users.

## Deployed

- Available to all users who have report access.

Select:

```text
Deployed
```

Click:

```text
Next
```

---

# Step 3: Define Report Records Set

Now define relationships between objects.

Example:

```text
Accounts
    ↓
Cases
```

Click:

```text
Click to relate another object
```

Select:

```text
Cases
```

---

## Relationship Options

### Option 1

```text
Each "A" record must have at least one related "B" record
```

Meaning:

```text
Accounts WITH Cases only
```

Example:

| Account | Cases |
|----------|----------|
| ABC Corp | 3 |
| XYZ Ltd | 1 |

Accounts without cases are excluded.

---

### Option 2

```text
"A" records may or may not have related "B" records
```

Meaning:

```text
Accounts WITH or WITHOUT Cases
```

Example:

| Account | Cases |
|----------|----------|
| ABC Corp | 3 |
| XYZ Ltd | 0 |

All accounts appear.

---

For this example select:

```text
"A" records may or may not have related "B" records
```

Click:

```text
Save
```

---

# Custom Report Type Created

Example Relationship:

```text
Accounts (A)
with or without related records from
Cases (B)
```

# Advantages of Custom Report Types

### 1. Custom Relationships

Create reports using multiple related objects.

### 2. Better Reporting Flexibility

Include only the required records.

### 3. More Fields Available

Expose additional fields not available in standard report types.

### 4. Improved User Experience

Users can create reports without complex filtering.

### 5. Business-Specific Reporting

Design reports according to business requirements.

---

# Standard Report Type vs Custom Report Type

| Feature | Standard Report Type | Custom Report Type |
|----------|----------|----------|
| Created by Salesforce | ✅ | ❌ |
| Custom Relationships | ❌ | ✅ |
| Add Custom Objects | Limited | ✅ |
| Control Record Visibility | ❌ | ✅ |
| Add Related Objects | Limited | ✅ |
| Business-Specific Reports | Limited | ✅ |

# Interview Questions

## What is a Custom Report Type in Salesforce?

A Custom Report Type defines the objects, relationships, and fields that users can use when creating reports.

---

## Why do we use Custom Report Types?

To create reports that are not possible using standard report types and to control which records and fields are available in reports.

---

## What is the difference between Standard and Custom Report Types?

Standard Report Types are provided by Salesforce, whereas Custom Report Types are created by administrators to meet specific business reporting requirements.

---

## What relationship options are available?

1. Each "A" record must have at least one related "B" record.
2. "A" records may or may not have related "B" records.

---

## Can we create reports using multiple objects?

Yes. Custom Report Types allow reports to be created using multiple related objects.

---

# Conclusion

Custom Report Types are a powerful Salesforce reporting feature that allows administrators to define:

- Objects
- Relationships
- Fields
- Record visibility

for report creation.

They provide greater flexibility than Standard Report Types and help organizations create reports tailored to specific business requirements.
