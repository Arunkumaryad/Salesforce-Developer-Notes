# Different Types of Reports in Salesforce | Tabular, Summary, Matrix, Joined Reports

## Introduction

Salesforce provides multiple report formats to analyze data in different ways. Depending on your business requirements, you can choose between:

1. Tabular Reports
2. Summary Reports
3. Matrix Reports
4. Joined Reports

Each report type serves a different purpose and helps users organize, summarize, and visualize data efficiently.

---

# 1. Tabular Reports

## Definition

**Tabular Reports are the simplest and fastest way to look at your data. Similar to a spreadsheet, they consist simply of an ordered set of fields in columns, with each matching record listed in a row.**

While easy to set up, they cannot be used to create groups of data. Consequently, they are best used for tasks like generating a mailing list.

---

## Features of Tabular Reports

### 1. Simple Layout

Data is displayed in rows and columns.

### 2. Fastest Report Type

Quick to create and easy to understand.

### 3. No Grouping

Records cannot be grouped into categories.

### 4. Suitable for Lists

Useful for:

- Mailing Lists
- Contact Lists
- Student Lists
- Product Lists

### 5. Chart Support

Tabular Reports can be used to generate:

- Horizontal Bar Chart
- Vertical Bar Chart
- Pie Chart

But cannot be used for:

- Scatter Chart
- Gauge Chart
- Metric Chart

---

## Example

| Account Name | Type | Rating |
|-------------|------|---------|
| Tech School | Customer | Warm |
| Tech School1 | Customer | Warm |
| Test Demo | Prospect | Cold |

This is a simple list of records without grouping.

---

## How to Create a Tabular Report

### Step 1: Open Reports Tab

Navigate to:

```text
Reports → New Report
```

### Step 2: Select Report Type

Example:

```text
Accounts
```

Click:

```text
Start Report
```

### Step 3: Choose Fields

Select fields such as:

- Account Name
- Account Owner
- Type
- Rating
- Last Modified Date

### Step 4: Run the Report

Click:

```text
Run
```

### Result

A simple report displaying records in tabular format.

---

## Use Cases

- Customer lists
- Student records
- Exporting data to Excel
- Mailing lists
- Simple record viewing

---

# 2. Summary Reports

## Definition

**Summary Reports allow users to group rows of data, view subtotals, create charts, and conditional highlighting.**

---

## Features of Summary Reports

### 1. Grouping Rows

Records can be grouped based on fields.

Example:

```text
Group Accounts by Type
```

---

### 2. Summarize Data

Allows viewing:

- Sum
- Average
- Minimum
- Maximum
- Median

Example:

```text
Total Annual Revenue by Account Type
```

---

### 3. Subgroups

Multiple levels of grouping can be created.

Example:

```text
Type
    → Rating
```

---

### 4. Conditional Highlighting

Available only for:

- Summary Reports
- Matrix Reports

Used only on summarized fields.

---

### 5. Add Charts

Charts can be added directly to reports.

Supported charts include:

- Bar Charts
- Pie Charts
- Line Charts
- Funnel Charts

---

## How to Create a Summary Report

### Step 1: Create an Account Report

```text
Reports → New Report → Accounts
```

### Step 2: Group Records

Drag a field into:

```text
GROUP ROWS
```

Example:

```text
Type
```

---

### Step 3: Add Another Group

Example:

```text
Rating
```

This creates:

```text
Type
   └── Rating
```

---

### Step 4: Add Summaries

Select a numeric field.

Example:

```text
Annual Revenue
```

Choose:

```text
Summarize
```

Then select:

- Sum
- Average
- Min
- Max
- Median

---

### Step 5: Run Report

Click:

```text
Run
```

---

## Example Output

```text
Type
 ├─ Customer
 │   Revenue: $50,000
 │
 ├─ Prospect
 │   Revenue: $20,000
 │
 └─ Partner
     Revenue: $10,000
```

---

## Use Cases

- Sales analysis
- Revenue reports
- Student grouping by department
- Employee grouping by team

---

# 3. Matrix Reports

## Definition

**Matrix Reports allow users to group data in rows and columns.**

---

## Features

### All Features of Summary Reports

Matrix Reports support everything available in Summary Reports, including:

- Row Grouping
- Column Grouping
- Subtotals
- Charts
- Conditional Highlighting
- Summary Calculations

---

## Additional Capability

Data can be grouped in both directions:

### Row Group

Example:

```text
Type
```

### Column Group

Example:

```text
Annual Revenue
```

---

## How to Create a Matrix Report

### Step 1: Create a Summary Report

Group records by:

```text
Type
```

---

### Step 2: Add Column Group

Drag another field into:

```text
GROUP COLUMNS
```

Example:

```text
Annual Revenue
```

---

### Step 3: Run Report

Click:

```text
Run
```

---

## Example Output

| Type | Low Revenue | Medium Revenue | High Revenue |
|--------|------------|---------------|--------------|
| Customer | 10 | 5 | 2 |
| Prospect | 8 | 3 | 1 |
| Partner | 4 | 2 | 0 |

---

## Benefits

- Cross-analysis
- Trend comparison
- Multi-dimensional reporting
- Better business insights

---

## Use Cases

- Revenue analysis by region
- Student count by department and year
- Opportunity analysis by stage and owner

---

# 4. Joined Reports

## Definition

**Joined Reports let you create different views of data from multiple report types. In a joined report, data is organized in blocks. Each block acts like a sub-report, with its own fields, columns, sorting, and filtering. You can add a chart to a joined report.**

**Maximum number of report blocks allowed: 5**

---

## Features

### 1. Add Report Types

Multiple report types can be combined.

Example:

```text
Accounts
+
Opportunities
```

---

### 2. Summarize Data

Each block can contain summaries and calculations.

---

### 3. Add Charts

Charts can be added to visualize combined data.

---

### 4. Independent Blocks

Each block has:

- Its own fields
- Filters
- Sorting
- Grouping

---

### 5. Multiple Data Sources

Analyze related objects together.

Example:

```text
Accounts
Opportunities
Cases
Contacts
```

---

## How to Create a Joined Report

### Step 1: Create New Report

```text
Reports → New Report
```

Select:

```text
Accounts
```

---

### Step 2: Change Format

Select:

```text
Joined Report
```

---

### Step 3: Add Block

Click:

```text
Add Block
```

---

### Step 4: Select Another Report Type

Example:

```text
Opportunities
```

---

### Step 5: Configure Fields

Each block can have different columns and filters.

---

### Step 6: Run Report

Click:

```text
Run
```

---

## Example Structure

```text
Block 1 → Accounts

Account Name
Type
Rating

--------------------------------

Block 2 → Opportunities

Opportunity Name
Amount
Stage
```

---

## Use Cases

- Accounts and Opportunities analysis
- Cases and Contacts reporting
- Multi-object business reporting
- Executive dashboards

---

| Charts | Limited | ✅ | ✅ | ✅ |
| Conditional Highlighting | ❌ | ✅ | ✅ | ❌ |
| Multiple Report Types | ❌ | ❌ | ❌ | ✅ |

# When to Use Which Report?

### Use Tabular Report When

- You need a simple list of records.
- You want to export data.
- No grouping is required.

### Use Summary Report When

- You need grouped records.
- You need subtotals.
- You want charts.

### Use Matrix Report When

- You need row and column grouping.
- You want cross-analysis.

### Use Joined Report When

- You need multiple report types.
- You want different data blocks in a single report.

---

# Interview Questions

## What is a Tabular Report?

A Tabular Report is the simplest report format that displays data in rows and columns without grouping.

---

## What is a Summary Report?

A Summary Report groups records and provides subtotals, summaries, charts, and conditional highlighting.

---

## What is a Matrix Report?

A Matrix Report groups data by both rows and columns for advanced analysis.

---

## What is a Joined Report?

A Joined Report combines data from multiple report types into separate blocks within a single report.

---

## How Many Blocks Can Be Added in a Joined Report?

Maximum:

```text
5 Blocks
```

---

## Which Report Supports Conditional Highlighting?

- Summary Reports
- Matrix Reports

---

# Summary

Salesforce provides four report formats to meet different reporting requirements. **Tabular Reports** display simple lists of records, **Summary Reports** group records and provide subtotals, **Matrix Reports** allow grouping by both rows and columns, and **Joined Reports** combine multiple report types into separate report blocks. Choosing the correct report type helps users analyze business data more effectively and create meaningful dashboards and insights.
