# Salesforce Dashboards for Tabular, Summary, Matrix & Joined Reports

## Overview

### What is a Dashboard?

A Dashboard is the graphical representation of the data generated by a report (or multiple reports) at a certain point in time.

A dashboard provides a visual snapshot of business data using charts, graphs, gauges, metrics, and tables, helping users analyze information quickly.

### Important Note

A dashboard is a snapshot of data.

To obtain the latest data, dashboards must be refreshed either:

- Manually
- Automatically using a scheduled refresh

---

# Why Use Dashboards?

Dashboards help users:

- Visualize report data graphically.
- Monitor KPIs and business performance.
- Analyze trends and patterns.
- Compare multiple reports in a single screen.
- Make faster business decisions.

---

# Dashboard and Report Relationship

```text
Reports
    ↓
Dashboard Components (Widgets)
    ↓
Dashboard
```

A dashboard cannot exist without at least one source report.

---

# Dashboard Types Based on Reports

Salesforce dashboards can be created using:

## 1. Dashboard for Summary Report

Summary Reports support:

- Vertical Bar Charts
- Horizontal Bar Charts
- Pie Charts
- Donut Charts
- Funnel Charts
- Line Charts
- Gauge Charts
- Metric Components

---

## 2. Dashboard for Matrix Report

Matrix Reports support:

- Vertical Bar Charts
- Horizontal Bar Charts
- Pie Charts
- Donut Charts
- Funnel Charts
- Gauge Charts
- Metrics
- Advanced Comparisons

---

## 3. Dashboard for Joined Report

Joined Reports support dashboard components and allow data from multiple report blocks to be displayed together.

---

## 4. Dashboard for Tabular Report

Tabular Reports can be used to generate:

- Horizontal Bar Chart
- Vertical Bar Chart
- Pie Chart

### Limitation

Tabular Reports cannot be used for:

- Scatter Charts
- Gauge Charts
- Metric Components

---

# Dashboard Components (Widgets)

Salesforce dashboards are made up of widgets.

Available dashboard components include:

| Component | Purpose |
|------------|----------|
| Horizontal Bar Chart | Compare values horizontally |
| Vertical Bar Chart | Compare values vertically |
| Pie Chart | Show percentage distribution |
| Donut Chart | Display proportions visually |
| Line Chart | Show trends over time |
| Gauge | Display progress toward target |
| Metric | Show a single KPI value |
| Funnel | Show process stages |
| Table | Display report data in tabular form |
| Scatter Chart | Compare two variables |

# Creating a Dashboard

## Navigation

```text
App Launcher
    ↓
Dashboards
```

or

```text
Dashboards Tab
```

Click:

```text
New Dashboard
```

---

# Step 1: Create Dashboard

Provide:

### Dashboard Name

Example:

```text
Test Dashboards
```

### Description

Optional

### Folder

Example:

```text
My Dashboards
```

Click:

```text
Create
```

---

# Step 2: Add Dashboard Widget

Click:

```text
+ Widget
```

Salesforce opens:

```text
Add Widget
```

window.

---

# Step 3: Select Source Report

Choose the report that will supply dashboard data.

Example:

```text
New Test Account Reports
```

---

# Step 4: Choose Display Type

Available options:

- Horizontal Bar Chart
- Vertical Bar Chart
- Pie Chart
- Donut Chart
- Gauge
- Metric
- Funnel
- Scatter Chart
- Table

Salesforce displays a preview before adding the widget.

---

# Step 5: Add Widget

Click:

```text
Add
```

The component appears on the dashboard canvas.

---

# Step 6: Add Multiple Widgets

A dashboard can contain components from multiple reports.

Example:

| Report | Component |
|----------|------------|
| New Test Account Reports | Horizontal Bar Chart |
| New Matrix Reports | Vertical Bar Chart |
| New Test Account Reports | Donut Chart |

This allows users to compare information from different reports.

---

# Step 7: Save Dashboard

Click:

```text
Save
```

Then click:

```text
Done
```

The dashboard becomes available for users.

# Dynamic Dashboard

## What is a Dynamic Dashboard?

A Dynamic Dashboard displays data based on the currently logged-in user.

Different users see different data without creating multiple dashboards.

---

## Dashboard Viewing Options

### Run as a Specified User

```text
Dashboard always shows data
for one selected user.
```

Example:

```text
Sales Manager
```

Every viewer sees the same data.

---

### Run as Logged-In User

```text
Dashboard displays data
based on the logged-in user.
```

Every user sees only the records they have access to.

This is known as a:

```text
Dynamic Dashboard
```

---

# Configure Dynamic Dashboard

Open dashboard.

Click:

```text
Edit
```

Click:

```text
Dashboard Properties (⚙️)
```

Under:

```text
View Dashboard As
```

Choose:

### Me

Dashboard runs as current owner.

---

### Another Person

Dashboard runs as selected user.

---

### The Dashboard Viewer

Dashboard runs as logged-in user.

This creates a:

```text
Dynamic Dashboard
```

Optional:

```text
Let dashboard viewers choose
whom they view the dashboard as
```

# Interview Questions

## What is a Dashboard in Salesforce?

A Dashboard is a graphical representation of data generated from one or more reports.

---

## Why is a Dashboard called a Snapshot?

Because it displays report data as it existed at the last refresh time.

---

## Can a Dashboard use multiple reports?

Yes.

A dashboard can contain multiple widgets from different reports.

---

## What is a Dynamic Dashboard?

A dashboard that displays data according to the logged-in user's access permissions.

---

## Which report type is required for Gauge and Metric components?

Summary Report or Matrix Report.

---

## Can Tabular Reports create Gauge Charts?

No.

Tabular Reports cannot create:

- Gauge Charts
- Metric Components
- Scatter Charts

---

## How do you make a Dashboard Dynamic?

Dashboard Properties → View Dashboard As → The Dashboard Viewer.

---

## Can Joined Reports be used in Dashboards?

Yes.

Dashboard components can be created using Joined Reports.
