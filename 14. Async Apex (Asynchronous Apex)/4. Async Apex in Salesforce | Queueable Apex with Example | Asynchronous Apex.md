# Async Apex in Salesforce | Queueable Apex with Example | Asynchronous Apex

## Overview

Queueable Apex is an asynchronous Apex framework that allows developers to run background jobs in Salesforce.

It is considered an enhanced version of Future Methods because it provides additional capabilities such as:

- Job monitoring using Job ID
- Passing non-primitive data types
- Chaining Queueable Jobs

Queueable Apex combines the simplicity of Future Methods with the flexibility of Batch Apex.

---

# What is Queueable Apex?

### Actual Definition (From Notes)

> Queueable Apex is essentially a superset of Future methods.

---

## Simple Definition

Queueable Apex is an asynchronous Apex feature that allows code to run in the background and provides more functionality than Future Methods.

It is useful when you need:

- Background processing
- Job monitoring
- Chaining jobs
- Passing complex objects

---

# Queueable Apex vs Future Method

Queueable jobs are similar to Future Methods because both are executed asynchronously.

However, Queueable Apex provides several additional benefits.

---

# Advantages Over Future Methods

## 1. Getting an ID for Your Job

### Actual Definition (From Notes)

> When you submit your job by invoking the System.enqueueJob method, the method returns the ID of the new job.

> This ID corresponds to the ID of the AsyncApexJob record.

> Use this ID to identify and monitor your job, either through the Salesforce UI (Apex Jobs page), or programmatically by querying your record from AsyncApexJob.

---

### Example

```apex
Id jobId = System.enqueueJob(jobObject);
```

The returned Job ID can be used to:

- Monitor job status
- Check execution progress
- Query AsyncApexJob object

---

# 2. Using Non-Primitive Types

### Actual Definition (From Notes)

> Your queueable class can contain member variables of non-primitive data types, such as sObjects or custom Apex types.

> Those objects can be accessed when the job executes.

---

## Future Method Limitation

Future methods only support:

```apex
Id
String
Integer
Boolean
List<Id>
Set<Id>
```

---

## Queueable Advantage

Queueable Apex supports:

```apex
sObjects
Custom Apex Classes
Complex Objects
Collections
```

---

# 3. Chaining Jobs

### Actual Definition (From Notes)

> You can chain one job to another job by starting a second job from a running job.

> Chaining jobs is useful if your process depends on another process to have run first.

---

## Example

```text
Job 1
 ↓
Job 2
 ↓
Job 3
```

Each Queueable Job can enqueue another Queueable Job.

---

# When Should We Use Queueable Apex?

Use Queueable Apex when:

- You need Job IDs.
- You need to pass sObjects.
- You need to chain jobs.
- You need background processing.

# Queueable Apex Example

## Queueable Class

```apex
public class EnqueApexTest
implements Queueable {

    public void execute(
        QueueableContext context
    ) {

        List<Lead> leadRecord =
        [
            SELECT Id,
                   FirstName,
                   LastName,
                   Status,
                   Company
            FROM Lead
        ];

        for(Lead rec : leadRecord) {

            rec.Company = 'HIC';

        }

        update leadRecord;

    }

}
```

# Chaining Queueable Jobs

One Queueable Job can start another Queueable Job.

This is called:

```text
Queueable Chaining
```

---

# First Queueable Class

```apex
public class EnqueApexTest
implements Queueable {

    public void execute(
        QueueableContext context
    ) {

        List<Lead> leadRecord =
        [
            SELECT Id,
                   FirstName,
                   LastName,
                   Status,
                   Company
            FROM Lead
        ];

        for(Lead rec : leadRecord) {

            rec.Company = 'cloud';

        }

        update leadRecord;

        // Call another Queueable Job

        EnqueApexTest2 obj =
        new EnqueApexTest2();

        Id jobId =
        System.enqueueJob(obj);

    }

}
```

---

# Second Queueable Class

```apex
public class EnqueApexTest2
implements Queueable {

    public void execute(
        QueueableContext context
    ) {

        List<Lead> leadList =
        [
            SELECT Id,
                   FirstName,
                   LastName,
                   Status,
                   Company
            FROM Lead
        ];

        for(Lead rec : leadList) {

            rec.Company = 'cloud - NEW';

        }

        update leadList;

    }

}
```

# How Chaining Works

### First Job

```apex
System.enqueueJob(
    new EnqueApexTest()
);
```

---

### Inside execute()

```apex

    new EnqueApexTest2()
);
```

is called.

### Salesforce Automatically

```text

      ↓
```
