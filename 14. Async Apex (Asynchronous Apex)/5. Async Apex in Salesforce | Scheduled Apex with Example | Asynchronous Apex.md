# Async Apex in Salesforce | Scheduled Apex with Example | Asynchronous Apex

## Overview

Scheduled Apex allows you to run Apex code automatically at a specific time or on a recurring schedule.

It is useful when a business process needs to execute without user intervention, such as:

- Daily data updates
- Weekly reports
- Monthly cleanup jobs
- Scheduled integrations
- Automated notifications

Scheduled Apex runs in the background according to a Cron Expression.

---

# What is Scheduled Apex?

### Actual Definition (From Notes)

> Scheduled Apex provides you an ability to run Apex code at specific times or periodically.

> You can only have 100 scheduled Apex jobs at one time.

---

## Simple Definition

Scheduled Apex is an asynchronous Apex feature that allows developers to schedule Apex classes to run automatically at a specific date and time or at recurring intervals.

---

# Real-Life Example

Suppose a company wants to:

- Update Lead records every day at 2:30 AM
- Generate reports every Monday
- Send reminder emails every Friday

Instead of manually running code, Salesforce can execute it automatically using Scheduled Apex.

# Scheduled Apex Example

## Apex Class

```apex
public class ScheduleExample1
implements Schedulable {

    public void execute(
        SchedulableContext context
    ) {

        List<Lead> leadList =
        [
            SELECT Id,
                   FirstName,
                   LastName,
                   Status,
                   Company,
                   LeadSource
            FROM Lead
        ];

        for(Lead rec : leadList) {

            rec.LeadSource = 'Web';

        }

        update leadList;
    }

}
```

---

# Code Explanation

## Step 1: Fetch Leads

```apex
List<Lead> leadList =
[
    SELECT Id,
           FirstName,
           LastName,
           Status,
           Company,
           LeadSource
    FROM Lead
];
```

Retrieves Lead records from Salesforce.

---

## Step 2: Update LeadSource

```apex
for(Lead rec : leadList) {

    rec.LeadSource = 'Web';

}
```

Updates the LeadSource field for each Lead.

---

## Step 3: Save Changes

```apex
update leadList;
```

Updates all modified records.

# Cron Expression

Salesforce uses Cron Expressions to determine when a job should run.

### Example

```apex
String cronExp =
'0 0 13 * * ? 2024';
```

---

# Cron Expression Format

```text
Seconds Minutes Hours Day_of_Month Month Day_of_Week Optional_Year
```

---

| Position | Name | Values |
|-----------|---------|---------|
| 1 | Seconds | 0-59 |
| 2 | Minutes | 0-59 |
| 3 | Hours | 0-23 |
| 4 | Day of Month | 1-31 |
| 5 | Month | 1-12 or JAN-DEC |
| 6 | Day of Week | 1-7 or SUN-SAT |
| 7 | Optional Year | 1970-2099 |

---

# Special Characters in Cron Expressions

## 1. Comma (,)

### Actual Definition (From Notes)

> Give multiple values.

### Example

```text
JAN,MAR,APR
```

Runs in January, March, and April.

---

## 2. Hyphen (-)

### Actual Definition (From Notes)

> Specifies a range.

### Example

```text
JAN-MAR
```

Runs from January through March.

---

## 3. Asterisk (*)

### Actual Definition (From Notes)

> Specifies all values.

### Example

```text
*
```

Represents every possible value.

---

## 4. Question Mark (?)

### Actual Definition (From Notes)

> Specifies no specific value.

### Notes

Used only in:

- Day of Month
- Day of Week

---

## 5. Slash (/)

### Actual Definition (From Notes)

> Specifies increments.

### Example

```text
1/5
```

Means every fifth day starting from day 1.

```text
2/5
```

Means every fifth day starting from day 2.

---

## 6. L

### Actual Definition (From Notes)

> Specifies the end of a range (last).

### Examples

```text
L
```

Last day of month.

```text
6L
```

Last Friday of month.

```text
2L
```

Last Monday of month.

---

## 7. W

### Actual Definition (From Notes)

> Specifies the nearest weekday (Monday-Friday) of the given day.

### Examples

```text
20W
```

Nearest weekday to the 20th day of the month.

```text
1W
```

Nearest weekday to the 1st day of the month.

---

## 8. #

### Actual Definition (From Notes)

> Specifies the nth day of the month.

### Example

```text
2#1
```

First Monday of every month.

Format:

```text
weekday#occurrence
```

---

# Query Scheduled Jobs

You can query the CronTrigger object.

## Example

```sql
SELECT Id,
       CronJobDetailId,
       NextFireTime,
       PreviousFireTime,
       State,
       StartTime,
       EndTime,
       CronExpression
FROM CronTrigger
WHERE Id = 'JOB_ID'
```

# Scheduled Apex vs Queueable Apex

| Feature | Scheduled Apex | Queueable Apex |
|----------|---------------|---------------|
| Runs at Specific Time | Yes | No |
| Runs Immediately | No | Yes |
| Uses Cron Expression | Yes | No |
| Supports Scheduling | Yes | No |
| Background Processing | Yes | Yes |
