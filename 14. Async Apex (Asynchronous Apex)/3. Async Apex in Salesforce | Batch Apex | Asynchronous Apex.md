# Async Apex in Salesforce | Batch Apex | Asynchronous Apex

## Overview

Batch Apex is used when you need to process a very large number of records that cannot be handled efficiently in a single transaction.

Instead of processing all records at once, Salesforce divides records into smaller chunks called **batches** and processes each batch separately.

Batch Apex is one of the most powerful asynchronous processing mechanisms in Salesforce.

---

# What is Batch Apex?

### Actual Definition (From Notes)

> Batch Apex can be used to build complex, long running processes that run on thousands of records on the Lightning platform.

> It operates over small batches/chunks of records.

> Also it can be scheduled.

---

## Simple Definition

Batch Apex is an asynchronous Apex feature that processes large volumes of records by dividing them into smaller chunks and executing them separately.

This helps avoid governor limit issues and improves performance.

---

# Why Do We Need Batch Apex?

Suppose you need to update:

```text
500
Records
```

A normal Apex transaction can handle it.

But if you need to update:

```text
500,000
Records
```

or

```text
Millions of Records
```

then Batch Apex should be used.

---

# Advantages of Batch Apex

### 1. Handles Large Data Volumes

```text
Thousands
Millions
```

of records can be processed.

---

### 2. Runs Asynchronously

Runs in the background without blocking users.

---

### 3. Fresh Governor Limits

Each batch execution gets a new set of governor limits.

---

### 4. Can Be Scheduled

Can run automatically at a future time.

---

### 5. Better Error Handling

If one batch fails, other batches can still continue.

# How to Implement Batch Apex

### Actual Definition (From Notes)

1. We need to implement `Database.Batchable` interface in Apex class.

2. Also, we need to implement:

```apex
start()
execute()
finish()
```

methods of this interface.

---

# Batch Apex Architecture

```text
Batch Class
      │
      ▼
start()
      │
      ▼
Records Retrieved
      │
      ▼
execute()
      │
      ▼
Process Batch 1

execute()
      │
      ▼
Process Batch 2

execute()
      │
      ▼
Process Batch N

      ▼
finish()
      │
      ▼
Post Processing
```

# 1. start() Method

### Actual Definition (From Notes)

- Executes only once
- Return batch scope or records
- Can return upto 50 million records

---

## Purpose

The start() method identifies which records need to be processed.

---

### Syntax

```apex
public Database.QueryLocator start(
    Database.BatchableContext BC
) {

}
```

---

### Example

```apex
public Database.QueryLocator start(
    Database.BatchableContext BC
) {

    return Database.getQueryLocator(
        'SELECT Id, Name FROM Lead'
    );

}
```

---

# Important Note

### Maximum Records Returned

```text
50 Million Records
```

can be returned by QueryLocator.

This is one of the biggest advantages of Batch Apex.

---

# 2. execute() Method

### Actual Definition (From Notes)

> Executes for each batch/chunk of records.

---

## Purpose

This method contains the actual business logic.

It processes records batch by batch.

---

### Syntax

```apex
public void execute(
    Database.BatchableContext BC,
    List<SObject> scope
) {

}
```

---

# Notes (From Screenshot)

### execute() Method

- Must have void return type.
- Each execution will get new set of governor limits.

---

# Example

```apex
public void execute(
    Database.BatchableContext BC,
    List<Lead> scope
) {

    for(Lead record : scope) {

        record.Company = 'Jio';

    }

    update scope;

}
```

---

# Fresh Governor Limits Example

Suppose:

```text
Batch Scope = 2000 Records
Batch Size  = 500
```

Then:

```text
2000 ÷ 500 = 4
```

So execute() will run:

```text
4 Times
```

---

### Execution Flow

```text
Batch 1 → 500 Records
Batch 2 → 500 Records
Batch 3 → 500 Records
Batch 4 → 500 Records
```

Each batch gets:

```text
Fresh SOQL Limits
Fresh DML Limits
Fresh CPU Limits
```

This is why Batch Apex can process huge datasets.

# 3. finish() Method

### Actual Definition (From Notes)

- Executes only once
- Called after all batches are processed
- Can be used for post processing

---

## Purpose

The finish() method runs after all execute() calls complete successfully.

---

### Syntax

# Complete Batch Apex Example

### Batch Class

```apex
public class BatchApexTest
implements Database.Batchable<SObject> {

    public Database.QueryLocator start(
        Database.BatchableContext BC
    ) {

        return Database.getQueryLocator(
            'SELECT Id,
                    FirstName,
                    LastName,
                    Status,
                    Company
             FROM Lead'
        );
    }

    public void execute(
        Database.BatchableContext BC,
        List<Lead> scope
    ) {

        System.debug('Inside Execute Methods');

        for(Lead record : scope) {

            record.Company = 'Jio';

        }

        update scope;
    }

    public void finish(
        Database.BatchableContext BC
    ) {

        System.debug('Batch Finished');

    }
}
```

# Summary

### Batch Apex

- Processes large datasets asynchronously.
- Handles millions of records.
- Operates on small batches/chunks.
- Can be scheduled.

### Required Interface

```apex
Database.Batchable<SObject>
```

### Required Methods

```apex
start()

execute()

finish()
```

### Method Behavior

| Method | Executes |
|----------|----------|
| start() | Once |
| execute() | For Every Batch |
| finish() | Once |

### Batch Sizes

```text
Minimum = 1
Default = 200
Maximum = 2000
```

### Key Benefit

Each execute() invocation receives a new set of governor limits, making Batch Apex ideal for processing large amounts of data efficiently.
