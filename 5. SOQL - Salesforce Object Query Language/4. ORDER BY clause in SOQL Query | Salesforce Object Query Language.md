# ORDER BY clause in SOQL Query | Salesforce Object Query Language

# Introduction

ORDER BY clause in SOQL is used to sort query results.

We can sort records:

- In Ascending Order
- In Descending Order

It is mainly used for arranging records in a readable format.

---

# Definition

ORDER BY clause is used to sort the records returned by a SOQL query based on one or more fields.

---

# Basic Syntax

```sql
SELECT Field1, Field2, Field3
FROM ObjectName
ORDER BY FieldName
```

---

# Default Sorting

By default, ORDER BY sorts records in:

```text
Ascending Order (ASC)
```

---

# Example 1 : ORDER BY Default (Ascending)

```sql
SELECT Id, Name, Branch__c, Roll_Number__c
FROM Student1__c
ORDER BY Roll_Number__c
```

---

# Explanation

This query sorts student records based on Roll Number in ascending order.

Result:

```text
1 → 23 → 87 → 76542
```

---

# ASC Keyword

ASC keyword is used for ascending order sorting.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
ORDER BY FieldName ASC
```

---

# Example 2 : ORDER BY ASC

```sql
SELECT Id, Name, Branch__c, Roll_Number__c
FROM Student1__c
ORDER BY Roll_Number__c ASC
```

---

# Explanation

This query explicitly sorts Roll Numbers in ascending order.

Result:

```text
1 → 23 → 87 → 76542
```

---

# DESC Keyword

DESC keyword is used for descending order sorting.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
ORDER BY FieldName DESC
```

---

# Example 3 : ORDER BY DESC

```sql
SELECT Id, Name, Branch__c, Roll_Number__c
FROM Student1__c
ORDER BY Roll_Number__c DESC
```

---

# Explanation

This query sorts records in descending order.

Result:

```text
76542 → 87 → 23 → 1
```

---

# ORDER BY with Standard Object

```sql
SELECT Id, Name, FirstName, City
FROM Lead
ORDER BY City
```

---

# Explanation

This query sorts Lead records based on City name.

---

# NULLS LAST

# Definition

NULLS LAST places records with null values at the end of query results.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
ORDER BY FieldName NULLS LAST
```

---

# Example 4 : NULLS LAST

```sql
SELECT Id, Name, FirstName, City
FROM Lead
ORDER BY City NULLS LAST
```

---

# Explanation

Records having NULL City values will appear at the bottom.

---

# NULLS FIRST

# Definition

NULLS FIRST places records with null values at the beginning of query results.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
ORDER BY FieldName NULLS FIRST
```

---

# Example 5 : NULLS FIRST

```sql
SELECT Id, Name, FirstName, City
FROM Lead
ORDER BY City NULLS FIRST
```

---

# Explanation

Records having NULL City values will appear at the top.

---

# Features of ORDER BY Clause

| Feature | Description |
|---|---|
| Sorting | Sorts records systematically |
| ASC Support | Ascending sorting |
| DESC Support | Descending sorting |
| NULL Handling | Supports NULLS FIRST and NULLS LAST |
| Readability | Makes data easy to understand |

---

# Advantages of ORDER BY

| Advantage | Description |
|---|---|
| Better Data Presentation | Records appear in proper order |
| Easy Analysis | Helps analyze data quickly |
| Supports Numeric Sorting | Useful for IDs, Salary, Roll Numbers |
| Supports Text Sorting | Useful for Name, City, Status |
| Improves Reporting | Helpful in reports and dashboards |

---

# Real-Time Use Cases

| Scenario | Query Purpose |
|---|---|
| Sort students by Roll Number | ORDER BY ASC |
| Show highest salary first | ORDER BY DESC |
| Display cities alphabetically | ORDER BY City |
| Push empty values to bottom | NULLS LAST |
| Show incomplete records first | NULLS FIRST |

---

# Important Notes

| Point | Description |
|---|---|
| Default Order | ASC |
| DESC | Descending sorting |
| ASC | Ascending sorting |
| NULLS LAST | Null values at bottom |
| NULLS FIRST | Null values at top |

---

# Difference Between ASC and DESC

| ASC | DESC |
|---|---|
| Small to Large | Large to Small |
| A to Z | Z to A |
| Default Sorting | Explicit descending sorting |
