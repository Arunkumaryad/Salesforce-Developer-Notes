# Apex Delete DML Operation | SOQL Delete Query Example

# Introduction

DML (Data Manipulation Language) in Salesforce Apex is used to perform operations on records stored in the database.

The four major DML operations are:

1. Insert
2. Update
3. Delete
4. Undelete

In this tutorial, we will learn the **Delete DML Operation**, which is used to remove records from Salesforce using Apex code.

---

# What is Delete DML?

## Definition

The **Delete DML operation** is used to remove existing records from the Salesforce database.

When a record is deleted:

- The record is removed from the object.
- It is moved to the Salesforce Recycle Bin.
- The record can be restored later using the Undelete operation.
- The record Id remains available while the record is in the Recycle Bin.

---

# Why Use Delete DML?

Delete DML is commonly used when:

- Duplicate records need to be removed.
- Old records are no longer required.
- Incorrect data must be deleted.
- Records need to be cleaned up during data migration.

---

# Syntax of Delete DML

## Delete a Single Record

```apex
ObjectName obj =
[
    SELECT Id
    FROM ObjectName
    LIMIT 1
];

delete obj;
```

---

## Delete Multiple Records

```apex
List<ObjectName> objList =
[
    SELECT Id
    FROM ObjectName
];

delete objList;
```

---

# Example: Delete Teacher Record

The following Apex code retrieves Teacher records and deletes them.

```apex
List<Teacher__c> teacherList =
[
    SELECT Id,
           Name,
           Dept__c
    FROM Teacher__c
    WHERE Name = 'Student 4'
];

delete teacherList;
```

---

# Explanation

## Step 1: Retrieve Record

```apex
List<Teacher__c> teacherList =
[
    SELECT Id,
           Name,
           Dept__c
    FROM Teacher__c
    WHERE Name = 'Student 4'
];
```

This SOQL query retrieves the Teacher record whose Name is **Student 4**.

---

## Step 2: Delete Record

```apex
delete teacherList;
```

The record is deleted from the Teacher object.

# What Happens After Deletion?

Salesforce does not permanently remove the record immediately.

Instead:

```text
Deleted Record
        ↓
Recycle Bin
        ↓
Can be Restored
```

This allows accidental deletions to be recovered.

---

# Salesforce Recycle Bin

After deleting a record:

1. Open App Launcher.
2. Search for **Recycle Bin**.
3. Open Recycle Bin.

The deleted record will appear there.

Example:

| Record Name | Type |
|------------|------|
| Student 4 | Teacher |

---

# Restore Deleted Records

Deleted records can be restored from Recycle Bin.

Steps:

1. Open Recycle Bin.
2. Select the deleted record.
3. Click **Restore**.

The record will be restored to its original object.

---

# Permanently Delete Records

To permanently remove deleted records:

1. Open Recycle Bin.
2. Select the record.
3. Click **Delete**.

OR

```text
Empty Org Recycle Bin
```

This permanently removes records from Salesforce.

# Using Database.delete()

Salesforce provides another way to delete records using the Database class.

---

# Syntax

```apex
Database.delete(records);
```

Example:

```apex
Database.delete(teacherList);
```

---

# Database.delete() with allOrNone

Database methods support partial success.

---

## Syntax

```apex
Database.delete(records, allOrNone);
```

---

## Example

```apex
Database.delete(teacherList, true);
```

or

```apex
Database.delete(teacherList, false);
```

---

# Understanding allOrNone Parameter

## allOrNone = true

```apex
Database.delete(teacherList, true);
```

### Definition

If one record fails deletion, the entire operation is rolled back.

### Result

- All records are deleted.
- OR no records are deleted.

### Partial Success

❌ Not Allowed

---

## allOrNone = false

```apex
Database.delete(teacherList, false);
```

### Definition

Valid records are deleted even if some records fail.

### Result

- Successful records are deleted.
- Failed records remain unchanged.

### Partial Success

✅ Allowed

---

# Example

```apex
List<Teacher__c> teacherList =
[
    SELECT Id,
           Name,
           Dept__c
    FROM Teacher__c
    WHERE Name = 'Student 4'
];

delete teacherList;
// Partial success is not allowed

Database.delete(teacherList, false);
// Partial success is allowed
```

---

# Difference Between delete and Database.delete

| Feature | delete | Database.delete |
|----------|---------|----------------|
| Standard DML | ✅ Yes |
| Database Method | ❌ No | ✅ Yes |
| Partial Success | ❌ No | ✅ Yes |
| allOrNone Support | ❌ No | ✅ Yes |
| Better Error Handling | ❌ Limited | ✅ Available |


---

# Bulk Delete Best Practice

✅ Good Practice

```apex
delete teacherList;
```

---

❌ Bad Practice

```apex
for(Teacher__c teacher : teacherList)
{
    delete teacher;
}
```

Avoid DML operations inside loops.

---

# Governor Limits

Salesforce allows:

```text
150 DML statements per transaction
```

Bad Example:

```apex
for(Teacher__c teacher : teacherList)
{
    delete teacher;
}
```

This may hit governor limits.

---

Good Example:

```apex
delete teacherList;
```

Only one DML statement is executed.

# Important Notes

- Deleted records move to Recycle Bin.
- Records can be restored before permanent deletion.
- Deleting a parent record may affect related child records depending on relationship settings.
- Always query records before deleting them.
- Use bulk delete whenever possible.
- Avoid delete statements inside loops.
