# Aggregate Functions in SOQL Query With GROUP BY and HAVING Clause

# Introduction

Aggregate Functions in SOQL are used to summarize and analyze data instead of returning individual records.

They help us perform calculations such as:

- Average value
- Minimum value
- Maximum value
- Total sum
- Record count
- Distinct value count

Aggregate functions are commonly used in reports, dashboards, analytics, and data summarization queries.

---

# What are Aggregate Functions?

## Definition

**Aggregate Functions in SOQL allow us to roll up and summarize data from multiple records into a single result or grouped results.**

Instead of displaying every record, aggregate functions return calculated values based on a set of records.

---

# Available Aggregate Functions in SOQL

| Function | Description |
|-----------|-------------|
| AVG() | Returns the average value of a numeric field |
| MIN() | Returns the minimum value of a field |
| MAX() | Returns the maximum value of a field |
| SUM() | Returns the total sum of a numeric field |
| COUNT() | Returns the number of rows matching the query criteria |
| COUNT_DISTINCT() | Returns the number of distinct non-null field values |

---

# GROUP BY Clause

## Definition

The **GROUP BY clause** helps aggregate functions work on groups of similar records instead of individual records.

Without GROUP BY:

```sql
SELECT AVG(Amount)
FROM Opportunity
```

Result:

```text
Average of all Opportunity Amount values
```

With GROUP BY:

```sql
SELECT LeadSource,
       AVG(Amount)
FROM Opportunity
GROUP BY LeadSource
```

Result:

```text
Average Amount for each Lead Source
```

---

# HAVING Clause

## Definition

The **HAVING clause** is used with GROUP BY to filter aggregated results.

HAVING works similarly to WHERE, but:

| WHERE | HAVING |
|---------|---------|
| Filters records before grouping | Filters records after grouping |
| Cannot use aggregate functions | Can use aggregate functions |

---

## Example

```sql
SELECT LeadSource,
       AVG(Amount)
FROM Opportunity
GROUP BY LeadSource
HAVING AVG(Amount) >= 100000
```

Only groups whose average amount is greater than or equal to 100000 will be displayed.

# COUNT(FieldName) vs COUNT()

## COUNT()

```sql
SELECT COUNT()
FROM Opportunity
```

Counts all records.

Example:

```text
84
```

---

## COUNT(FieldName)

```sql
SELECT COUNT(LeadSource)
FROM Opportunity
```

Counts only records where LeadSource is not NULL.

Example:

```text
83
```

---

# 6. COUNT_DISTINCT()

## Definition

**COUNT_DISTINCT() returns the number of distinct non-null field values matching the query criteria.**

Duplicate values are counted only once.

---

## Syntax

```sql
SELECT COUNT_DISTINCT(FieldName)
FROM ObjectName
```

---

## Example 1

```sql
SELECT COUNT_DISTINCT(Company)
FROM Lead
```

### Explanation

Returns the total number of unique companies present in Lead records.

---

## Example 2

```sql
SELECT COUNT_DISTINCT(LeadSource)
FROM Opportunity
```

### Explanation

Returns the number of unique Lead Sources available in Opportunity records.

Example Result:

```text
1
```

This means all Opportunities currently belong to one unique Lead Source.

# Aggregate Functions with GROUP BY

## Syntax

```sql
SELECT FieldName,
       AggregateFunction(FieldName)
FROM ObjectName
GROUP BY FieldName
```

Example:

```sql
SELECT LeadSource,
       SUM(Amount)
FROM Opportunity
GROUP BY LeadSource
```

---

# Aggregate Functions with HAVING

## Syntax

```sql
SELECT FieldName,
       AggregateFunction(FieldName)
FROM ObjectName
GROUP BY FieldName
HAVING AggregateFunction(FieldName) Condition
```

Example:

```sql
SELECT LeadSource,
       AVG(Amount)
FROM Opportunity
GROUP BY LeadSource
HAVING AVG(Amount) >= 100000
```

---

# Difference Between WHERE and HAVING

| WHERE | HAVING |
|---------|---------|
| Filters records before grouping | Filters groups after aggregation |
| Cannot use aggregate functions | Can use aggregate functions |
| Applied before GROUP BY | Applied after GROUP BY |

---

## Example Using WHERE

```sql
SELECT LeadSource,
       AVG(Amount)
FROM Opportunity
WHERE Amount > 1000
GROUP BY LeadSource
```

Records are filtered before grouping.

---

## Example Using HAVING

```sql
SELECT LeadSource,
       AVG(Amount)
FROM Opportunity
GROUP BY LeadSource
HAVING AVG(Amount) > 1000
```

Groups are filtered after aggregation.

# Interview Questions

## What are Aggregate Functions in SOQL?

Aggregate functions summarize data and return calculated values such as AVG, SUM, MIN, MAX, COUNT, and COUNT_DISTINCT.

---

## Why is GROUP BY used?

GROUP BY groups similar records so aggregate functions can calculate values for each group.

---

## Why is HAVING used?

HAVING filters grouped results returned by aggregate functions.

---

## Can aggregate functions be used inside WHERE?

No.

Aggregate functions can only be used in HAVING clauses.

---

## What is the difference between COUNT() and COUNT(Field)?

COUNT() counts all records.

COUNT(Field) counts only records where the specified field is not NULL.

---

## What does COUNT_DISTINCT() do?

It returns the number of unique non-null values in a field.

---

# Key Points for Revision

- Aggregate functions summarize data.
- AVG() returns the average value.
- SUM() returns the total sum.
- MAX() returns the highest value.
- MIN() returns the lowest value.
- COUNT() returns total matching records.
- COUNT(Field) ignores NULL values.
- COUNT_DISTINCT() counts unique values.
- GROUP BY groups similar records.
- HAVING filters aggregated results.
- Aggregate functions can be used with GROUP BY.
- Aggregate functions cannot be used in WHERE clauses.
- HAVING is executed after GROUP BY.
- Aggregate queries are heavily used in reports and dashboards.
