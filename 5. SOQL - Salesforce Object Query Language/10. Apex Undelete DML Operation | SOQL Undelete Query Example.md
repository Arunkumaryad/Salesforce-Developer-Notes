# Apex Undelete DML Operation | SOQL Undelete Query Example

# Introduction

DML (Data Manipulation Language) in Salesforce Apex is used to manipulate records stored in the database.

The four major DML operations are:

1. Insert
2. Update
3. Delete
4. Undelete

In this tutorial, we will learn the **Undelete DML Operation**, which is used to restore deleted records from the Salesforce Recycle Bin using Apex.

---

# What is Undelete DML?

## Definition

The **Undelete DML operation** is used to restore records that have been deleted and are currently available in the Salesforce Recycle Bin.

When a record is undeleted:

- The record is restored to its original object.
- All field values are restored.
- Relationships are restored whenever possible.
- The record becomes available again for users.

---

# Why Use Undelete DML?

Undelete is useful when:

- A user accidentally deletes records.
- Important business records need to be restored.
- Records are removed by mistake during data cleanup.
- Deleted records need to be recovered from the Recycle Bin.

---

# How Undelete Works

```text
Record Created
       ↓
Record Deleted
       ↓
Recycle Bin
       ↓
Undelete
       ↓
Record Restored
```

---

# Syntax of Undelete DML

## Restore a Single Record

```apex
undelete recordName;
```

Example:

```apex
Teacher__c teacher =
[
    SELECT Id
    FROM Teacher__c
    WHERE IsDeleted = true
    ALL ROWS
    LIMIT 1
];

undelete teacher;
```

---

## Restore Multiple Records

```apex
List<Teacher__c> teacherList =
[
    SELECT Id
    FROM Teacher__c
    WHERE IsDeleted = true
    ALL ROWS
];

undelete teacherList;
```

---

# Important Keyword: ALL ROWS

Normally SOQL queries do not return deleted records.

To access deleted records, Salesforce provides:

```sql
ALL ROWS
```

---

## Example

```apex
SELECT Id,
       Name
FROM Teacher__c
WHERE IsDeleted = true
ALL ROWS
```

### Explanation

- `IsDeleted = true` returns deleted records.
- `ALL ROWS` allows SOQL to query records from the Recycle Bin.

Without `ALL ROWS`, deleted records cannot be retrieved.

---

# Example: Restore Deleted Teacher Records

Suppose several Teacher records were deleted previously.

The following Apex code restores them.

```apex
List<Teacher__c> undeleteRecord =
[
    SELECT Id,
           Name,
           Dept__c
    FROM Teacher__c
    WHERE IsDeleted = true
    ALL ROWS
];

undelete undeleteRecord;
```

---

# Explanation

## Step 1: Query Deleted Records

```apex
List<Teacher__c> undeleteRecord =
[
    SELECT Id,
           Name,
           Dept__c
    FROM Teacher__c
    WHERE IsDeleted = true
    ALL ROWS
];
```

This query retrieves records that are currently in the Recycle Bin.

---

## Step 2: Restore Records

```apex
undelete undeleteRecord;
```

All records returned by the query are restored.

# Using Database.undelete()

Salesforce also provides the Database class method.

---

## Syntax

```apex
Database.undelete(records);
```

Example:

```apex
Database.undelete(undeleteRecord);
```

---

# Database.undelete() with allOrNone

The Database class supports partial success operations.

---

## Syntax

```apex
Database.undelete(records, allOrNone);
```

---

## Example

```apex
Database.undelete(undeleteRecord, true);
```

or

```apex
Database.undelete(undeleteRecord, false);
```

---

# Understanding allOrNone Parameter

## allOrNone = true

```apex
Database.undelete(undeleteRecord, true);
```

### Definition

If any record fails to restore, the entire operation is rolled back.

### Result

- Either all records are restored.
- Or none of the records are restored.

### Partial Success

❌ Not Allowed

---

## allOrNone = false

```apex
Database.undelete(undeleteRecord, false);
```

### Definition

Valid records are restored even if some records fail.

### Result

- Successful records are restored.
- Failed records remain deleted.

### Partial Success

✅ Allowed

---

# Example

```apex
List<Teacher__c> undeleteRecord =
[
    SELECT Id,
           Name,
           Dept__c
    FROM Teacher__c
    WHERE IsDeleted = true
    ALL ROWS
];

undelete undeleteRecord;
// Partial success is not allowed

Database.undelete(undeleteRecord, false);
// Partial success is allowed
```

---

# Difference Between undelete and Database.undelete

| Feature | undelete | Database.undelete |
|----------|-----------|------------------|
| Standard DML | ✅ Yes |
| Database Method | ❌ No | ✅ Yes |
| Partial Success | ❌ No | ✅ Yes |
| allOrNone Support | ❌ No | ✅ Yes |
| Better Error Handling | ❌ Limited | ✅ Available |

---

# Bulk Undelete Example

```apex
List<Teacher__c> deletedTeachers =
[
    SELECT Id
    FROM Teacher__c
    WHERE IsDeleted = true
    ALL ROWS
];

undelete deletedTeachers;
```

This restores all deleted Teacher records returned by the query.

---

# Governor Limits

Salesforce allows:

```text
150 DML statements per transaction
```

Avoid:

```apex
for(Teacher__c teacher : deletedTeachers)
{
    undelete teacher;
}
```

---

Use:

```apex
undelete deletedTeachers;
```

Only one DML statement is executed.

---

# Best Practices

✅ Query deleted records using `ALL ROWS`.

✅ Use bulk undelete whenever possible.

✅ Use `Database.undelete()` when partial success is required.

✅ Handle errors properly in production code.

❌ Avoid DML operations inside loops.

- Undelete restores records to their original object.
- Use bulk operations for better performance.
- Avoid undelete statements inside loops.
