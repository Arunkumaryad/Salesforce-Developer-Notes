# LIMIT and OFFSET Keywords in SOQL Query | Salesforce Object Query Language

# Introduction

LIMIT and OFFSET are SOQL keywords used to control the number of records returned by a query.

These keywords are commonly used for:

- Pagination
- Displaying limited records
- Improving query performance
- Loading data page by page

---

# LIMIT Keyword

# Definition

LIMIT keyword is used to specify the maximum number of records that should be returned by a SOQL query.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
LIMIT Number
```

---

# Example 1 : LIMIT Keyword

```sql
SELECT Id, FirstName, LastName, Name, CreatedDate
FROM Lead
LIMIT 5
```

---

# Explanation

This query returns only the first 5 Lead records.

Even if the object contains hundreds of records, only 5 records will be displayed.

---

# LIMIT with ORDER BY

LIMIT is frequently used together with ORDER BY to retrieve a specific number of sorted records.

---

# Example 2 : ORDER BY + LIMIT

```sql
SELECT Id, FirstName, LastName, Name, CreatedDate
FROM Lead
ORDER BY Name
LIMIT 5
```

---

# Explanation

This query:

- Sorts records by Name
- Returns only the first 5 records after sorting

---

# LIMIT with Date Field

```sql
SELECT Id, FirstName, LastName, Name, CreatedDate
FROM Lead
ORDER BY CreatedDate
LIMIT 5
```

---

# Explanation

This query returns the oldest 5 records based on CreatedDate.

---

# LIMIT with DESC

```sql
SELECT Id, FirstName, LastName, Name, CreatedDate
FROM Lead
ORDER BY CreatedDate DESC
LIMIT 5
```

---

# Explanation

This query returns the latest 5 records because records are sorted in descending order of CreatedDate.

---

# OFFSET Keyword

# Definition

OFFSET keyword is used to skip a specified number of records before returning query results.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
OFFSET Number
```

---

# Example 3 : OFFSET Keyword

```sql
SELECT Id, FirstName, LastName, Name, CreatedDate
FROM Lead
ORDER BY Name
OFFSET 4
```

---

# Explanation

This query:

- Sorts records by Name
- Skips the first 4 records
- Returns records starting from the 5th record

---

# LIMIT and OFFSET Together

LIMIT and OFFSET are often used together for pagination.

---

# Syntax

```sql
SELECT Field1, Field2
FROM ObjectName
LIMIT Number
OFFSET Number
```

---

# Example 4 : LIMIT + OFFSET

```sql
SELECT Id, FirstName, LastName, Name, CreatedDate
FROM Lead
ORDER BY Name
LIMIT 5
OFFSET 2
```

---

# Explanation

This query:

- Sorts records by Name
- Skips first 2 records
- Returns next 5 records

---

# Pagination Using LIMIT and OFFSET

LIMIT and OFFSET are mainly used for pagination.

---

# Page 1

```sql
SELECT Id, Name
FROM Lead
ORDER BY Name
LIMIT 10
OFFSET 0
```

Returns records:

```text
1 to 10
```

---

# Page 2

```sql
SELECT Id, Name
FROM Lead
ORDER BY Name
LIMIT 10
OFFSET 10
```

Returns records:

```text
11 to 20
```

---

# Page 3

```sql
SELECT Id, Name
FROM Lead
ORDER BY Name
LIMIT 10
OFFSET 20
```

Returns records:

```text
21 to 30
```

---

# Why Use LIMIT?

| Benefit | Description |
|----------|------------|
| Faster Queries | Retrieves fewer records |
| Better Performance | Reduces processing time |
| Data Preview | Shows sample records |
| Dashboard Use | Displays limited data |
| Pagination | Works with OFFSET |

---

# Why Use OFFSET?

| Benefit | Description |
|----------|------------|
| Skip Records | Ignores specified records |
| Pagination | Displays page-wise data |
| Better Navigation | User-friendly record browsing |
| Dynamic Loading | Useful in web applications |

---

# LIMIT vs OFFSET

| LIMIT | OFFSET |
|---------|---------|
| Restricts returned records | Skips records |
| Used to define page size | Used to define starting point |
| Example: LIMIT 5 | Example: OFFSET 5 |
| Returns records | Ignores records |

---

# Real-Time Use Cases

| Scenario | Query Usage |
|-----------|------------|
| Show Top 10 Leads | LIMIT 10 |
| Display Recent Records | ORDER BY CreatedDate DESC LIMIT 10 |
| Pagination | LIMIT + OFFSET |
| Infinite Scrolling | OFFSET with LIMIT |

# Important Notes

| Point | Description |
|---------|------------|
| LIMIT | Restricts returned records |
| OFFSET | Skips records |

# Key Points for Revision

- LIMIT restricts the number of records returned.
- OFFSET skips a specified number of records.
- LIMIT is used to define page size.
- OFFSET is used to define starting position.
- LIMIT and OFFSET are commonly used for pagination.
- ORDER BY should be used with LIMIT and OFFSET for consistent results.
- LIMIT improves query performance.
- OFFSET helps navigate large datasets.
- LIMIT 10 OFFSET 0 = First Page.
- LIMIT 10 OFFSET 10 = Second Page.
- LIMIT 10 OFFSET 20 = Third Page.
- LIMIT and OFFSET are frequently used in Lightning Components, LWC, Visualforce, and custom applications.
