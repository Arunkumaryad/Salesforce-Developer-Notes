# Apex Update DML Operation | SOQL Update Query Example

# Introduction

DML (Data Manipulation Language) is used in Apex to perform operations on Salesforce records.

The four major DML operations are:

1. Insert
2. Update
3. Delete
4. Undelete

In this tutorial, we will learn the **Update DML Operation**, which is used to modify existing records in Salesforce using Apex code.

---

# What is Update DML?

## Definition

The **Update DML operation** is used to modify existing records in Salesforce.

When an update operation is executed:

- Existing records are retrieved from Salesforce.
- Required field values are changed.
- Updated records are saved back to the database.

The record Id remains the same because only field values are modified.

---

# Why Use Update DML?

Update DML is used when:

- A student's roll number changes.
- A customer's status changes.
- An employee's department changes.
- Existing Salesforce records need modification.

---

# Syntax of Update DML

## Single Record Update

```apex
ObjectName obj =
[SELECT Id, FieldName FROM ObjectName LIMIT 1];

obj.FieldName = 'New Value';

update obj;
```

---

## Multiple Record Update

```apex
List<ObjectName> objList =
[SELECT Id, FieldName FROM ObjectName];

for(ObjectName obj : objList)
{
    obj.FieldName = 'New Value';
}

update objList;
```

---

# Example 1: Update Multiple Student Records

The following code retrieves Student records and updates their Roll Number.

```apex
List<Student1__c> stdList =
[
    SELECT Id,
           Name,
           Roll_Number__c,
           Branch__c
    FROM Student1__c
];

for(Student1__c student : stdList)
{
    student.Roll_Number__c = 7;
}

update stdList;
```

---

# Explanation

## Step 1: Retrieve Records

```apex
List<Student1__c> stdList =
[
    SELECT Id,
           Name,
           Roll_Number__c,
           Branch__c
    FROM Student1__c
];
```

This query fetches all Student records from Salesforce.

---

## Step 2: Loop Through Records

```apex
for(Student1__c student : stdList)
```

Iterates through each Student record.

---

## Step 3: Modify Field Value

```apex
student.Roll_Number__c = 7;
```

Changes the Roll Number field value.

---

## Step 4: Save Changes

```apex
update stdList;
```

Updates all modified records in Salesforce.

# Database.update() with allOrNone Parameter

Database methods provide additional control over DML operations.

---

# Syntax

```apex
Database.update(records, allOrNone);
```

---

## Example

```apex
Database.update(stdList, true);
```

or

```apex
Database.update(stdList, false);
```

---

# Understanding allOrNone

## allOrNone = true

```apex
Database.update(stdList, true);
```

### Definition

If even one record fails validation or encounters an error, Salesforce rolls back the entire transaction.

### Result

- All records update successfully.
- OR all records fail.

### Partial Success

❌ Not Allowed

---

## allOrNone = false

```apex
Database.update(stdList, false);
```

### Definition

Successful records are updated even if some records fail.

### Result

- Valid records are updated.
- Invalid records return errors.

### Partial Success

✅ Allowed

---

# Example

```apex
List<Student1__c> stdList =
[
    SELECT Id,
           Name,
           Roll_Number__c,
           Branch__c
    FROM Student1__c
];

for(Student1__c student : stdList)
{
    student.Roll_Number__c = 11;
}

update stdList;
// Partial success not allowed

Database.update(stdList, false);
// Partial success allowed
```

# Difference Between update and Database.update

| Feature | update | Database.update |
|----------|---------|----------------|
| Standard DML | ✅ Yes | ❌ No |
| Exception Handling Control | ❌ Limited | ✅ Better |
| Partial Success Support | ❌ No | ✅ Yes |
| allOrNone Parameter | ❌ No | ✅ Yes |

# Best Practices

## Use Bulk Update

Good:

```apex
update stdList;
```

Bad:

```apex
for(Student1__c s : stdList)
{
    update s;
}
```

---

## Query Only Required Fields

Good:

```apex
SELECT Id, Name
FROM Student1__c
```

Avoid querying unnecessary fields.

---

## Use Collections

Always update records in Lists whenever possible.

---

## Handle Exceptions

```apex
try
{
    update stdList;
}
catch(Exception e)
{
    System.debug(e.getMessage());
}
```

---

## Use Database.update(false)

When partial success is required.

```apex
Database.update(records, false);
```

---

# Governor Limits

Salesforce allows:

```text
150 DML statements per transaction
```

Bad Practice:

```apex
for(Student1__c s : stdList)
{
    update s;
}
```

Good Practice:

```apex
update stdList;
```

# Interview Questions

## What is Update DML in Salesforce?

Update DML is used to modify existing Salesforce records.

---

## Why must Id be queried before update?

Salesforce uses the record Id to identify which record should be updated.

---

## What is the difference between update and Database.update?

Database.update provides additional control and supports partial success.

---

## What does allOrNone = false mean?

Valid records are updated even if some records fail.

---

## Does update create a new record?

No.

It modifies an existing record and keeps the same record Id.

---

# Key Points for Revision

- Update DML modifies existing Salesforce records.
- Records must contain an Id before update.
- Use update keyword for standard DML updates.
- Use Database.update() for advanced control.
- allOrNone = true means partial success is not allowed.
- allOrNone = false means partial success is allowed.
- Always perform bulk updates using Lists.
- Avoid DML statements inside loops.
- Update operations count against governor limits.
- Database.update(false) is preferred when some records may fail.
- The record Id never changes during update operations.
