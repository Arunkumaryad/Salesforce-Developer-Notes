# Apex Insert DML Operation | Create Record in Salesforce Using Apex | SOQL Insert

# Introduction

DML (Data Manipulation Language) is used in Apex to perform operations on Salesforce records.

The most commonly used DML operations are:

1. Insert
2. Update
3. Delete
4. Undelete

In this tutorial, we will focus on the **Insert DML Operation**, which is used to create new records in Salesforce using Apex code.

---

# What is Insert DML?

# Definition

The **Insert DML operation** is used to save new records into Salesforce database from Apex code.

When the insert statement executes successfully:

- A new record is created.
- Salesforce generates a unique Record Id.
- The record becomes available in the object.

---

# DML Operations in Salesforce

| Operation | Purpose |
|------------|----------|
| Insert | Create new records |
| Update | Modify existing records |
| Delete | Remove records |
| Undelete | Restore deleted records |

---

# Syntax of Insert DML

## Single Record Insert

```apex
ObjectName obj = new ObjectName();
obj.FieldName = 'Value';

insert obj;
```

---

## Multiple Record Insert

```apex
List<ObjectName> objList = new List<ObjectName>();

objList.add(record1);
objList.add(record2);

insert objList;
```

---

# Example 1: Insert a Single Record

The following code creates a Teacher record.

```apex
Teacher__c teacher =
new Teacher__c(
    Name = 'Madhur',
    Dept__c = 'IT'
);

insert teacher;
```

---

# Explanation

## Step 1: Create Object Instance

```apex
Teacher__c teacher =
new Teacher__c(
    Name = 'Madhur',
    Dept__c = 'IT'
);
```

Creates a new Teacher record in memory.

---

## Step 2: Save Record

```apex
insert teacher;
```

Inserts the record into Salesforce database.

---

# Record Created

After execution Salesforce creates a new Teacher record.

Example:

| Teacher Name | Department |
|-------------|------------|
| Madhur | IT |

# Example 2: Insert Multiple Records

Instead of inserting records one by one, Salesforce allows inserting multiple records using a List.

```apex
Teacher__c teacher1 =
new Teacher__c(
    Name='Madhur1',
    Dept__c='IT'
);

Teacher__c teacher2 =
new Teacher__c(
    Name='Madhur2',
    Dept__c='Chem'
);

Teacher__c teacher3 =
new Teacher__c(
    Name='Madhur3',
    Dept__c='Civil'
);

List<Teacher__c> techList =
new List<Teacher__c>();

techList.add(teacher1);
techList.add(teacher2);
techList.add(teacher3);

insert techList;
```

---

# Explanation

## Create Multiple Records

```apex
Teacher__c teacher1
Teacher__c teacher2
Teacher__c teacher3
```

Creates three Teacher records.

---

## Add Records to List

```apex
techList.add(teacher1);
techList.add(teacher2);
techList.add(teacher3);
```

Adds records into a List collection.

---

## Insert Records

```apex
insert techList;
```

All records are inserted using a single DML statement.

---

# Result

The following records are created:

| Teacher Name | Department |
|-------------|------------|
| Madhur1 | IT |
| Madhur2 | Chem |

# Database.insert() Method

Apex also provides the Database class for performing DML operations.

---

# Syntax

```apex
Database.insert(records);
```

---

# Example

```apex
Database.insert(techList);
```

---

# Database.insert() with allOrNone Parameter

The allOrNone parameter controls partial success behavior.

---

# Syntax

```apex
Database.insert(records, allOrNone);
```

---

# Example

```apex
Database.insert(techList, true);
```

or

```apex
Database.insert(techList, false);
```

---

# Understanding allOrNone

## allOrNone = true

```apex
Database.insert(techList, true);
```

### Definition

If even one record fails, Salesforce rolls back the entire transaction.

### Result

- All records succeed OR
- All records fail

### Partial Success

❌ Not Allowed

---

## allOrNone = false

```apex
Database.insert(techList, false);
```

### Definition

Valid records are inserted even if some records fail.

### Result

- Successful records are saved.
- Failed records return errors.

### Partial Success

✅ Allowed

---

# Comparison

| Method | Partial Success |
|----------|----------------|
| insert records | No |
| Database.insert(records) | No |
| Database.insert(records, true) | No |
| Database.insert(records, false) | Yes |

# Bulk Record Creation Using Loop

Instead of manually creating records, we can generate records dynamically.

```apex
List<Teacher__c> teacherList =
new List<Teacher__c>();

for(Integer i = 0; i < 5; i++)
{
    Teacher__c tech =
    new Teacher__c();

    tech.Name = 'Student ' + i;
    tech.Dept__c = 'Chemicals';

    teacherList.add(tech);
}

insert teacherList;
```

---

# Explanation

### Create Empty List

```apex
List<Teacher__c> teacherList =
new List<Teacher__c>();
```

Stores Teacher records.

---

### Loop Execution

```apex
for(Integer i = 0; i < 5; i++)
```

Runs five times.

---

### Create Records Dynamically

```apex
tech.Name = 'Student ' + i;
```

Creates:

```text
Student 0
Student 1
Student 2
Student 3
Student 4
```

---

### Insert All Records

```apex
insert teacherList;
```

Creates all records using one DML statement.

---

# Why Use Bulk Insert?

Instead of:

```apex
insert teacher1;
insert teacher2;
insert teacher3;
```

Use:

```apex
insert teacherList;
```

Benefits:

- Better performance
- Fewer DML statements
- Governor limit friendly
- Faster execution

---

# DML Governor Limits

Salesforce imposes limits to ensure platform performance.

### Maximum DML Statements

```text
150 DML statements per transaction
```

### Recommended

```apex
insert teacherList;
```

instead of

```apex
insert teacher1;
insert teacher2;
insert teacher3;
```

---

# Best Practices

## Use Bulk DML

Good:

```apex
insert teacherList;
```

Bad:

```apex
insert teacher1;
insert teacher2;
insert teacher3;
```

---

## Use Collections

Use:

```apex
List<Teacher__c>
```

for bulk processing.

---

## Use Database.insert(false)

When partial success is required.

```apex
Database.insert(records, false);
```

---

## Handle Exceptions

```apex
try
{
    insert teacher;
}
catch(Exception e)
{
    System.debug(e.getMessage());
}
```
