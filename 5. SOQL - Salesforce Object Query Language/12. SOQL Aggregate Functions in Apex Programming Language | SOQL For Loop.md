# SOQL Aggregate Functions in Apex Programming Language | SOQL For Loop

# Introduction

SOQL Aggregate Functions can be used directly inside Apex code to perform calculations on Salesforce data without retrieving every record individually.

Instead of looping through all records and calculating values manually, Aggregate Functions allow Salesforce to perform calculations at the database level, making code faster and more efficient.

Common aggregate functions used in Apex are:

- AVG()
- SUM()
- MIN()
- MAX()
- COUNT()
- COUNT_DISTINCT()

The result of an aggregate query is stored inside an **AggregateResult** object.

---

# What is AggregateResult?

## Definition

**AggregateResult is a special Apex data type used to store the results returned by SOQL Aggregate Functions.**

Whenever an aggregate query is executed, Salesforce returns the calculated values inside AggregateResult records.

Example:

```apex
AggregateResult[] result =
[
    SELECT AVG(Amount)
    FROM Opportunity
];
```

---

# Why Use Aggregate Functions?

Without Aggregate Functions, we need to retrieve all records and perform calculations manually.

Example:

```apex
List<Opportunity> oppList =
[
    SELECT Amount
    FROM Opportunity
];

for(Opportunity opp : oppList)
{
    System.debug('Amount : ' + opp.Amount);
}
```

### Output

```text
Amount : 15000
Amount : 125000
Amount : 220000
Amount : 85000
Amount : 15000
Amount : 270000
Amount : 120000
Amount : 250000
```

This approach retrieves every record.

For large datasets, Aggregate Functions are more efficient because Salesforce performs the calculation directly in the database.

# Program Code 1 – Aggregate Query Without Alias

## Average Opportunity Amount

```apex
AggregateResult[] result =
[
    SELECT AVG(Amount)
    FROM Opportunity
];

for(AggregateResult res : result)
{
    System.debug('AVG Amount : ' + res.get('expr0'));
}
```

### Output

```text
AVG Amount : 1566883014.309525
```

---

# Why expr0?

When aggregate functions are used without aliases, Salesforce automatically assigns names:

| Aggregate Function | Auto Generated Name |
|-------------------|--------------------|
| First Function | expr0 |
| Second Function | expr1 |
| Third Function | expr2 |
| Fourth Function | expr3 |

# Program Code 2 – Multiple Aggregate Functions Without Alias

```apex
AggregateResult[] result =

    FROM Opportunity
];

for(AggregateResult res : result)
{
    System.debug(

        + ' MAX Amount : ' + res.get('expr1')
    );
}
```

### Output

```text
AVG Amount : 1566883014.309525
MAX Amount : 656546461248.00
```

# Aggregate Query with GROUP BY

We can use GROUP BY in Apex exactly as we use it in SOQL.

Example:

```apex
AggregateResult[] result =
[
    SELECT LeadSource,
           AVG(Amount),
           MAX(Amount)
    FROM Opportunity
    GROUP BY LeadSource
];

for(AggregateResult res : result)
{
    System.debug(
        'LeadSource : ' + res.get('LeadSource')
        + ' AVG Amount : ' + res.get('expr0')
        + ' MAX Amount : ' + res.get('expr1')
    );
}
```

---

## Output

```text
LeadSource : null
AVG Amount : 450.0
MAX Amount : 450.00

LeadSource : Phone Inquiry
AVG Amount : 16050968930.512196
MAX Amount : 656546461248.00
```

### Explanation

The query groups Opportunity records by LeadSource and returns:

- Lead Source
- Average Amount
- Maximum Amount

for each group.

---

# Using Alias in Aggregate Queries

## Definition

An **Alias** is a custom name assigned to an aggregate function result.

Aliases make code more readable because we do not need to use expr0, expr1, expr2, etc.

---

## Syntax

```apex
SELECT AVG(FieldName) AliasName
FROM ObjectName
```

---

# Program Code 3 – Aggregate Query Using Alias

```apex
AggregateResult[] result =
[
    SELECT LeadSource,
           AVG(Amount) avgAmount,
           MAX(Amount) maxAmount
    FROM Opportunity
    GROUP BY LeadSource
];

for(AggregateResult res : result)
{
    System.debug(
        'LeadSource : ' + res.get('LeadSource')
        + ' AVG Amount : ' + res.get('avgAmount')
        + ' MAX Amount : ' + res.get('maxAmount')
    );
}
```

---

## Output

```text
LeadSource : null
AVG Amount : 450.0
MAX Amount : 450.00

LeadSource : Phone Inquiry
AVG Amount : 16050968930.512196
MAX Amount : 656546461248.00
```

### Benefits of Alias

- Improves readability
- Easy to understand
- Eliminates expr0, expr1 confusion
- Recommended for production code

# SOQL For Loop with Aggregate Query

Aggregate Queries can also be used directly inside a SOQL For Loop.

---

## Program Code 4

```apex
for(
    AggregateResult res :
    [
        SELECT LeadSource,
               AVG(Amount) avgAmount,
               MAX(Amount) maxAmount
        FROM Opportunity
        GROUP BY LeadSource
    ]
)
{
    System.debug(
        'LeadSource : ' + res.get('LeadSource')
        + ' AVG Amount : ' + res.get('avgAmount')
        + ' MAX Amount : ' + res.get('maxAmount')
    );
}
```

---

## Output

```text
LeadSource : null
AVG Amount : 450.0
MAX Amount : 450.00

LeadSource : Phone Inquiry
AVG Amount : 16050968930.512196
MAX Amount : 656546461248.00
```

---

# Difference Between Normal Query and Aggregate Query

| Normal SOQL Query | Aggregate Query |
|------------------|----------------|
| Returns records | Returns calculated values |
| Stored in SObject lists | Stored in AggregateResult |
| Retrieves complete data | Retrieves summarized data |
| Uses Opportunity, Account etc. | Uses AggregateResult |
| Suitable for record processing | Suitable for reporting and analytics |

# AggregateResult Methods

## get()

Used to retrieve values from an AggregateResult.

Syntax:

```apex
result.get('FieldOrAliasName')
```

Example:

```apex
res.get('avgAmount');
```

Example:

```apex
res.get('LeadSource');
```

Example:

```apex
res.get('expr0');
```

# Interview Questions

## What is AggregateResult in Apex?

AggregateResult is a special Apex object used to store results returned by aggregate SOQL queries.

---

## Why do we use Aggregate Functions?

To summarize data and perform calculations directly at the database level.

---

## What is expr0?

expr0 is the default name Salesforce assigns to the first aggregate function when no alias is provided.

---

## What is the advantage of Alias?

Aliases provide meaningful names for aggregate results and improve code readability.

---

## Can Aggregate Queries be used in SOQL For Loops?

Yes.

Aggregate queries can be directly used inside SOQL For Loops.

Example:

```apex
for(AggregateResult res :
    [SELECT AVG(Amount) avgAmount FROM Opportunity])
{
    System.debug(res.get('avgAmount'));
}
```

---

## Can we use GROUP BY in Aggregate Queries?

Yes.

Example:

```apex
SELECT LeadSource,
       AVG(Amount)
FROM Opportunity
GROUP BY LeadSource
```

---

## Which object stores aggregate query results?

```apex
AggregateResult
```

# Key Points for Revision

- Aggregate Functions summarize Salesforce data.
- Aggregate query results are stored in AggregateResult.
- Use get() to retrieve aggregate values.
- Without alias Salesforce generates expr0, expr1, expr2.
- Alias makes aggregate queries easier to read.
- GROUP BY creates grouped aggregate results.
- Aggregate queries can be used in SOQL For Loops.
- Aggregate queries improve performance because calculations happen in the database.
- AggregateResult is heavily used in reports, dashboards, analytics, and trigger optimization.
- Using aliases is considered a best practice in Apex development.
