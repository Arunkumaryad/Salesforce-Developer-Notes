# Apex Primitive Data Types | Integer, Long, Double, Decimal, Blob, Id, etc.

## Introduction

Data Types in Apex define the type of data a variable can store.

Apex provides:
- Primitive Data Types
- Derived/Structured Data Types

Understanding data types is important for:
- Apex programming
- Variable declaration
- Memory management
- Business logic implementation

---

# Primitive Data Types

## Definition

Basic Data Types which can not be defined in terms of other data types.

Primitive data types store:
- Single values
- Simple values
- Basic information

---

# Derived/Structured Data Types

## Definition

Data Types which can be defined in terms of other data types.

Example:
```text
Class
```

---

# Syntax of Variable Declaration

```apex
Data_Type Variable = Literal;
```

Example:

```apex
Integer x = 3;
```

---

# Primitive Data Types in Apex

| S.N. | Data Type | Description |
|------|------------|-------------|
| 1 | Integer | Used to save integer values |
| 2 | Long | Used to save large integer values |
| 3 | Double | Used to save decimal point values |
| 4 | Decimal | Used to save precise decimal values |
| 5 | String | Used to save string values |
| 6 | Boolean | Used to save true or false |
| 7 | Date | Used to save date |
| 8 | Time | Used to save time |
| 9 | Datetime | Used to save date and time |
| 10 | Blob | Saves binary data |
| 11 | Id | Used to save Salesforce record Id |

---

# 1. Integer

## Definition

- Used to save integer values
- Variable occupies 32 bit memory
- Range:
```text
-2^31 to 2^31 - 1
```

---

# Example

```apex
Integer a = 3;
```

---

# 2. Long

## Definition

- Used to save integer values
- Variable occupies 64 bit memory
- Range:
```text
-2^63 to 2^63 - 1
```

---

# Example

```apex
Long b = 5L;
```

---

# 3. Double

## Definition

- Used to save decimal point values
- Example:
```text
3.2
```
- Variable occupies 64 bit memory

---

# Example

```apex
Double c = 3.2;
```

---

# 4. Decimal

## Definition

- Used to save decimal point values
- Example:
```text
3.2
```
- Decimals have no bounds and can grow to any size

---

# Example

```apex
Decimal d = 5.7;
```

---

# 5. String

## Definition

- Used to save string values
- Example:
```text
'Hello World'
```
- Value should be enclosed in single quote

Example:
```text
'Parag Jambhulkar'
```

---

# Example

```apex
String e = 'Arun Kumar';
```

---

# 6. Boolean

## Definition

Used to save boolean values:
```text
true or false
```

---

# Example

```apex
Boolean f = true;
```

---

# 7. Date

## Definition

Used to save date values.

---

# Example

```apex
Date g = Date.newInstance(2026,05,23);
```

---

# 8. Time

## Definition

Used to save time values.

---

# Example

```apex
Time h = Time.newInstance(06,30,32,5);
```

---

# 9. Datetime

## Definition

Used to save both:
- Date
- Time

---

# Example

```apex
Datetime i = DateTime.newInstance(g,h);
```

---

# 10. Blob

## Definition

- Saves binary data
- Used to save:
  - Images
  - Audio files
  - Multimedia files

---

# Example

```apex
Blob j = Blob.valueOf('arun');
```

---

# 11. Id

## Definition

- Used to save Salesforce:
  - Record Id
  - Object Id
  - Metadata Id

- It can be:
  - 15 digits
  - 18 digits

---

# Important Notes About Id

- 15 digit Id is case sensitive
- 18 digit Id is case insensitive
- If you allocate 15 digit Id then it gets automatically converted into 18 digit Id

---

# Example

```apex
Id k = '00D5B000001DVM9t';
```

---

# Important Note

If you are not assigning any value to variable, then by default its value is set to:
```text
null
```

---

# Complete Apex Program Example

```apex
public class DataTypes {

    public Integer a = 3;

    public Long b = 5L;

    public Double c = 3.2;

    public Decimal d = 5.7;

    public String e = 'Arun Kumar';

    public Boolean f = true;

    public Date g = Date.newInstance(2026,05,23);

    public Time h = Time.newInstance(06,30,32,5);

    public Datetime i = DateTime.newInstance(g,h);

    public Blob j = Blob.valueOf('arun');

    public Id k = '00D5B000001DVM9t';

    public void show() {

        System.debug('value of Integer a : ' + a);

        System.debug('value of long b : ' + b);

        System.debug('value of double c : ' + c);

        System.debug('value of decimal d : ' + d);

        System.debug('value of string e : ' + e);

        System.debug('value of boolean f : ' + f);

        System.debug('value of date g : ' + g);

        System.debug('value of time h : ' + h);

        System.debug('value of datetime i : ' + i);

        System.debug('value of blob j : ' + j);

        System.debug('value of id k : ' + k);
    }
}
```

---

# Execute Anonymous Window Code

```apex
DataTypes obj = new DataTypes();

obj.show();
```

|--------|---------|

| Limited precision | High precision |
| Faster calculations | Better for financial calculations |

# Interview Questions

## What are Primitive Data Types in Apex?
Primitive Data Types are basic data types that cannot be defined in terms of other data types.

---

## Difference between Integer and Long?
- Integer uses 32 bit memory
- Long uses 64 bit memory

---

## Difference between Double and Decimal?
Decimal provides higher precision and is preferred for financial calculations.

---

## What is Blob in Apex?
Blob is used to store binary data like images, audio, and multimedia files.

---

## What is Salesforce Id?
Id data type stores Salesforce record Ids and metadata Ids.

---

## Difference between 15 digit and 18 digit Salesforce Id?
- 15 digit Id is case sensitive
- 18 digit Id is case insensitive

# Key Revision Points

- Primitive Data Types store simple values
- Integer → 32 bit
- Long → 64 bit
- Decimal supports high precision
- Blob stores binary data
- Id stores Salesforce record Ids
- Default variable value is null
- 18 digit Id is case insensitive
