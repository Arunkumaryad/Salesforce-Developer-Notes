# Apex Access Modifiers | Access Modifiers in Apex Class Salesforce

## Introduction

Access Modifiers in Apex are used to control the accessibility and visibility of:
- Variables
- Methods
- Classes
- Constructors

They define from where a class member can be accessed.

Access modifiers help provide:
- Security
- Encapsulation
- Controlled access
- Better code organization

---

# Access Modifiers in Apex

1. Private
2. Protected
3. Public
4. Global

---

# Accessibility Level

```text
Private < Protected < Public < Global
```

As we move from:
```text
Private → Global
```

the accessibility increases.

---

# 1. Private Access Modifier

## Definition

`private` members are accessible only inside the same class.

They cannot be accessed:
- Outside the class
- By child classes
- By other classes

---

# Example of Private Access Modifier

```apex
public class AccessModifiers {

    Integer a = 10;

    void showValue() {
        System.debug('Value of Variables : ' + a);
    }
}
```

---

# Execute Anonymous Code

```apex
AccessModifiers obj = new AccessModifiers();

System.debug('Value is : ' + obj.a);

AccessModifiers.showValue();
```

---

# Output/Error

```text
Variable is not visible: AccessModifiers.a
```

---

# Explanation

Since variable `a` is private by default, it cannot be accessed outside the class.

That is why Salesforce generates:
```text
Variable is not visible
```

error.

---

# Important Point

In Apex:
- Variables and methods are private by default if no access modifier is specified.

---

# 2. Public Access Modifier

## Definition

`public` members can be accessed:
- Inside the same class
- Outside the class
- Within the same Salesforce org

---

# Example of Public Access Modifier

```apex
public class AccessModifiers {

    public Integer a = 10;

    public void showValue() {

        System.debug('Value of Variables : ' + a);
    }
}
```

---

# Execute Anonymous Code

```apex
AccessModifiers obj = new AccessModifiers();

System.debug('Value is : ' + obj.a);

obj.showValue();
```

---

# Output

```text
Value is : 10
Value of Variables : 10
```

---

# Explanation

Since variable `a` and method `showValue()` are public, they can be accessed outside the class.

---

# 3. Protected Access Modifier

## Definition

`protected` members are accessible:
- Inside the same class
- By child classes using inheritance

They cannot be accessed by unrelated classes.

---

# Example of Protected Access Modifier

## Parent Class

```apex
abstract public class AccessModifiers {

    protected Integer a = 10;

    public void showValue() {

        System.debug('Value of Variables : ' + a);
    }
}
```

---

# Child Class

```apex
public class B extends AccessModifiers {

    public void display() {

        System.debug('Value inside the B class : ' + a);
    }
}
```

---

# Execute Anonymous Code

```apex
B obj = new B();

obj.display();
```

---

# Output

```text
Value inside the B class : 10
```

---

# Explanation

Variable `a` is protected.

So:
- Child class `B`
- Can access variable `a`
- Using inheritance

---

# 4. Global Access Modifier

## Definition

`global` members can be accessed:
- Inside the same org
- Outside the org
- Through managed packages
- Through AppExchange applications

Global is the highest accessibility level in Apex.

---

# Example Scenario of Global

```text
Org1  ←→  AppExchange  ←→  Org2
```

Global classes and methods can be shared across organizations.

---

# Difference Between Access Modifiers

| Access Modifier | Accessibility |
|-----------------|---------------|
| Private | Same class only |
| Protected | Same class + Child class |
| Public | Entire Salesforce org |
| Global | Across orgs and packages |

---

# Important Notes

- Default access modifier in Apex is `private`
- `global` is mainly used in managed packages
- `protected` supports inheritance
- `public` is commonly used in Apex classes and methods
